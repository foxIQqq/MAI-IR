В openSUSE Build Service 1.5 появилась поддержка сборки дистрибутивов

Анонсирован выход инструментария openSUSE Build Service 1.5 (OBS) в котором теперь поддерживается возможность создания собственного дистрибутива, ранее возможности системы были ограничены лишь сборкой пакетов. Кроме того, OBS 1.5 поддерживает средства для автоматического учета зависимостей, создания дерева директорий для размещения на FTP и подготовки установочных образов, включая LiveCD, LiveUSB и образы для виртуальных машин Xen и VMware. Также в новой версии появилось средство для создания дополнений к продукту, таких как проприетарные дополнения (nonfree) к openSUSE. При этом сервис полностью опробован в промышленной эксплуатации, сборка релиза openSUSE 11.1 была полностью проведена при помощи OBS. В версии 1.5 также добавлены экспериментальные средства для кросс-архитектурной сборки (например, ARM), фильтрации результатов сборки через web-интерфейс мониторинга (например, следить только за ошибками или сборкой под определенный дистрибутив) и возможность формирования пакета для загрузки по запросу (пользователь может сам задать для какого дистрибутива ему нужен пакет и система ему такой пакет подготовит). Кроме OpenSUSE, система поддерживает Linux дистрибутивы CentOS, Debian, Fedora, Mandriva, Red Hat Enterprise Linux и Ubuntu. Для разработчиков программ openSUSE Build Service предоставляет один из самых простых путей поддержания пакетов для всех популярных дистрибутивов, использующих пакеты в формате RPM или DEB. Например, можно создать spec файл или подключить репозиторий пакетов представленный на сайте software.opensuse.org и одной командой собрать последнюю версию заданной программы в виде бинарного пакета под нужную систему.
