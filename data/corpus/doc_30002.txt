Доступна первая бета-версия Red Hat Enterprise Linux 6.1

Компания Red Hat анонсировала начало бета-тестирования дистрибутива Red Hat Enterprise Linux 6.1, в котором представлено исправление ошибок, реализация поддержки нового оборудования и добавление новых возможностей. Установочные образы с тестовой версией RHEL 6.1 доступны для загрузки только зарегистрированным пользователям Red Hat Network (RHN). Бета-версия поставляется в следующих вариантах: Red Hat Enterprise Linux Client 6 для архитектур x86 и AMD64/Intel 64 Red Hat Enterprise Linux High Performance Compute Node 6 для архитектуры AMD64/Intel 64 Red Hat Enterprise Linux Server для архитектур x86, AMD64/Intel 64, IBM System z и IBM POWER; Red Hat Enterprise Linux Workstation для архитектур x86 и AMD64/Intel 64. Ключевые новшества Red Hat Enterprise Linux 6.1: Оптимизация производительности механизмов виртуализации, главным образом направленная на приближение производительности запущенных в виртуальных окружениях систем к скорости выполнения без использования средств виртуализации. Уменьшены задержки (latency) выполнения сетевых операций за счет встраивания соответствующей функциональности в ядро; Повышение производительности дискового ввода/вывода за счет поддержки механизма virtqueue-нотификации ioeventfd, используемого в KVM и позволяющего увеличить производительность virtio-драйверов; Поддержка кэширования мета-данных для образов виртуальных машин в формате qcow2, что позволяет сократить число выполняемых операций ввода/вывода; Оптимизация использования FPU-регистров; Расширение поддержки оборудования и увеличение надёжности, поддержка новейших многоядерных CPU и периферийных устройств Планировщик задач расширен средствами для более полной поддержки возможностей архитектуры NUMA; Добавлена возможность динамического добавления памяти и процессоров для некоторых серверных систем; Обеспечена поддержка интерфейса PCI express 3.0, позволяющего добиться более высокой пропускной способности; Обновлено большое количество разнообразных драйверов для сетевых адаптеров, устройств хранения и графических плат; Повышена операционная эффективность, расширены возможности системных сервисов, добавлены новые механизмы контроля за ресурсами и производительностью Добавлена поддержка ограничения интенсивности ввода/вывода через группы контроля ресурсами Cgroup; Увеличена эффективность выполнения сетевых операций на многоядерных системах за счет оптимизации привязки к CPU потока пакетов (пакеты в рамках TCP-сессий неразрывно обрабатываются одним и тем же ядром CPU); В device-mapper-multipath увеличена производительность обработки сбоев при обращении к хранилищу; Улучшена поддержка сетевых хранилищ, включая поддержку бриджинга для FCoE и offload-акселерации для iSCSI. Улучшена поддержка IPv6 в работе виртуальных сетей и DHCP-сервисов; Добавлена поддержка создания прозрачных прокси на базе squid; Новые инструменты для разработки и мониторинга. Расширение средств для диагностики, основанных на использовании системы динамической трассировки systemtap; В отладчике Gdb улучшена поддержка C++ и Python; В системе отслеживания потребления памяти Valgrind улучшена поддержка многоядерных CPU; Обновлена среда разработки Eclipse, добавлены новые средства для работы с точками останова и генерации кода для C/C++ и Java; Обеспечение повышенной отказоустойчивости Обеспечена поддержка инфраструктуры для обеспечения автоматической миграции KVM-систем на запасной сервер в случае сбоя первичного; Добавлен агент для мониторинга NFSv4-ресурсов; Значительно расширена документация.
