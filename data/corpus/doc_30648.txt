Релиз Linux-дистрибутива Fedora 15

Официально представлен релиз Linux-дистрибутива Fedora 15. Для загрузки доступны варианты установочного LiveCD с графическими окружениями на базе KDE, GNOME, XFCE и LXDE, а также полный комплект, оформленный в виде DVD и сокращенный образ (200 Мб) для установки по сети. Из ключевых улучшений Fedora 15 можно отметить переход на системный менеджер systemd, изменение наименования сетевых интерфейсов, интеграцию десктоп-окружения GNOME 3 с оболочкой GNOME Shell, сборку пакетов при помощи GCC 4.6, использование Linux-ядра 2.6.38, добавление реализации динамического межсетевого экрана, задействование по умолчанию DNSSEC и переход на использование директории "/run" вместо "/var/run". Основные новшества, появившиеся в Fedora 15: Пользователю Интеграция графического окружения GNOME 3 и использование по умолчанию оболочки GNOME Shell. С обзором новшеств GNOME 3 можно познакомиться здесь. Дополнительно доступны сборки дистрибутива с графическими окружениями KDE 4.6 и Xfce 4.8; Использование вместо OpenOffice.org офисного пакета LibreOffice; В поставку включена последняя версия обучающей оболочки Sugar 0.92; Использование веб-браузера Firefox 4, поставляемого с полностью открытым Java-плагином IcedTea-Web; Обновление видеодрайверов: Intel 2.14.0 c поддержкой Sandybridge; Radeon 6.14.0 с поддержкой 2D, 3D и видеоакселерации для GPU Evergreen, а также 2D и видеоакселерации для GPU Northern Islands; Nouveau с поддержкой 2D, 3D и видеоакселерации для GPU Fermi; Использование X.Org-сервера 1.10 и предварительной версии Mesa 7.11; Разработчику Интеграция в дистрибутив Python 3.2 и новых версий Rails (3.0.3) и OCaml (3.12). Обновление библиотек Boost до версии 1.46; Интеграция системы сборки и управления Java-проектами Maven 3; Обновление отладчика GDB до версии 7.3 (релиз ещё не вышел), включение в состав набора компиляторов GCC 4.6; Системные изменения Интеграция системного менеджера systemd в качестве системы инициализации по умолчанию. Systemd нацелен на более интенсивную параллелизацию выполнения сервисов на этапе загрузки системы, вобрав в себя лучшие черты таких систем, как launchd (Mac OS X), SMF (Solaris) и Upstart (Ubuntu, Fedora). Кроме запуска сервисов, systemd обеспечивает функции контроля за выполнением фоновых процессов. Подробнее о systemd можно прочитать в ранее опубликованных обзорах (здесь, здесь, здесь, здесь и здесь). Сравнение систем инициализации systemd, upstart и SysVinit можно посмотреть здесь; Перенос "/run" из директории "/var" в корень с целью решения проблемы с недоступностью /var/run на ранней стадии загрузки. Подробное обоснование целесообразности подобного шага можно прочитать здесь; Изменение наименования сетевых интерфейсов. Вместо ethN имена устройств будут формироваться в зависимости от типа подключения сетевого адаптера: "em{port}" - для встроенных в материнскую плату сетевых карт и "pci{slot}#{port}" - для карт с PCI-интерфейсом. Переход на GCC 4.6 для сборки пакетов; Улучшение работы системы управления питанием, поддержка автоматического тюнинга параметров работы системы, с целью обеспечения оптимального баланса между производительностью и энергопотреблением. Интеграция утилиты PowerTOP 2.0 для выявление узких мест, приводящих к повышенному энергопотреблению; Задействование системы сжатия XZ для формирования Live-образов позволило сократить размер образа на 10%; Добавление LessFS, FUSE-модуля для автоматического объединения дубликатов в файлах; Обновление пакетного менеджера RPM до версии 4.9, в которой реализована поддержка подключаемого генератора зависимостей и встроенной фильтрации сгенерированных зависимостей, увеличена производительность; Переход на использование Linux-ядра 2.6.38. В инсталляторе, используемом на установочном DVD, в меню добавлена возможность установки с использованием файловой системы Btrfs (ранее требовалась передача инсталлятору специальных опций). Отдельно отмечается продолжение работы над утилитой btrfsck, предназначенной для восстановления целостности поврежденных Btrfs-разделов, и улучшение работы плагина yum-plugin-fs-snapshot, позволяющего возвращать систему в первоначальное состояние после неудачной установки пакетов или обновления. Сообщается, что разработчики рассматривают возможность использования Btrfs в качестве файловой системы по умолчанию уже начиная со следующего выпуска (Fedora 16). Виртуализация и безопасность Добавление DynamicFirewal, фонового процесса для динамического управления правилами пакетного фильтра через DBus. Использование DynamicFirewall позволит вносить изменения в настройки межсетевого экрана без необходимости перезагрузки правил пакетного фильтра и без разрыва установленных соединений. Для динамического управления межсетевым экраном представлена утилита firewall-cmd, которая значительно упрощает создание правил, отталкиваясь не от IP-адресов, сетевых интерфейсов и номеров портов, а от названий служб (например, для открытия доступа к сервису samba на 90 секунд можно выполнить "firewall-cmd --enable --service=samba --timeout=10"). В Fedora 16 планируется создание дополнительных графических инструментов. В отдаленной перспективе (Fedora 17) планируется реализация интерактивного режима блокировки, при котором в случае поступления неавторизированных внешних запросов от локальных приложений или обращения из вне к сетевым сервисам, пользователю выводится интерактивная форма с предложением принять или отклонить запрос с возможностью открытия доступа только на определенное время; C целью повышения безопасности все setuid-приложения будут переведены на использование "capabilities", позволяющих выборочно организовать выполнение только требуемых привилегированных действий. Например, утилита ping получит возможность доступа только к raw-сокету (CAP_NET_RAW), без делегирования остальных root-прав; Интеграция BoxGrinder, инструмента для упрощения формирования сборок для развертывания внутри различных систем виртуализации (KVM, Xen, EC2); Улучшение поддержки шифрованной файловой системы eCryptfs. Использование authconfig для автоматического монтирования зашифрованных частей домашней директории в момент входа пользователя в систему; Использование по умолчанию DNSSEC в конфигурациях рабочих станций. Для преобразования имен в NetworkManager использует DNS-сервер BIND; Поддержка технологии Spice в virt-manager; GUI-интерфейс Setroubleshoot переработан с целью упрощения диагностики проблем с SELinux. Дополнительные программы Добавление в дистрибутив ERP-платформы Tryton, предназначенной для разработки приложений для управления ресурсами предприятия (финансы, управление продажами и закупками, аналитика, инвентаризация и т.п.); Новые группы пакетов: Graphics suite переименован в Design group, добавлена новая группа Robotics Package Suite, в которую вошли инструменты для проектирования роботов, симуляции их работы и создания программного обеспечения; Интеграция в дистрибутив тестовой версии конфигуратора NetworkManager 0.9. Обновление версий программ: Apache 2.2.17, MySQL 5.5.10, PostgreSQL 9.0.3; Одновременно представлен выход сборки Russian Fedora Remix, адаптированной для отечественных пользователей и содержащей "из коробки" полный набор мультимедиа кодеков и проприетарных драйверов. Дистрибутив включает GNOME 3, KDE 4.6.3, XFCE 4.8, LibreOffice 3.3, Kernel 2.6.38, мультимедийные кодеки, Adobe Flash, проприетарные драйверы для видеокарт ATI и NVIDIA, сетевых адаптеров Ralink и Broadcom. Развиваемые в рамках проекта RPM Fusion "free" и "nonfree" репозитории в настоящий момент недоступны для Fedora 15, но будут готовые в ближайшее время. Репозитории проекта RPM Fusion содержат пакеты с дополнительными мультимедиа приложениями (MPlayer, VLC, Xine), видео/аудио кодеками, поддержкой DVD, проприетарными драйверами ATI и Nvidia (драйвер Nividia из RPMFusion на данный момент не работает из-за ошибки 623742), игровыми программами, эмуляторами.
