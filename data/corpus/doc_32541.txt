Обновление Java SE 6 Update 30 и Java SE 7 Update 2

Доступны корректирующие выпуски Java SE 6 Update 30 и Java SE 7 Update 2. Java SE 7 Update 2 является первым корректирующим выпуском после первого стабильного релиза Java SE 7, представленного в июле (в выпуске Java SE 7 Update 1 были добавлены только устранения уязвимостей). В новой версии отмечено около 680 исправлений. Из изменений, представленных в Java SE 7 Update 2, можно отметить: До 22 версии обновлена виртуальная машина HotSpot, в которой значительно увеличена надёжность и производительность; Обеспечена поддержка Oracle Solaris 11 и Firefox 5-8. В состав интегрирован JavaFX SDK 2.0.2; Коллекция примеров и демонстрационных приложений выделена из JDK в отдельный пакет; В Deployment Toolkit добавлена поддержка установки в неблокирующем режиме; Изменён метод формирования цифровых подписей для JAR-файлов, вместо подписи для каждого элемента, теперь формируется одна подпись для всего содержимого. Подобный подход позволил уменьшить размер подписанных JAR-файлов примерно на 10%; По умолчанию активирована поддержка кэширования полученного по сети контента для приложений запущенных в режиме Web Start; В Web Start включена поддержка JNLP. Deployment Toolkit теперь может для запуска приложения использовать копию JNLP-файла, встроенную в web-страницу, что сокращает число сетевых соединений необходимых для первого запуска приложения в режиме Web Start с использованием браузера; Возможность передачи Web Star-приложениям безопасных аргументов JVM из web-страниц при помощи Deployment Toolkit, что может использоваться для передачи java-приложению динамических параметров из web-страницы; Улучшена обработка контента, сжатого методом gzip (в кэше контент сохраняется в сжатом виде); Улучшена поддержка JNLP-приложений, например, обеспечена более корректная и полная деинсталляция JavaFX-программ; В Java Control Panel добавлены настройки, позволяющие управлять выводом предупреждений для небезопасных версий JRE (свойство deployment.insecure.jres в файле deployment.properties); Проведена большая работа по улучшению совместимости с существующими проектами на языке Java. Исправлено огромное число недоработок и ошибок. В Java SE 6 Update 30 представлено 42 исправления, обеспечена поддержка Red Hat Enterprise Linux 6, обновлена база часовых поясов, осуществлён переход на новую версию системы профилирования Java VisualVM 1.3.2, устранено появившееся в версии 6u29 регрессивное изменение, которое мешало созданию SSL-соединений с использованием TLS_DH_anon_WITH_AES_128_CBC_SHA. Исправленные в Java SE 6 Update 30 и Java SE 7 Update 2 ошибки не связаны с проблемами безопасности. Последними корректирующими релизами, направленными на устранение уязвимостей, были выпуски Java SE 6 Update 29 и Java SE 7 Update 1.
