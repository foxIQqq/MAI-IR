ownCloud 1.0, попытка создания альтернативы Dropbox от проекта KDE

Вышел первый стабильный релиз, развиваемой разработчиками KDE в рамках проекта Social Desktop, системы ownCloud 1.0, нацеленной на организацию работы Cloud-хранилища, напоминающего по возможностям сервис DropBox, но работающего на собственном локальном сервере. Используя ownCloud можно обеспечить доступ к хранимым данным со всех своих устройств из любой точки сети и организовать совместный доступ к файлам группы людей. Система поддерживает автоматическое резервное копирование, версионный контроль изменений (можно получить доступ к старым версиям файлов и проследить историю изменений) и шифрование передачи данных. Доступ к хранилищу может быть обеспечен при помощи монтирования сетевого раздела, WebDAV, KDE KIO-Slaves, приложения для мобильных телефонов или через web-интерфейс. Данные также могут быть синхронизированы с локальной копией для последующего offline-использования. ownCloud поддерживает Open Collaboration Services API, что дает возможность отправки уведомлений, например, при отправке файла другим пользователем, через стандартный механизм нотификации KDE. Исходные тексты системы распространяются в рамках лицензии AGPL. Из планов на будущее отмечается: Поддержка в качестве бэкенда для хранения данных системы управления исходными текстами Git и организация версионного трекинга изменений; Создание простого механизма для организации совместного доступа к файлам других пользователей; Интеграция возможности сохранения данных в ownCloud в стандартных приложениях KDE; Создание системы синхронизации данных между несколькими ownCloud-серверами, что позволит создавать внутри организации приватные ownCloud-серверы, которые будут синхронизированы с публично доступными серверами; Написание клиента для доступа к данным в offline-режиме; Развитие API для написания плагинов, в качестве примеров возможных плагинов приводятся: плагин для совместного поддержания фотоальбома; музыкальный сервер, позволяющий прослушивать собственную музыкальную коллекцию с любого устройства в сети; хранилище подкастов и видеороликов с доступом через web-интерфейс или через медиа плеер.
