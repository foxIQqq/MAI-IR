Релиз Linux-дистрибутива Fedora 22

Официально представлен релиз Linux-дистрибутива Fedora 22. Для загрузки доступны 32- и 64-разрядные сборки продуктов Fedora Workstation, Fedora Server и Fedora Cloud, а также набор "спинов" c Live-сборками десктоп-окружений KDE, Xfce, LXDE, MATE-Compiz и SOAS (Sugar on a Stick). Дополнительно поставляется образ для Docker и сборки для различных устройств с процессорами ARM. Выпуск примечателен переходом на пакетный менеджер DNF, использованием по умолчанию в серверной сборке файловой системы XFS, переводом экрана входа в систему на Wayland, поставкой KDE Plasma 5 и Xfce 4.12. Кроме спинов началось формирование lab-сборок, в рамках которых подготовлены специализированные варианты дистрибутива для научных исследований, тестирования безопасности, игр, дизайна и создания мультимедийного контента, сочинения и обработки музыки, разработки роботов. Наиболее заметные изменения в Fedora 22: Замена YUM на пакетный менеджер DNF, обладающий заметно более высокой скоростью работы, низким потреблением памяти и более качественным управлением зависимостями. DNF является ответвлением от Yum 3.4, созданным для развития некоторых новых идей, таких как использование библиотеки hawkey в качестве бэкенда для разрешения зависимостей. C точки зрения опций командной строки и файлов кофигурации, DNF почти полностью совместим с YUM, основные изменения заложены в его внутреннюю архитектуру. Пакет dnf-yum осуществляет автоматический редирект на dnf для обеспечения полной совместимости с yum. При этом yum оставлен в дистрибутиве, но переименован в yum-deprecated. Для разрешения зависимостей в DNF задействован SAT solver, реализованный в библиотеке libsolv (hawkey выступает в роли надстройки над libsolv), созданной в рамках проекта openSUSE. Для расширения функциональности DNF предоставляет фиксированный API для плагинов и интеграции с другими приложениями, такими как инсталлятор Anaconda. В серверной сборке дистрибутива по умолчанию задействована файловая система XFS, которая даёт возможность создания хранилищ большего размера и рассматривается как более оптимальный вариант для серверных систем. Поддержка Ext4 полностью сохранена. Экран входа в систему переведён по умолчанию на использование Wayland. Экран входа является достаточно обособленной системой, поэтому переведя его на Wayland в первую очередь, у разработчиков появится возможность окончательно убедиться в отсутствии подводных камней и незамеченных ошибок, без негативного влияния на работу приложений и рабочего стола. Пользовательское окружение GNOME по умолчанию по-прежнему будет работать с использованием X-сервера, при том, что полноценный сеанс на базе Wayland предоставляется в качестве штатной опции. Переход на Wayland по умолчанию запланирован на Fedora 23; Для управления устройствами ввода в X11 и Wayland задействована библиотека libinput; В дистрибутиве задействован рабочий стол GNOME 3.16. Из улучшений можно отметить новую тему оформления GNOME Shell, переработанную систему вывода уведомлений, стабилизированную и доработанную поддержку Wayland, модернизированный интерфейс файлового менеджера Nautilus, поддержку Google Drive; Включены наработки по снижению энергопотребления. Добавлен инструмент для тестирования автономной работы от аккумулятора; В интерфейс для управления установкой приложений GNOME Software добавлена поддержка работы со сторонними компонентами, такими как мультимедийные кодеки для GStreamer и шрифты; Усовершенствованы темы оформления для Qt4 и Qt5, позволяющие обеспечить бесшовную интеграцию с GNOME Qt-приложений и программ из KDE, интерфейс которых будет унифицирован с приложениями GTK. Усовершенствован GNOME Terminal, в том числе обеспечена возможность вывода уведомлений при завершении длительно работающих заданий, таких как компиляция, что позволит не отвлекаться, постоянно заглядывая в терминал для оценки статуса завершения работы; В инструменте для автоматической отправки сообщений о проблемах ABRT (Automatic Bug Reporting Tool) улучшена организация вывода уведомлений и предоставлены усиленные средства соблюдения приватности и контроля за отправляемой вовне информацией; Спин Fedora KDE по умолчанию переведён на окружение KDE Plasma 5. Спин Fedora Xfce обновлён до Xfce 4.12; В инструментарий Rolekit добавлена поддержка развёртывания и управления серверных "ролей" на базе СУБД PostgreSQL. Под ролью подразумевается определённая серверная функциональность, работу которой обеспечивает набор пакетов (например, почтовый сервер может потребовать установки пакетов с SMTP и IMAP серверами). Rolekit позволяет разом установить и настроить все пакеты, требуемые для реализации сервиса. Обновлён Cockpit, web-интерфейс для настройки и мониторинга одного или нескольких серверов. Позволяет администратору управлять хранилищами, запуском сервисов, анализировать логи, инспектировать и администрировать контейнеры Docker, в том числе отслеживать потребление ресурсов и задавать ограничения. В редакцию для облачных систем включены последние версии пакетов rpm-ostree и rpm-ostree-toolbox, при помощи которых можно сгенерировать собственную начинку базового окружения Atomic, содержащую свой набор пакетов. В состав также включены утилиты командной строки для управления контейнерами в окружении Atomic Host и проведения обновления начинки Atomic Host. Напомним, что в рамках проекта Atomic развивается концепция минималистичной ОС для обеспечения запуска изолированных контейнеров, подготовленных и управляемых при помощи инструментария Docker. Предоставляемое Atomic базовое окружение (Atomic Host) содержит только минимальный набор компонентов (systemd, journald, docker, geard и т.п.), необходимых для запуска и управления изолированными контейнерами. Для обновления системного окружения Atomic используется технология OSTree, при которой системный образ атомарно обновляется из Git-подобного хранилища, позволяющего применять методы версионного контроля к компонентам дистрибутива (например, можно быстро откатить систему к прошлому состоянию). RPM-пакеты транслируются в репозиторий OSTree при помощи специальной прослойки rpm-ostree. Предоставлены образ Fedora Atomic Host и Fedora Cloud для использования с инструментарием Vagrant, который может использоваться для быстрого развёртывания виртуальных окружений в системах виртуализации на базе KVM и VirtualBox. Пример использования Vagrant в Fedora 22. Обновлены образы, которые можно использовать в качестве основы для запуска приложений в контейнерах Dockеr. В состав дистрибутива включен пакет fedora-dockerfiles, который в комбинации с специальным git-репозиторием, позволяет формировать сборки контейнеров приложений на основе базовой начинки Fedora 22 и дополнительных пакетов; Представлен Tunir, новый легковесный инструмент непрерывной интеграции, нацеленный на ускорение тестирования образов для облачных систем; Задействование ядра Linux 4.0 и набора компиляторов GCC 5.1. В ближайшие часы для Fedora 22 ожидается введение в строй "free" и "nonfree" репозиториев проекта RPM Fusion, в которых доступны пакеты с дополнительными мультимедиа приложениями (MPlayer, VLC, Xine), видео/аудио кодеками, поддержкой DVD, проприетарными драйверами AMD и NVIDIA, игровыми программами, эмуляторами.
