Начало бета-тестирования Fedora 15

Доступна для тестирования бета-версия Linux-дистрибутива Fedora 15. Начиная с данного момента в процессе подготовки релиза будут исправляться только серьезные ошибки, прием новшеств и несущественных исправлений прекращен. Релиз намечен на 24 мая, кандидат в релизы выйдет 10 мая. Для загрузки доступны Live-сборки с GNOME 3 (562 Мб) и KDE 4.6 (691 Мб). Из наиболее интересных улучшений, добавленных после выхода альфа-версии можно отметить перенос "/run" из директории "/var" в корень с целью решения проблемы с недоступностью /var/run на ранней стадии загрузки. Подробное обоснование целесообразности подобного шага можно прочитать здесь. Вторым важным изменением, внесённым в последний момент, стала интеграция в дистрибутив тестовой версии конфигуратора NetworkManager 0.9. Основные новшества, появившиеся в Fedora 15: Пользователю Интеграция графического окружения GNOME 3 и использование по умолчанию оболочки GNOME Shell. С обзором новшеств GNOME 3 можно познакомиться здесь. Дополнительно доступны сборки дистрибутива с графическими окружениями KDE 4.6 и Xfce 4.8; Использование вместо OpenOffice.org офисного пакета LibreOffice; В поставку включена последняя версия обучающей оболочки Sugar 0.92; Использование веб-браузера Firefox 4, поставляемого с полностью открытым Java-плагином IcedTea-Web; Разработчику Интеграция в дистрибутив Python 3.2 и новых версий Rails (3.0.3) и OCaml (3.12). Обновление библиотек Boost до версии 1.46; Интеграция системы сборки и управления Java-проектами Maven 3; Обновление отладчика GDB до версии 7.3 (релиз ещё не вышел), включение в состав набора компиляторов GCC 4.6; Системные изменения Интеграция системного менеджера systemd в качестве системы инициализации по умолчанию. Systemd нацелен на более интенсивную параллелизацию выполнения сервисов на этапе загрузки системы, вобрав в себя лучшие черты таких систем, как launchd (Mac OS X), SMF (Solaris) и Upstart (Ubuntu, Fedora). Кроме запуска сервисов, systemd обеспечивает функции контроля за выполнением фоновых процессов. Подробнее о systemd можно прочитать в ранее опубликованных обзорах (здесь, здесь, здесь, здесь и здесь); Изменение наименования сетевых интерфейсов. Вместо ethN имена устройств будут формироваться в зависимости от типа подключения сетевого адаптера: "em{port}" - для встроенных в материнскую плату сетевых карт и "pci{slot}#{port}" - для карт с PCI-интерфейсом. Переход на GCC 4.6 для сборки пакетов; Улучшение работы системы управления питанием, поддержка автоматического тюнинга параметров работы системы, с целью обеспечения оптимального баланса между производительностью и энергопотреблением. Интеграция утилиты PowerTOP 2.0 для выявление узких мест, приводящих к повышенному энергопотреблению; Задействование метода сжатия LZMA для формирования Live-образов; Добавление LessFS, FUSE-модуля для автоматического объединения дубликатов в файлах; Обновление пакетного менеджера RPM до версии 4.9; Переход на использование Linux-ядра 2.6.38. Виртуализация и безопасность C целью повышения безопасности все setuid-приложения будут переведены на использование "capabilities"; Интеграция BoxGrinder, инструмента для упрощения формирования сборок для развертывания внутри различных систем виртуализации (KVM, Xen, EC2); Добавление DynamicFirewal, фонового процесса для динамического управления правилами пакетного фильтра через DBus. Использование DynamicFirewall позволит вносить изменения в настройки межсетевого экрана без необходимости перезагрузки правил и без разрыва установленных соединений; Улучшение поддержки шифрованной файловой системы eCryptfs. Использование authconfig для автоматического монтирования зашифрованных частей домашней директории в момент входа пользователя в систему; Использование по умолчанию DNSSEC в конфигурациях рабочих станций. Для преобразования имен в NetworkManager использует DNS-сервер BIND; Поддержка технологии Spice в virt-manager; GUI-интерфейс Setroubleshoot переработан с целью упрощения диагностики проблем с SELinux. Дополнительные программы Добавление в дистрибутив ERP-платформы Tryton, предназначенной для разработки приложений для управления ресурсами предприятия (финансы, управление продажами и закупками, аналитика, инвентаризация и т.п.); Новые группы пакетов: Graphics suite переименован в Design group, добавлена новая группа Robotics Package Suite, в которую вошли инструменты для проектирования роботов, симуляции их работы и создания программного обеспечения;
