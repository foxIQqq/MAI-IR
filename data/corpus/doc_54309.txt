Отчёт о становлении дистрибутива Rocky Linux, идущего на смену CentOS

Проект Rocky Linux, нацеленный на создание новой свободной сборки RHEL, способной занять место классического CentOS, опубликовал первый отчёт. Некоторые интересные моменты: Для курирования разработки, взаимодействия со спонсорами и юридического представления проекта в США зарегистрирована некоммерческая организация The Rocky Linux Foundation. Наработки проекта планируется публиковать под лицензией BSD, но вопрос ещё окончательно не решён и обсуждается в сообществе. Основное внимание сосредоточено на подготовке инфраструктуры для сопровождения проекта и разработки первого релиза, который ожидается в начале второго квартала 2021 года. В ближайшее время будет опубликован график запуска сборочных систем, инфраструктуры для автоматической сборки пакетов, репозитория для тестирования пакетов, компонентов для тестирования инсталлятора. В качестве первичной платформы для запуска сборочных серверов будет использован сервис Amazon Web Services (AWS), что позволит упростить поддержание и обеспечение безопасности инфраструктуры. Элементы физической инфраструктуры скорее всего будут размещены в OSUOSL (Oregon State University Open Source Lab). Для организации сборочных процессов решено использовать Koji, Mock и MBS (Module Build Service), применяемые в Fedora Linux. Началась разработка скриптов, которые будут использованы для загрузки исходных текстов пакетов RHEL, ребрендинга и сборки при помощи инструментариев Koji и Mock. Из основных задач для команды, отвечающей за инфраструктуру, выделяются: Предоставление автоматических и безопасных сборок пакетов и компонентов дистрибутива, которые можно будет использовать для тестирования сообществом без опасности компрометации и нарушения доверия к результирующим пакетам. Обеспечение целостности и безопасности всей сборочной цепочки и процессов разработки (supply chain). Независимость, насколько это возможно, от отдельных поставщиков и платформ. Избегание привязки к вендорам. Прозрачность проекта для сообщества. Более 370 человек зарегистрировалось в качестве волонтёров и желающих поддержать проект. Несколько компаний выразили желание стать спонсорами проекта и готовы предоставить разработчиков, оборудование, финансирование и облачные окружения для инфраструктуры. Изначально для организации общения участников был задействован сервис Slack, но его планируется заменить на собственный сервер на базе открытой платформы Mattermost. Рассматривается возможность создания дополнительных редакций (SIG, Special Interest Groups) для поставки различных пользовательских окружений, специализированной сборки для ноутбуков, расширенной защиты, развёртывания Kubernetes/Cloud Native, минимальной установки, создания хранилищ и организации высокопроизводительных вычислений.
