Компания Red Hat возглавила разработку OpenJDK 6

Компания RedHat анонсировала принятие на себя ведущей роли в проекте OpenJDK 6. Этот шаг увеличивает вклад компании в развитие и расширение Java-комьюнити и в будущее Java как ведущей платформы для критически-важных корпоративных приложений. Кроме принятия лидерства в сообществе OpenJDK 6, RedHat будет играть ведущую роль в определении путей развития проекта OpenJDK. В совете OpenJDK компанию будет представлять Эндрю Хэйли (Andrew Haley), который от имени Red Hat примет участие в определении будущего Java и OpenJDK. Оставаясь ведущим контрибьютором сообщества OpenJDK на протяжении более пяти лет, согласно соглашению с Sun Microsystems, компания Red Hat считает, что усилия стоит направлять на улучшение общей производительности и управляемости, одновременно расширяя функциональность в области динамической масштабируемости и облачных вычислений. Решение возглавить разработку OpenJDK 6, возможно, связано с недавней новостью о сворачивании проекта IcedTea и концентрации текущих усилий Oracle исключительно на разработке Java 7+. Напомним, что компания Oracle прекратила выпуск обновлений для ветки Java SE 6, поддержка которой завершилась в феврале 2013 года, после чего обновления не будут публиковаться публично и останутся доступны только в рамках платного расширенного сервиса поддержки. Правила поддержки Java гласят, что выпуск обновлений может быть прекращён не ранее, чем спустя три года с момента выхода рассматриваемой ветки Java, не ранее, чем спустя год с момента выхода первого релиза следующей ветки Java и не ранее, чем через 6 месяцев после того как следующая ветка официально будет рекомендована для использования по умолчанию на java.com. Тем не менее, несмотря на выход Java 7 и многочисленные призывы к миграции на новую ветку, большое число предприятий по прежнему продолжают использовать Java 6. Для удовлетворения потребности в продолжении жизненного цикла Java 6, компания Red Hat намерена обеспечить формирование обновлений и исправлений уязвимостей для OpenJDK 6.
