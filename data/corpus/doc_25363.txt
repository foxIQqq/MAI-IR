Вышел openSUSE Build Service 1.7. Соглашение с LPI о сертификации специалистов

Анонсирован выход инструментария openSUSE Build Service 1.7, позволяющего разработчикам кросс-компилировать пакеты для большинства основных дистрибутивов Linux или собирать собственные дистрибутивы на основе заданной пакетной базы. На сегодняшний день поддерживаются дистрибутивы CentOS, Debian, Fedora, Mandriva, openSUSE, SUSE Linux Enterprise (SLE), Red Hat Enterprise Linux (RHEL) и Ubuntu. Для коммерческих производителей программного обеспечения, openSUSE Build Service предоставляет один из самых простых путей поддержания пакетов программ для всех популярных дистрибутивов, использующих пакеты в формате RPM или DEB, и поддерживающих архитектуры i386, x86_64 и ARM. Например, можно создать spec файл или подключить репозиторий пакетов представленный на сайте software.opensuse.org и одной командой собрать последнюю версию заданной программы в виде бинарного пакета под нужную систему или сформировать готовое минималистичное окружение для выполнения в системах виртуализации, cloud-окружениях или для загрузки в виде Live-дистрибутива. Из добавленных в openSUSE Build Service 1.7 улучшений можно отметить: Новая система для организации хранения информации, связанной с проектами или пакетами. Позволяет использовать openSUSE Build Service в качестве базы данных для хранения разработчиками различных атрибутов; Увеличение скорости сборки и анализа зависимостей за счет интеграции SAT Solver. Утверждается, что в некоторых ситуациях скорость возрастает в 1000 раз. Зависимости теперь разрешаются за секунды, в то время как раньше выполнение этого действия занимало минуты; Возможность использования в локально установленной версии openSUSE Build Service собственных логотипов, названий и прочих элементов брендинга. Также поддерживается смена визуальных тем в web-интерфейсе; Добавлена возможность генерации образов для быстрого запуска в виртуальных окружениях или в виде live-систем с поддержкой выпуска обновленных версий окружений, которые можно установить без потери накопленных в прошлой версии окружения пользовательских данных; В интерфейс обработки запросов добавлен механизм рецензирования кода; В системе автоматического разрешения конфликтов улучшен процесс выполнения операции слияния исходных текстов. Кроме того, можно отметить публикацию компанией Novell пресс-релиза в котором объявлено о заключении соглашения с международной организацией Linux Professional Institute (LPI) в плане стандартизации подготовки и сертификации специалистов по SUSE Linux. Специалисты имеющие статус LPIC-1 отныне приравнены к лицам получившим сертификат Novell Certified Linux Administrator (CLA), подтверждающий умение устанавливать SUSE Linux Enterprise Server и выполнять типовые ежедневные задачи в области администрирования.
