Выпуск дистрибутива Linux Mint 17.1

Представлен дистрибутив Linux Mint 17.1, в рамках которого сформирован второй выпуск, основанный на пакетной базе Ubuntu 14.04 LTS. Начиная с ветки Linux Mint 17 дистрибутив перешёл к новой схемe формирования выпусков, подразумевающей использование только релизов Ubuntu с длительным сроком поддержки (LTS). При этом промежуточные выпуски Linux Mint отличаются поставкой новых версий рабочих столов Cinnamon и MATE, а также бэкпортированием некоторых новых версий программ, сохраняя при этом совместимость с репозиториями базового LTS-выпуска Ubuntu. Ветка Linux Mint 17 будет поддерживаться до 2019 года. От Ubuntu дистрибутив Linux Mint отличается существенно иным подходом к организации интерфейса пользователя и другим составом используемых по умолчанию приложений. Разработчики Linux Mint предоставляют десктоп-окружение, соответствующее классическим канонам организации рабочего стола, которое является более привычным для пользователей не принимающих новые методы построения интерфейса Unity и GNOME 3. Для загрузки доступны DVD-сборки на базе оболочек MATE 1.8 (1.4 Гб) и Cinnamon 2.4 (1.4 Гб). Кроме отличий в организации рабочего стола, в дистрибутиве задействован ряд оригинальных приложений, упрощающих настройку и работу в системе начинающих пользователей. Например, в Linux Mint используется переработанная система меню, собственный менеджер установки и обновления приложений, оригинальный интерфейс для настройки системы mintConfig, "мастер" для настройки различных параметров системы для начинающих пользователей в стиле вопрос-ответ, интерфейс для настройки параметров десктопа, средство для выполнения резервного копирования mintBackup и т.п. Основные новшества Linux Mint 17.1: В состав включены новые версии десктоп-окружений MATE 1.8 и Cinnamon 2.4, оформление и организация работы в которых продолжает развитие идей GNOME 2 - пользователю предлагается рабочий стол и панель с меню, областью быстрого запуска, перечнем открытых окон и системным лотком с запущенными апплетами. Cinnamon основан на GTK3+ и технологиях GNOME 3. Проект развивает оболочку GNOME Shell и оконный менеджер Mutter в направлении предоставления окружения в стиле GNOME 2 с более современным оформлением и задействованием элементов из GNOME Shell, дополняющих средства классического десктопа. В новой версии Cinnamon полностью переработаны интерфейсы выбора тем оформления и настройки обоев рабочего стола, для настройки сетевых соединений задействован новый конфигурационный модуль GNOME, добавлен конфигуратор для системы вывода уведомлений (Notification Settings) и экран управления приватностью (Privacy Settings). Проведена работа по повышению отзывчивости интерфейса и снижению потребления памяти. Расширены возможности файлового менеджера Nemo (форк Nautilus 3.4). MATE продолжает развитие кодовой базы GNOME 2.32, основан на GTK2+ и полностью избавлен от пересечений с GNOME 3, что позволяет использовать традиционный рабочий стол GNOME 2 параллельно с рабочим столом GNOME 3. Новая сборка примечательная обеспечением возможности использования композитного менеджера Compiz, помимо предлагаемого ранее оконного менеджера Marco (форк Metacity). Compiz готов для включения из коробки и позволяет более полно задействовать возможности современных графических систем, в том числе применять интересные визуальные эффекты на рабочем столе, например, переход между виртуальными рабочими столами в форме трёхмерного куба. Включение Compiz производится через меню "Menu->Desktop Settings->Windows". Для более тонкой настройки в состав включён конфигуратор CompizConfig. В менеджере установки обновлений обеспечена группировка пакетов по их принадлежности к исходному пакету. Каждая строка в списке теперь соответствует одному или нескольким пакетам, связанным с обновляемым приложением. Подобный подход позволяет защитить пользователя от неполной выборочной установки обновлений, так как связь предложенных для обновления пакетов теперь явно выделена. Например, обновление Mesa содержит 18 пакетов, если установить лишь часть из них, то можно нарушить нормальное функционирование графической подсистемы. Кроме того, в новом выпуске менеджера обновлений добавлены сокращённые описания, прекращено скрытие основного окна после установки обновлений и добавлена возможность использования прокси для загрузки сведений об изменениях; Переработан интерфейс выбора ядра Linux, в котором теперь более наглядно представлены сведения о том, какое ядро установлено, какие новые ядра доступны для установки и какие в них присутствуют исправления уязвимостей и регрессивные изменения. Переработано оформление диалога настройки языка и параметров локали. Добавлена возможность выбора метода ввода. Разделены настройки языка и локали, определяющие язык, на котором говорит пользователь и нормы отображения данных, принятые в стране, где живёт пользователь; Модернизировано оформление диалога настройки параметров экрана входа. Добавлена кнопка для предварительного просмотра изменений. Добавлена панель с пиктограммами для быстрой ориентации по категориям настроек. Термин "заставка", вводящий некоторых пользователей в заблуждение, заменён на "тему оформления"; Добавлена новая команда pastebin, позволяющая быстро сохранить в одноимённом сервисе команды или содержимое текстовых файлов (например, 'echo "Hello World!" | pastebin' или 'pastebin myfile.txt'). Переданные в pastebin данные сохраняются в online в течение двух дней; В bash обеспечено автодополнение ввода команды apt; По умолчанию задействованы шрифты Noto; Добавлена возможность изменения цвета любой директории через клик правой кнопкой мыши, что позволяет быстро визуально выделить важные позиции в больших списках; В менеджере установки программ организован вывод более явного предупреждения в случаях, когда операция может привести к удалению других пакетов; Существенно ускорена проверка скорости зеркал репозиториев, обеспечено удаление проблемных зеркал из списка.
