Релиз десктоп-окружения Trinity R14.0, продолжающего развитие KDE 3.5

После двух лет разработки увидел свет релиз десктоп-окружения Trinity R14, продолжающего развитие кодовой базы KDE 3.5.x и Qt 3. Параллельно ведётся портирование кодовой базы на Qt 4, но данный процесс пока далёк до завершения (степень готовности 60%) и относится к категории второстепенных приоритетов, в основном из-за более высокого потребления процессорных ресурсов и памяти в Qt 4. До завершения миграции на Qt4 силами проекта обеспечено поддержание кодовой базы Qt3, в которую продолжают вноситься исправления ошибок и добавляться улучшения, несмотря на официальное прекращение поддержки Qt3. Из особенностей, присущих окружению Trinity можно отметить новые средства для управления параметрами экрана, улучшенный конфигуратор сети и механизмы аутентификации пользователей. Окружение Trinity может быть установлено и использовано одновременно с KDE4, в том числе предоставлена возможность использования в Trinity уже установленных в системе KDE4-приложений. Также присутствуют средства для корректного отображения интерфейса GTK-программ без нарушения единого стиля оформления. Готовые бинарные пакеты доступны для Ubuntu (Lucid, Precise, Trusty), Debian (Wheezy), RHEL/CentOS, Fedora, Mageia и openSUSE. Для тестирования без установки сформирован специальный LiveCD. В новом выпуске: Новая прослойка для работы с оборудованием, основанная на udev и не привязанная к HAL. Для BSD-систем, для которых недоступен udev, сохранена возможность использования HAL; Новый интерфейс для настройки работы с оборудованием, основанный на использовании udev; Добавлена поддержка сетевого конфигуратора network-manager 0.9; Новый композитный менеджер Compton-TDE (форк Compton с расширениями TDE), заменивший собой Kompmgr; Существенно улучшена совместимость со спецификациями XDG; Добавлены сборки для архитектур armel/armhf для Debian и armhf для Raspbian Wheezy, а также экспериментальные сборки для PowerPC и MIPS для Debian Wheezy; Система определения mine-типов переведена на использование libmagic; С целью избежания конфликтов с другими десктоп-окружениями и обеспечения возможности параллельной установки с KDE проведена значительная работа по переименованию приложений, библиотек и пакетов; Задействован новый выпуск TQt3, форка Qt 3.3.8. В новой версии TQt3 добавлен новый движок управления стилями, реализована поддержка многопоточности и увеличена производительность; Значительно переработаны элементы оформления; Добавлены дополнительные движки тем оформления; Продолжена миграция с autotools на cmake; Обеспечена интеграция с LibreOffice; Реализован опциональный режим переключения окон в стиле OS X; Добавлен плагин сортировки Kate; Поддержка ручной перегруппировки программ в панели задач; В панель добавлен режим показа только пиктограмм приложений; Добавлен эффект плавного перехода при смене обоев рабочего стола; Улучшена локализация; Заметно повышена стабильность работы, устранено множество ошибок; Изменена схема нумерации версий. Вместо продолжения нумерации 3.5.x, для того чтобы избежать путаницы из-за сравнений с KDE, выпуски Trinity теперь идентифицируются как Rx.y; Добавлены новые приложения: GTK3/TQt Theme Engine Kasablanca Kate sort plugin KBibTex KBiff KControl Autostart KControl LDAP Manager KDbg KFTPGrabber KLCDDimmer KNmap KRecipes KScope KSensors KShowmail KShutdown LDAP/Kerberos Toolkit MPlayerthumbs Qt4/TQt Theme Engine Tdeio-ftps Tdeio-sword TorK В версии R15 планируется перейти с Automake на систему сборки CMake и задействовать браузерный движок WebKit для всех компонентов, выполняющих обработку и отображение web-страниц.
