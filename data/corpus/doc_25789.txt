Релиз свободного web-браузера Rekonq 0.4

Вышел стабильный релиз открытого web-браузера Rekonq 0.4, построенного с использованием QtWebKit и технологий KDE 4.4. Изначально, как и проект Arora, браузер Rekonq был основан на коде приложения QtDemoBrowser, демонстрирующего возможности интегрированного в библиотеку Qt движка WebKit. В дальнейшем вектор развития Rekonq был направлен в сторону задействования технологий KDE и построения полнофункционального web-браузера для данной платформы, который мог бы конкурировать с Konqueror. На текущей стадии развития, по сравнению с другими браузерами, возможности Rekonq выглядят минималистичными, но у автора проекта в запасе имеется множество идей по построению идеального с его точки зрения браузера. Rekonq развивается достаточно интенсивно, радует еще и то, что в последнее время к создателю проекта для работы над браузером стали присоединяться заинтересовавшиеся разработкой энтузиасты. Новшества, добавленные в версии 0.4: Поддержка KWallet, позволяющая использовать сохраненные пароли совместно с Konqueror; Добавлен механизм AdBlock для блокирования назойливой рекламы; Использование ThreadWeaver для фонового резолвинга DNS-имен в отдельном потоке, без кратковременного блокирования пользовательского интерфейса; Режим управляемой загрузки контента, обрабатываемого плагинами (например, для просмотра Flash-ролика появляется кнопка "clickToFlash"); Web Inspector теперь встраивается в док, примерно как это делает дополнение Firebug в Firefox; Встроенная поддержка просмотра содержимого локальных и ftp директорий, без нагромождений подобных файловому менеджеру в Konqueror; Улучшенная поддержка просмотра страниц в полноэкранном режиме; Базовые возможности Rekonq: Основанный на вкладках (табах) интерфейс; Реализация домашней страницы (аналог speeddial из Opera), на которой отображаются кнопки для открытия избранных сайтов, последние закрытые табы, история работы и закладки (см. скриншот); Поддержка пользовательских сессий и восстановления открытых табов в случае аварийного закрытия браузера; Использование для загрузки файлов подсистемы загрузки из KDE; Возможность совместного использование закладок с Konqueror; Полная интеграция с KDE-подсистемой KIO, которая используется для получения настроек работы через прокси, хранения cookie; Режим анонимной навигации по сети; Поддержка инспектирования содержимого web-страниц. Из планов на будущее отмечено: поддержка RSS, переработка организации скроллинга, возможность быстрой отмены случайно закрытой вкладки, улучшение операций по перетаскиванию табов, поддержка просмотра истории работы из Konqueror.
