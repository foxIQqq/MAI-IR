Релиз открытой биллинговой системы ABillS 0.84

Доступен релиз открытой биллинговой системы ABillS 0.84, компоненты которой поставляются под лицензией GPLv2. Новые возможности: Добавлена возможность показывать пользователям баннер об использовании cookies. Клиенты>Компании>Администратор. Добавлено отображение названия компании. Добавлена возможность настраивать права администраторов на отдельные типы оплат. Добавлена возможность искать пользователей, не учитывая удалённых. Кнопка авторизации через Google изменена в соответствии с требованиями Google. Добавлена возможность при удалении абонента автоматически добавлять к его логину заданный суффикс, возможно, с датой удаления. Добавлена возможность активации кредита на все сервисы абонента. Реализована возможность «Одноразовые услуги»: однократная, одновременная оплата и списание денежных средств за услугу. Группы. Добавлена возможность запретить для группы оплату через кассу. Шаблоны системы. Теперь, если используются изменённые шаблоны, при обновлении отслеживается изменение оригиналов шаблонов. Если оригинал изменился — администратору отправляется оповещение. Добавлена возможность задавать внешний обработчик при заведении и приостановке услуг. Новое RESTful JSON API (бета). Кредит. Добавлена ступенчатая форма кредитования. Серверы доступа. Добавлена возможность импорта IP-пулов с файлов в формате CSV или JSON. Добавлена возможность двухфакторной авторизации для абонента. Добавлена возможность строгого подтверджения действия. Добавлена панель быстрых кнопок. Добавлено сохранение настроек сортировки для таблиц. Добавлена ночная тема для интерфейса администратора. Отчёт>Оплаты. Добавлена детализация по администраторам. Добавлена возможность задавать обязательные для заполнения поля при добавлении нового пользователя. Список абонентов. Добавлено дополнительное поле «Последние списания». Добавлена проверка доступа к выбранным ранее резиновым полям. RADIUS-пары. Добавлена возможность отмечать пары восклицательным знаком с помощью checkbox. Модуль Internet+ Добавлен billd-плагин обновления CID из таблицы online. Улучшен модуль синхронизации сессий для IPoE. Тарифные планы>Интервалы — форматирование чисел скорости и трафика изменено на более читаемое. Игнорируемые IP тепрь учитываются при ручном присваивании. Отчёт «Последнее подключение»: добавлены фильтры по типам (AUTH, GUEST_MODE, HANGUP), по тексту в сообщении (IPOE_LOGIN_NOT_EXIST, USER_NOT_EXIST). Добавлен параметр конфигурации $conf{INTERNET_PROHIBIT_DUPLICATE_NAS_PORT}, который запрещает прописать абоненту дублирующийся NAS/Порт. Для тарифных планов добавлено описание, которое видят только администраторы. При смене ТП система теперь производит автоматическую активацию при наличии денежных средств. Модуль Iptv Доработан модуль Iptv Conax — предусмотрен поиск абонентов по дополнительным новым полям — номер карты ЦТВ, серийный номер ресивера ЦТВ. Trinity_tv. Добавлена проверка дубликата MAC при добавлении экрана. При создании подписки в кабинете пользователя теперь всегда указывается дата активации. Исправлена логика чекбоксов тарифных планов. Microimpuls. Добавлена возможность создавать несколько аккаунтов. Добавлены отчёты для отт провайдеров. Stalker: В кабинете пользователя добавлено отображение логина пользователя в Модуль Stalker. Trinity TV: Изменено добавление ТП на Filter ID. Omega TV: Добавлена возможность изменения идентификатора клиента с UID на Логин. Добавлена возможность создавать несколько расписаний смены тарифного плана для одной услуги. Добавлены списания за каналы в отчёт списаний. При просмотре ТП добавлено отображение каналов. Добавлены списания по дате Активации. Модуль Maps2 Сделана проверка на добавление дубликата района. Добавлена возможность загрузки изображения с увеличенной версией видимой карты Модуль Cablecat Добавлена возможность отображения всех муфт (схем коммутации) в одном слое. Добавлена возможность задавать вручную информацию по затуханию. Модуль Storage Добавлено объединение товаров, если при добавлении уже существует такой товар. Добавлены фильтры поиска товара из подотчётности при установке оборудования пользователю. Добавлено отображение оставшихся месяцев предоставленного оборудования в рассрочку. Добавлен фильтр по складу. Добавлена индикация окончания продажи в рассрочку. Если оплата в рассрочку окончена, то она больше не отображается в баннере. Добавлена проверка, заполнены ли обязательные поля при добавлении товара на склад. Добавлено отображение администратора, который выполнил инвентаризацию. Добавлена возможность удалять накладные. Добавлена возможность указывать количество товара, перемещаемого со склада на другой склад. Добавлена проверка, существует ли уже введенный SN, чтобы нельзя было создать два товара с одинаковым SN. Добавлены дополнительные переменные для шаблонов документов (договор/квитанция). В ЛК абонента добавлено полное отображение информации по его расходам в месяц (аренда, рассрочка, периодические платежы и т.д.). Модуль Msgs Добавлена возможность массовой смены статуса сообщений. Добавлен канбан сообщений. Кнопка «Цитата» теперь не перезагружает страницу для того, чтобы вставить цитату. Добавлена возможность назначать ответственного по разделу. Тогда, когда в этот раздел отправляется сообщение, ответственный назначается ему автоматически. Чат: добавлена анимация подпрыгивания кнопки «Чат», когда поступает новое сообщение. Удалена кнопка редактирования сообщения абонентом (абонент не должен иметь такой возможности). Добавлена возможность просмотра вложения-картинки на весь экран прямо на странице сообщения. Модуль Paysys Рефакторинг модуля Privat_terminal Privat terminal: добавлена возможность поиска по ЄДРПОУ для юридических лиц. Добавлена функция поддержка рекурентных (регулярных) платежей и «подписка» для платежной системы Liqpay. Добавлена платёжная система Ощадбанк. Добавлено дублирование приходов средств на внешнюю систему. Добавлена платёжная система Walletone. Добавлен отчёт для платежной системы Ipay. Меню Отчёт>Платежные системы>Внутренние отчеты. Добавлен отчёт для платежной системы Ibox. Меню Отчёт>Платежные системы>Внутренние отчеты. Paymaster: Добавлена поддержка рекуррентных платежей. Добавлена платёжная система CardPay. Платёжный модуль Telcell.pm переведён на новую схему. Добавлена платёжная система Armenian Card. Добавлена платёжная система IDRAM. Paymaster: добавлена возможность при оплате отправлять дополнительные данные для предоставления чека и отправка его на e-mail (правила РФ). Теперь при негативном депозите в портале абонента отображаются кнопки только тех платёжных систем, которые доступные для оплаты. Добавлен скрипт для включения и выключения интернета при негативном депозите на определенное время для осуществления оплаты. Usmp: добавлена возможность оплаты с портала абонента. Добавлен отчёт для платежной системы City24. Меню Отчёт>Платежные системы>Внутренние отчеты. Улучшено введение настроек. Платежный модуль Global_Money: добавлена возможность функции Balance (проверка задолженности Global_Money перед провайдером). Сбербанк Онлайн: теперь при оплате с терминала передаётся рекомендованная сумма оплаты. Добавлена платёжная система Sberbank online. Добавлена платёжная система Internet Plategka. Добавлена платёжная система Octo (Узбекистан). В настройках контрагентов checkbox заменены на элементы toggle. Теперь в отчёт>Настройка заносятся действия связанные с Paysys: добавление, изменение и удаление платёжных систем и контрагентов, удаление настроек контрагента для групп и платежей. Модуль Equipment Добавлен billd-плагин для автоматической регистрации ONU на Huawei. Bdcom: добавлена возможность удалять ONU с OLT. На странице ONU выпадающий список периодов для графиков заменён на компонент выбора даты, теперь можно смотреть графики за любой выбранный период в прошлом. Модуль Sms Добавлена возможность отправки абоненту SMS после регистрации. Добавлен СМС-шлюз MsgAm. Добавлен СМС-шлюз Cablenet. Turbosms: реализована отправка на Viber. Отчёт>Sms>Личные: добавлен фильтр по номеру телефона. Turbosms: добавлена возможность синхронизировать статусы сообщений с базой данных Turbosms. Добавлен СМС-шлюз Likon Sms. Telegram Bot Telegram Bot. Добавлена локализация. Теперь бот поддерживает три языка: английский, русский, украинский. Добавлен интерфейс администратора. Теперь администратор может открыть сообщение в браузере (кнопка «Открыть сообщение»), или ответить прямо с бота (кнопка «Ответить»). Переработан Telegram bot для абонента. Добавлена отправка сообщений с стороны клиента. Модуль Employees Добавлена возможность перемещения средств между кассами. Табель работ дополнен кнопкой выбора года. Добавлена возможность выбора типа оплат прихода кассы по умолчанию. Данный тип будет отображаться при оплате абонента в кассе. Модуль Sqlcmd Добавлена возможность мониторить использование памяти mysqld. Модуль Accident Добавлена возможность отображения аварий по адресу абонента с карточки абонента. Модуль Abon Добавлен многоуровневый доступ управления услугой в кабинете абонента (только чтение, управление услугой, не отображать). В поиске поле ввода тарифного плана заменено на выпадающий список. Модуль Rwizard Добавлен фильтр по домам. Модуль Notepad Для заметки добавлено поле «Время». ABills мобильные клиенты Ajuster (Beta). Запущенно бета версию мобильного приложения под ОС Android для монтажников Разработано приложение для iOS. Модуль Cams Добавлено отображение количества подписок. Модуль Sharing Добавлена возможность отображать файлы в клиентском кабинете в виде списка. Модуль Callcenter Добавлен плагин Binotel. Модуль Userportal В разделе «Телевидение» добавлена возможность отображения дополнительных полей «Абон.плата за месяц» и «Дневная а/п».
