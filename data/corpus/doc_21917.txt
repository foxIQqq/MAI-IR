Представлен первый релиз сервера службы каталогов CentOS Directory Server

Разработчики проекта CentOS объявили о выпуске первой публичной версии LDAP-совместимого сервера каталогов CentOS Directory Server (CDS), основанного на кодовой базе Red Hat Directory Server 8.1 и, в отличие от другого свободного ответвления - 389 Directory Server (бывший Fedora Directory Server), нацеленного на максимальную совместимость с базовым продуктом, без внесения улучшений и изменений. CentOS Directory Server в сочетании с CentOS по задумке разработчиков должен обеспечить решение, идентичное связке Red Hat Directory Server и Red Hat Enterprise Linux. Загрузить CentOS Directory Server можно из репозитория CentOS Extras в сборке для CentOS 5.3 (более ранние версии не поддерживаются из-за привязки пакета к OpenJDK). В планы входит портирование пакета для ветки CentOS 4. Напомню, что в 2004 году было произведено частичное открытие исходных текстов Netscape Directory Server, после покупки прав на данный продукт компанией Red Hat. На базе открытых исходных текстов был основан полностью свободный сервер каталогов Fedora Directory Server, в рамках которого постепенно были переписаны все части кода, которые было невозможно открыть. Начиная с релиза Red Hat Directory Server 8.0, вышедшего в январе 2008 года, все проприетарные участки кода были переписаны и пакет начал распространяться целиком в исходных текстах.
