Релиз LTSP 5.2, пакета для организации работы тонких клиентов

После двух лет разработки представлен релиз проекта LTSP 5.2 (Linux Terminal Server Project), нацеленного на упрощение организации работы тонких клиентов на базе устаревших ПК. С момента прошлого выпуска в репозитории исходных текстов отмечено 994 коммита от 14 разработчиков. Одновременно с новой версией LTSP выпущены развивающийся параллельно экранный менеджер LDM (LTSP Display Manager) и модуль LTSPfs 0.6 для монтирования удаленных файловых систем поверх локальных. Готовые установочные пакеты подготовлены для Ubuntu Linux и скоро будут доступны для других Linux дистрибутивов. В анонсе отмечается, что релиз LTSP 5.2 является знаковым событием в развитии проекта. Из основных возможностей LTSP 5.2 отмечено: Возможность организации бездисковой загрузки Debian/Fedora/Gentoo/Ubuntu с использованием PXE (dhcp + tftp) с последующей организацией соединения с сервером приложений при помощи SSH и X11; Поддерживается возможность запуска всего сеанса на удаленном сервере или выполнения отдельных приложений локально, для задействования всех возможностей аппаратного обеспечения, таких как использование 3D-акселерации; При запуске тонкого клиента на базе Ubuntu, поддерживается функция запуска всех приложений локально, за исключением ряда выборочных программ, которые будут запущены с терминального сервера (режим Fat Client); Поддержка инициирования RDP-сессий при помощи rdesktop; Возможность использования локальных блочных устройств, таких как жесткие диски и CD-ROM, при помощи ltspfs; Легко расширяемая архитектура на основе плагинов; Поддержка интернационализации, возможность использования многоязычного интерфейса в LDM и большинстве скриптов; Поддержка масштабирования при помощи LTSP-Cluster, позволяющего создать кластер из терминальных серверов, которые могут обслуживать единую сеть из тысяч тонких клиентов.
