Доступна система управления контейнерной виртуализацией Docker 0.11

Представлен релиз инструментария для управления изолированными Linux-контейнерами Docker 0.11, который позволяет автоматизировать выполнение задач формирования, обслуживания и сопровождения начинки контейнеров для выполнения отдельных приложений или программных стеков в изолированном окружении. Для создания контейнеров могут использоваться libcontainer (обёртка над namespaces и cgroups), lxc, libvirt, systemd-nspawn и т.п. Код Docker написан на языке Go и распространяется под лицензией Apache 2.0. Новая версия позиционируется в качестве первого кандидата в релизы, перед стабильным выпуском 1.0. Из улучшений можно отметить: Возможность запуска демона docker с поддержкой SELinux (опция "--selinux-enabled"); Режим "host networking" при котором в контейнере не изолируются сетевые компоненты (используется общий с хост-системой сетевой стек); Поддержка обнаружения связанных контейнеров по именам хостов; Новый сервис для периодической проверки работоспособности основного демона Docker; Возможность зеркалирования реестра доступа между несколькими контейнерами; Поддержка SHA-512 для некоторых SSL-сертификатов.
