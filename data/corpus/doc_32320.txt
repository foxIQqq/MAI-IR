Релиз дистрибутива openSUSE 12.1

Официально анонсирован релиз Linux-дистрибутива openSUSE 12.1. Дистрибутив доступен для 32- и 64-разрядных x86-систем в четырёх сборках: универсальный установочный DVD (4.7 Гб), LiveCD с GNOME 3.2, LiveCD с KDE 4.7 и сокращённая версия для установки по сети. openSUSE 12.1 является первым выпуском после перехода на новую нумерацию релизов. Следующая версия openSUSE 12.2 будет выпущена в июле 2012 года. Ключевые улучшения, добавленные в openSUSE 12.1: Пользовательское окружение GNOME обновлено до версии GNOME 3.2 (в прошлом выпуске поставлялся GNOME 2.32), в качестве интерфейса по умолчанию при выборе GNOME используется GNOME Shell. Кроме GNOME дистрибутив в равной мере поддерживает окружение на основе KDE 4.7.2. В качестве легковесного десктоп-окружения поддерживаются Xfce 4.8 и LXDE 0.5.5; Любители KDE 3 отныне могут установить данное окружение из официального репозитория, дополнительные программы для KDE 3 могут быть установлены из поддерживаемого сообществом репозитория KDE:KDE3. Пакеты с KDE 3 включают дополнительные исправления и устранения проблем безопасности, подготовленные энтузиастами из различных проектов, включая openSUSE, Chakra и ALT Linux. Среди наиболее существенных изменений - переписанный mediamanager, который теперь не привязан к подсистеме HAL; Воспользовавшись репозиторием KDE:Active, отдельно можно установить оболочку Plasma Active, подготовленную для устройств с сенсорными экранами; В дистрибутив интегрирована система управления цветом Oyranos, которая позволяет централизованно управлять цветовыми профилями для разных устройств. Oyranos задействован как при использовании KDE, так и в GNOME; Для пользователей KDE представлен новый графический интерфейс управления пакетами - Apper, представляющий собой простой в использовании фронтэнд к PackageKit; В состав включено приложение Snapper, позволяющее при использовании файловой системы Btrfs в наглядном режиме просматривать различия между текущим и прошлыми состоянием файлов в системе, при необходимости изменения можно откатить, например, вернуть случайно удалённый или перезаписанный файл. Интеграция с пакетным менеджером zypper позволяет использовать Snapper для отката отдельных системных обновлений или изменений конфигурации; Интегрирована поддержка развиваемого в рамках проекта KDE сервиса ownCloud 2 для организации хранения данных на внешних серверах. От таких сервисов, как Google Docs, Dropbox и Ubuntu One система ownCloud отличается предоставлением полного контроля над своими данными - информация не привязывается ко внешним закрытым облачным системам хранения, а размещается только на подконтрольных пользователю локальных и внешних системах; Обновление ядра Linux 3.1, набора компиляторов GCC 4.6.2, системной библиотеки glibc 2.14, программы ведения системных журналов syslog-ng 3.3 и пакетного менеджера RPM 4.9; Переход по умолчанию на системый менеджер systemd, который заменил собой классическую систему инициализации. Systemd нацелен на более интенсивную параллелизацию выполнения сервисов на этапе загрузки системы, вобрав в себя лучшие черты таких систем как launchd (Mac OS X), SMF (Solaris) и Upstart (Ubuntu). Кроме запуска сервисов, обеспечиваются функции контроля за выполнением фоновых процессов. Подробнее о systemd можно прочитать в ранее опубликованных обзорах (здесь, здесь, здесь, здесь и здесь). Сравнение систем инициализации systemd, upstart и SysVinit можно посмотреть здесь; Обновлены версии пользовательских программ: браузеры Firefox 7, Thunderbird 7, Chromium 17, Opera 11.52, Rekonq 0.8, Midori 0.4.1; аудиоплееры Amarok 2.4.3, Clementine 0.7.3, Banshee 2.2; менеджеры фотографий Digikam 2.2.0, Shotwell 0.11.5, F-spot 0.8.2; офисные пакеты LibreOffice 3.4.3; KOffice 2.3.1, Calligra 2.4 beta 3; программы для работы с графикой Inkscape 0.48.2, Scribus 1.4rc6, GIMP 2.6.11, Blender 2.59, Kdenlive 0.8.2, OpenShot 1.4, PiTiVi 0.15; Audacity 1.3.13, Gnash 0.8.8, k3b 2.0.2, Pidgin 2.10; Обновлены серверные приложения: Apache 2.2.21, lighttpd 1.4.29, ngninx 1.0.4, Samba 3.6.1, PostgreSQL 9.1.1, MySQL 5.5.16, MariaDB 5.2.9; Обновлены компоненты, связанные с виртуализацией: Eucalyptus, OpenNebula и OpenStack (доступны через отдельный репозиторий). В качестве механизмов виртуализации поддерживаются Xen 4.1, KVM и VirtualBox 4.1.4, для управления которыми могут быть использованы интерфейсы virt-manager и open-vm-tools; В состав дистрибутива включён пакет с реализацией языка программирования Go. В комплект также включена предварительная версия llvm/clang 3.0, которую можно использовать помимо GCC. Обновлены версии Perl 5.14.2, Python 2.7.2 и PHP 5.3.8; Программа для настройки параметров сети NetworkManager обновлена до версии 0.9, в которой реализована поддержка GNOME Shell, представлен более удобный интерфейс управления сетевыми параметрами, реализована поддержка быстрого переключения между пользовательскими сеансами, добавлена система разграничения доступа, поддержка настройки соединений WiMAX, улучшенная поддержка WiFi-роуминга (возможность использования нескольких различных точек доступа), новый внутренний API на базе D-Bus и единое хранилище настроек; Возобновлена поддержка конфигуратора параметров X-сервера - SAX3, базирующегося на новой структуре файлов конфигурации X.Org (xorg.conf.d). SAX3 позволяет осуществить тонкую настройку монитора, мыши, клавиатуры и тачпада, используя как графические интерфейсы на базе Gtk+ и Qt, так и консольный конфигуратор на базе ncurses; Улучшена работа конфигуратора YaST и добавлен обновлённый административный web-интерфейс webYaST. В YaST переработан стиль интерфейса и решена такая важная проблема, как перезапись вручную изменённых файлов. В пакетном менеджере Zypper увеличена гибкость при обновлении ядра Linux, например, удаление прошлого ядра может выполняться только после первой успешной перезагрузки с использованием нового ядра; Вместо Oracle JDK в дистрибутиве отныне используется пакет с OpenJDK; В экспериментальном режиме в качестве загрузчика можно использовать Grub2. Дополнительно отмечается, что пользователи openSUSE 12.1 могут воспользоваться репозиторием Tumbleweed, в рамках которого реализована идея непрерывного цикла выпуска обновлений (rolling-обновления). В отличие от репозитория Factory в рамках Tumbleweed поддерживаются только стабильные версии пакетов. Используя Tumbleweed, пользователи могут на свое усмотрение воспользоваться всегда свежими версиями программ, не опасаясь возможных проблем со стабильностью, встречающихся при использовании Factory.
