Релиз Linux-дистрибутива openSUSE 12.3

После шести месяцев разработки представлен релиз Linux-дистрибутива openSUSE 12.3. Дистрибутив доступен для 32- и 64-разрядных x86-систем в четырёх сборках: универсальный установочный DVD (4.7 Гб), Live с GNOME 3.6, Live с KDE 4.10 и сокращённая версия для установки по сети. Кроме того, впервые в рамках штатного выпуска поставляется стабильная сборка openSUSE для ARMv7, экспериментальная сборка для ARM64 (AArch64) и специализированная Live-сборка для восстановления после сбоев. Особенности openSUSE 12.3: Поставка новых версий пользовательских окружений KDE 4.10, GNOME 3.6, Xfce 4.10, LXDE, KDE 3 и Razor-Qt 0.5. В состав дистрибутива также включено десктоп-окружение Enlightenment E17 и добавлены оконные менеджеры Sawfish и Awesome. Переработана инфраструктура дистрибутива: Удалена поддержка классической системы инициализации Sysvinit. Завершён перевод дистрибутива на системный менеджер systemd, который обновлён в openSUSE 12.3 до версии 195. Для ведения системных логов задействована подсистема journald, но по умолчанию вывод дублируется в обычную реализацию syslog (rsyslog, syslog-ng). Библиотеки и исполняемые файлы перенесены из корня в иерархию директории /usr. Точки монтирования перенесены в каталог /run/user/. Для обеспечения перехода в спящий и ждущий режимы, а также для управления индикаторами на ноутбуках, задействованы механизмы systemd; По умолчанию вместо MySQL теперь предлагается СУБД MariaDB. Поддержка MySQL сохранена и пользователи могут установить данную СУБД из штатных репозиториев, но все зависимости для требующих MySQL сторонних пакетов теперь связаны с MariaDB; Реализована полноценная поддержка загрузки в режиме UEFI и экспериментальная поддержка загрузки в режиме UEFI Secure Boot, основанная на использовании загрузчика Shim, подписанного ключом Microsoft. В режиме UEFI Secure Boot возможна не только верификация начального загрузчика, но и проверка загружаемых в дальнейшем GRUB2 и ядра Linux, при этом для их проверки используются цифровые подписи на основе ключей openSUSE (также предусмотрен вариант использования собственных ключей пользователя); Обновлены версии системных компонентов: ядро Linux 3.7, Glibc 2.16, zsh 5.0, libpng 1.5, binutils 2.23; Новые версии пользовательских приложений и графического стека: X-сервер 1.13, Mesa 9, свежие видеодрайверы, LibreOffice 3.6 (из дополнительных репозиториев доступен LibreOffice 4.0), DigiKam 3.0, Amarok 2.7, rhythmbox 2.98, Banshee 2.6, Blender 2.64, Ekiga 4.0, Scribus 1.4.2, Rekonq 2.0, KTorrent 4.3.1; Новые версии серверных пакетов и средств виртуализации: PostgreSQL 9.2, OpenStack "Folsom", Virtualbox 4.2, QEMU 1.3, Squid 3.2.6, Nagios 3.4.4; Новые версии средств для разработчиков: LLVM 3.2, gdb 7.5, GCC 4.7.2, Python 3.3 (по умолчанию по прежнему Python 2.7.3), Mono 3, Qt Creator 2.6, Glade 3.14, MonoDevelop 3, Valgrind 3.8, KDevelop 4.4; Осуществлён переход на GStreamer 1.0 и звуковой сервер PulseAudio 3. В состав включена библиотека libopus с поддержкой звукового кодека OPUS; Для работы с JPEG-изображениями по умолчанию задействован пакет jpeg-turbo; Из состава дистрибутива удалён давно устаревший конфигуратор SuSEconfig, созданный много лет назад для замены /etc/rc.config и давно критикуемый за свои прямые модификации файлов конфигурации в процессе установки групп пакетов; Используемая пакетным менеджером библиотека libzypp обновлена до версии 12.6. Проведена работа по увеличению надёжности средств для работы с пакетами. Полностью переписан бэкенд PackageKit для zypper; Размер образов LiveCD расширен до 1 Гб, что позволило вместить больше программ, но исключило возможность записи на CD. В частности, в Live образ были добавлены полный LibreOffice, GIMP, OpenJDK, набор полезных инструментов для восстановления после сбоя и резервного копирования, таких как gparted, grsync, dd_rescue и photorec; Подготовлена официальная версия openSUSE 12.3 для архитектуры ARMv7. Дистрибутив доступен в сборках для платформ на базе процессоров Cortex-A (ARMv7) - Texas Instruments OMAP3/OMAP4 и Freescale I.MX51, а также устройств и плат на их базе, таких как BeagleBoard (xM), Panda Board, Calxeda Highbank, i.MX53 Loco, CuBox, Origen Board и Efika MX. Неофициальная поддержка обеспечена для нетбука Toshiba AC100. Базовое десктоп-окружение ARM-редакции openSUSE построено на основе Xfce 4.10, но в качестве опции можно использовать GNOME , KDE, IceWM, Enlightenment или WindowMaker; Обновление утилиты snapper до версии 0.1.0. Утилита предназначена для управления созданием снапшотов со срезами состояния файловой системы и позволяет в наглядном режиме просматривать различия между текущим и прошлыми состояниями файлов в системе, при необходимости изменения можно откатить, например, вернуть случайно перезаписанный файл или восстановить состояние системы после установки пакетов. Новая версия примечательна обеспечением возможности работы Snapper с правами обычного пользователя, при этом отдельным пользователям необходимо назначить область их полномочий. Например, пользователю можно разрешить создание снапшотов только для его домашней директории. Добавлена экспериментальная поддержка работы со снапшотами LVM; Внесена порция изменений, связанных с безопасностью: в sshd активирован более жесткий режим изоляции. Вместо пакета pwdutils, который является форком shadow, задействован основной пакет shadow. По умолчанию для новых установок теперь включаются PAM-модули pam_unix.so и pam_cracklib.so;
