План развития почтового клиента Evolution

Разработчики входящего в состав GNOME почтового клиента Evolution опубликовали план разработки следующей версии - Evolution 3.4, релиз который выйдет одновременно с GNOME 3.4 в марте следующего года. Из грядущих новшеств отмечены: Перенос некоторых функций обработки почты в новый сервис D-Bus, созданный по аналогии с ранее разработанными сервисами для адресной книги и календаря. Сервис позволит реализовать такие вещи как вывод уведомления о получении новых писем или доступ других приложений к хранилищу почты без необходимости запуска Evolution; Переход на использование WebKit/GTK+ для отображения содержимого писем с разметкой HTML, вместо ранее используемого устаревшего движка GtkHtml. Использование WebKit позволит улучшить качество отображения, привести к единому стилю вывод текста сообщений, контактов, задач и заметок, реализовать поддержку JavaScript и CSS. Примечательно, что развиваемый отдельно редактор писем пока остаётся на базе GtkHtml и скорее всего будет заменен только в более отдаленных выпусках; Режим автоматической настройки параметров почтового аккаунта для всех популярных почтовых сервисов. Например, достаточно будет ввести email и пароль, все остальные настройки будут заданы автоматически; Переход от GConf к dconf с доступом через GSettings API для хранения настроек; Новый бэкенд для взаимодействия с Microsoft Exchange - evolution-ews, подготовленный при участии работников компаний Intel и Novell. Бэкенд поддерживает работу с Microsoft Exchange 2007 и 2010, и реализован по аналогии с Exchange Web Services, используя полностью документированный SOAP API; Новый бэкенд для Kolab Groupware, платформы для организации совместной коллективной работы. Изменение формата хранения привязанных к аккаунту данных, вместо XML будут использованы обычные текстовые файлы с простым ini-подобным синтаксисом, который легко прочитать и отредактировать вручную. Для управления этими файлами будет подготовлен специальный сервис D-Bus.
