Выпуск VirtualBox 6.1.18

Компания Oracle опубликовала корректирующий релиз системы виртуализации VirtualBox 6.1.18, в котором отмечено 14 исправлений. В списке изменений явно не обозначено (Oracle не отражает исправления уязвимостей в общем списке изменений), но можно предположить об исправлении раскрытой на днях уязвимости, которой были подвержены выпуски вплоть до VirtualBox 6.1.16. Проблема позволяла получить доступ к хост-системе из виртуальной машины при наличии у атакующего в гостевой системе доступа к загрузке модулей ядра, при включении в VirtualBox драйвера для SCSI-контроллера, помеченного в качестве загрузочного. Основные изменения: Устранено зависание при запуске вложенных гостевых систем в режиме SMP на хостах с некоторыми процессорами Intel. В компонентах для интеграции с OCI (Oracle Cloud Infrastructure) улучшен разбор параметров облачных окружений при импорте. Решена проблема с выдачей IP-адресов гостевой системе через DHCP после возвращения хоста из спящего режима. Добавлены оптимизации для macOS BigSur. Решена проблема с воспроизведением звука после ухода хоста в спящий режим. В дополнениях для хостов и гостевых систем с Linux реализована поддержка ядра Linux 5.10. В компонентах для хостов с Solaris устранена регрессия, нарушающая работу текстового режима VGA. Решены проблемы со сборкой дополнений для гостевых систем в CentOS 8.2-2004 и более новых выпусках, а также в системах с ядрами Linux с 3.2.0 по 3.2.50. В дополнениях для гостевых систем устранён крах виртуальной машины при копировании из совместного буфера обмена на системах с X11. В реализации совместных каталогов (Shared Folder) устранена ошибка с повторным монтированием в гостевых системах с Linux.
