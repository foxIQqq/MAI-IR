Выпуск VirtualBox 5.1.12

Компания Oracle выпустила корректирующий релиз системы виртуализации VirtualBox 5.1.12, в котором отмечено 19 исправлений. Среди изменений: В хост-окружениях с ядром Linux 2.6.18 реализовано автоматическое отключение асинхронного ввода/вывода, так как при высокой интенсивности ввода/вывода могли возникать сбои; Устранены проблемы с компиляцией компонентов для хостов при использовании ядра Linux 4.9; Решены проблемы со сборкой компонентов для хостов и гостевых систем в окружениях с ядром Linux 2.6.28; В дополнения для Linux-систем добавлено предупреждение о проблемах с работой 3D при использовании старых ОС в гостевых системах (Debian 7), для решения которых требуется внесение изменений вручную; В дополнениях для Linux-систем устранены проблемы при сборке графических драйверов при использовании ядра Linux 4.10 и новее; Решены проблемы с выводом ошибки VERR_IEM_ASPECT_NOT_IMPLEMENTED в некоторых гостевых окружениях, если включены паравиртуальные драйверы для KVM; Устранена проблема с выводом ошибки VERR_VMX_UNABLE_TO_START_VM при некотором стечении обстоятельств; Устранён крах GUI при завершении работы VM на ранней стадии запуска; В GUI решены проблемы с перемещением окна гостевой системы в бесшовном режиме при включении захвата клавиатуры; Обеспечена корректная обработка сброса накопителей NVMe при выполнении выгрузки и загрузки модуля ядра в гостевой системе (rmmod nvme; modprobe nvme); Решены проблемы с созданием снапшотов работающей виртуальной машины при наличии контроллера NVMe; Устранена ошибка, приводившая к потере запросов в гостевых окружениях, работающих в режиме SMP, при использовании SCSI-контроллера LsiLogic; В эмуляторе сетевого адаптера E1000 устранена проблема, приводившая к выводу ошибки "TX unit hang" в гостевых системах с Linux; В API добавлены дополнительные проверки, позволяющие избежать краха при передаче специально оформленных имён VM.
