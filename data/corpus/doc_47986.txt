Выпуск проприетарного драйвера NVIDIA 390.25

Компания NVIDIA представила первый выпуск новой стабильной ветки проприетарного драйвера NVIDIA 390.25, которая будет развиваться в рамках длительного цикла поддержки (LTS) до января 2019 года. Драйвер доступен для Linux (ARM, x86, x86_64), FreeBSD (x86, x86_64) и Solaris (x86_64). Ветка 390.x станет последней, доступной для 32-разрядных операционных систем. В дальнейшем поддержка новых GPU будет осуществляться только в 64-разрядных драйверах. Основные новшества: Добавлена поддержка GPU GeForce GTX 1060 5GB, Quadro P620, GeForce GTX 1050 Ti с Max-Q Design, GeForce GTX 1070 Ti, GeForce MX110, GeForce MX130, Quadro P500, TITAN V, TITAN Xp COLLECTORS EDITION; Добавлены настройки "EGLVisibleDGPUDevices" и "EGLVisibleTegraDevices", определяющие дискретное или Tegra GPU будут доступны через EGL; В модуле nvidia.ko запрещены события ACPI_VIDEO_NOTIFY_PROBE для ядер, допускающих переопределение обработчика данного события, что позволяет улучшить взаимодействие между драйвером NVIDIA и acpi_video при обработке событий горячего подключения экрана; В nvidia-settings страница настройки мозаичной раскладки SLI обновлена для поддержки топологий, включающих до 32 экранов; Код композитинга в драйвере для X11 расширен поддержкой стереовывода; В nvidia-settings добавлен режим предпросмотра стереовывода через OpenGL на обычном экране; Драйвер изменён для предотвращения восстановления консольных режимов фреймбуфера на шлемах виртуальной реальности; В nvidia.ko добавлен код для самостоятельной установки регистров PAT (Page Attribute Table) для оптимизации хранения в памяти, если ядро не предоставляет встроенную поддержку PAT; Исправлена проблема с периодической блокировкой ядра при использовании трансформации изображения при выводе на экран. В частности, проблема могла проявляться при использовании опции ForceCompositionPipeline в xorg.conf.
