Релиз платформы для организации групповой работы Kolab 3.0

Компания Kolab Systems представила релиз Kolab 3.0 - платформы для организации совместной коллективной работы, базирующейся на технологиях KDE. Основу платформы составляет серверное ПО Kolab Server, позволяющее организовать работу серверной инфраструктуры, реализующей средства групповой работы, единой адресной книги, календаря-планировщика и почтового сервиса. В качестве клиентских приложений могут выступать как стандартные программы из состава KDE PIM (Kontact, KMail, KOrganizer и KAddressbook), так и сторонние системы, в том числе доступен набор коннекторов для Microsoft Outlook (Bynari Connector, плагин Toltec Connector 2 и MAPI-хранилище KONSEC Konnektor). В качестве базового web-клиента отныне в составе платформы поставляется Roundcube Webmail, в полной мере поддерживающий вторую и третью версии формата хранилища Kolab. Для интеграции с Thunderbird и SeaMonkey проектом развивается специальный плагин SyncKolab. Модуль интеграции также доступен для GNOME Evolution. Для организации работы с мобильными системами предоставляется ряд решений по синхронизации данных, таких как ActiveSync-совместимые решения Kolab ActiveSync Server и Z-Push. Для организации работы в Kolab Server задействованы стандартные открытые компоненты, такие как сервер директорий 389, Apache, Postfix, Cyrus IMAP, SASL, OpenSSL. Для управления системой подготовлен специальный административный web-интерфейс. Поддерживается ведение единой адресной книги, интегрируемой с почтовым сервисом с поддержкой POP3 и IMAP4. Для шифрования почты предлагается S/MIME-шифрование на стороне клиента. Бинарные сборки с Kolab 3 подготовлены в виде пакетов для Red Hat Enterprise Linux, CentOS, Ubuntu, Debian и OpenSUSE . Особенности новой версии: Новый web-клиент на основе системы Roundcube (в прошлых версиях предлагался Horde), разработанной при поддержке компании Kolab Systems. Roundcube отличается современным интерфейсом, расширенными возможностями по интеграции со службами директорий и поддержкой режимов для смартфонов и планшетов; Новый web-интерфейс администратора, предоставляющий расширенные возможности по управлению стационарными и мобильными клиентами, имеющий гибкие средства для управления ресурсами, пользователями, группами, почтовыми доменами, псевдонимами, правами доступа и ролями; Новая унифицированная утилита для управления из командной строки; Расширена масштабируемость решений на базе Kolab, все функциональные компоненты Kolab теперь могут быть запущены обособленно на отдельных серверах; В Kolab daemon добавлена поддержка мониторинга за почтовыми потоками, что позволяет ограничивать получателей и устанавливать квоты; Обеспечена возможность реализации дополнительной функциональности по обработке почтовых сообщений на лету, через подключение модулей-дополнений на языке Python. Например, через модули к сообщениям могут быть автоматически добавлены примечания о политике конфиденциальности корпоративной переписки или организован перевод писем; Вместо OpenLDAP по умолчанию задействован сервер директорий 389. Поддержка OpenLDAP оставлена в качестве опции; Реализована простая система указания статуса пользователя (занят или свободен), работающая без зависимости от календаря; Для подсоединения мобильных клиентов к серверу задействован протокол ActiveSync. Вместо устаревшего стека Z-Push отныне по умолчанию используется Syncroton в качестве реализации протокола ActiveSync, отличающийся улучшенной производительностью и расширенными возможностями интеграции; Новая версия формата хранения данных Kolab, использующая форматы xCal и xCard для сохранения данных календаря и адресной книги.
