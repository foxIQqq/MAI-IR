Релиз Red Hat Enterprise Virtualization 3.2. KVM адаптирован для архитектур Power и MIPS

Компания Red Hat представила новый значительный выпуск промышленной платформы для организации управления инфраструктурой виртуализации - Red Hat Enterprise Virtualization 3.2. Платформа RHEV основана на Linux-дистрибутиве Red Hat Enterprise Linux 6 и использует в работе технологию виртуализации KVM (Kernel Virtual Machine). Для организации работы тонких клиентов используется протокол SPICE. Продукт является коммерческим, готовые сборки доступны для загрузки, но обновления предоставляются бесплатно только в течение 60 дней. Тем не менее разработка ведётся в виде серии открытых проектов, все создаваемые для RHEV наработки публикуются под лицензией GPLv2. Используемые в RHEV технологии управления виртуальными машинами развиваются в рамках открытого проекта oVirt, разработка гипервизора KVM ведётся в рамках основной ветки ядра Linux. В виде открытых проектов распространяются средства для запуска и развёртывания виртуальных машин, управляющий web-интерфейс, CLI-интерфейс, универсальный API, набор агентов для гостевых систем, инсталлятор пакетов, генератор отчётов, SDK для создания дополнений, а также сопутствующие инструменты и библиотеки, такие как libvirt и v2v. Особенности Red Hat Enterprise Virtualization 3.2: Поддержка Live-миграции хранилища, что позволяет переносить диски виртуальных машин между разными устройствами хранения, не требуя остановки виртуальных машин. Поддержка обновления доменов хранения в Live-режиме - данные виртуальных машин могут быть перенесены со старого типа датацентра на новый домен V3 без остановки виртуальных машин; Возможность создания на лету снапшотов виртуальных машин; Поддержка особенностей новых процессоров Intel и AMD, включая серии Intel Haswell и AMD Opteron G5; Многочисленные улучшения интерфейсов управления хранилищами, сетью, разграничением доступа и энергопотреблением. Также расширены средства мониторинга, журналирования и организации работы с удалённым рабочим столом; Реализация фреймворка для разработки и распространения плагинов, через которые можно создавать дополнительные элементы пользовательского интерфейса, интегрируемые в web-портал администратора (WebAdmin). Для разработки плагинов используется язык JavaScript и специальный API для интеграции с функциями web-портала oVirt. Плагины выполняются в контексте клиентского web-браузера и взаимодействуют с серверной частью только через доступный API. Из подготовленных сторонними вендорами плагинов отмечаются: подготовленный компанией NetApp интерфейс для управления виртуальными хранилищами Virtual Storage Console (VSC); система реагирования на угрозы и обеспечения отказоустойчивости Veritas Cluster Serve, созданная компанией Symantec; созданный компанией HP интерфейс Insight Control для управления серверамим HP ProLiant; Режим зеркалирования портов, позволяющий направить копию сетевого трафика в определённую виртуальную машину. Виртуальный сетевой адаптер при этом начинает отслеживать весь трафик выбранной логической сети, который включает в себя трафик других виртуальных машин; Расширение интеграции с распределенной файловой системой GlusterFS: средства для импорта существующих кластеров GlusterFS, поддержка однонаправленной гео-репликации, поддержка выполнения асинхронных работ с разделами Gluster, настройка конфигурации через CLI-интерфейс с автоматической синхронизацией изменений, средства мониторинга работы GlusterFS и просмотра детальной информации через web-интерфейс, инструмент для оптимизации Gluster-разделов для использования в системах виртуализации; Переработана система квот, в том числе логика работы, метод расчёта квот и мониторинг за их соблюдением; Поддержка проброса смарткарт в виртуальные машины пользователей. Проброс может быть организован через административный интерфейс, web-интерфейс пользователя или через REST API; Дополнительно можно отметить несколько событий, также связанных с развиваемыми компанией Red Hat технологиями виртуализации: Анонсирован новый продукт Red Hat Enterprise Linux OpenStack Platform, нацеленный на развёртывание локальных облачных инфраструктур IaaS (Infrastructure-as-a-Service) на базе платформы OpenStack. Помимо Red Hat Enterprise Virtualization и платформы для организации управления системами и приложениями, работающими в облачных окружениях, Red Hat CloudForms, новая платформа Red Hat Enterprise Linux OpenStack Platform входит в серию продуктов Open Hybrid Cloud, направленных на обеспечения работы сервисов предприятия в гибридных облачных окружениях, сочетающих локальные мощности и внешние сервисы; Компания IBM представила новое решение для организации системы виртуализации на серверах линейки Power, поставляемых с Linux. Продукт построен на основе гипервизора KVM (Kernel Virtual Machine), портированного для работы на процессорах с архитектурой Power7. Работа по портированию велась с 2011 года. В списке рассылки разработчиков ядра Linux опубликованы патчи с реализацией порта KVM для процессоров MIPS. Порт KVM для MIPS базируется на использовании расширения MIPS-VZ, доступного в процессорах OCTEON III. Изначально KVM был доступен только для процессоров на базе архитектуры x86, поддерживающих специальные расширения Intel-VT и AMD-V для виртуализации. В прошлом году KVM был адаптирован для архитектуры ARM с задействованием имеющихся в процессорах ARM Cortex-A15 расширенных инструкций для виртуализации.
