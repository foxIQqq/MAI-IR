Релиз дистрибутива Arch Linux 2012.07.15

Представлен обновленный установочный образ дистрибутива Arch Linux 2012.07.15. Представленный выпуск имеет смысл только для формирования новых установок, так как дистрибутив Arch Linux разрабатывается с использованием модели непрерывных обновлений (Rolling-release), при которой обновления пакетов выходят постоянно и пользователь в любой момент имеет возможность перейти на последние версии программ без ожидания формирования очередного релиза дистрибутива. Таким образом, данный выпуск интересен не новыми версиями программ, а обновленным инсталлятором и набором утилит для формирования сборок дистрибутива. Вместо набора из шести отдельных сборок, на этот раз выпущена одна комбинированная сборка размером 370 Мб, сочетающая в себе набор пакетов для 32- и 64-разрядных архитектур, и пригодная для загрузки в Live-режиме и установки дистрибутива по сети. Iso-образ оформлен в гибридном виде, пригодном для установки с CD и с USB Flash, а также для загрузки по сети. Сборка поставляется с ядром Linux 3.4.4 и набором программ по состоянию на 15 июля. Сборки, включающие в себя срез репозитория core, предназначенные для автономной установки, больше не формируются. В будущем обновления iso-образов Arch Linux планируется выпускать ежемесячно, а не раз в год. Из ключевых отличий от прошлого выпуска отмечается прекращение поставки инсталлятора AIF (the Arch Installation Framework), вместо которого предлагается использовать набор простых установочных скриптов, подготовленных для упрощения установки дистрибутива на диск. Иными словами, для установки Arch Linux больше нельзя использовать интерактивный инсталлятор с управлением через меню. Для сглаживания прекращение поддержки AIF, планируется подготовить дополнительную документацию по установке для начинающих пользователей. В качестве причины удаления AIF отмечается стагнация пакета, вызванная отсутствием мэйнтейнера и заинтересованных в развитии разработчиков. В случае если найдутся заинтересованные энтузиасты и разработка AIF возобновится, фреймворк будет возвращён в состав сборок Arch Linux. Из заслуживающих внимания улучшений можно выделить: Использование цифровой подписи для верификации целостности и неизменности установочного образа. Перед началом установки рекомендуется проверить корректность загруженной сборки, используя команду "pacman-key -v iso-file.sig"; Включение по умолчанию проверки достоверности источника пакетов на основании цифровых подписей и автоматическая инициализация локального хранилища проверочных ключей на этапе загрузки. Таким образом, верификация по цифровым подписям доступна как в Live-режиме, так и после установки дистрибутива на диск. Указанная возможность позволяет гарантировать, что устанавливаемый из репозитория пакет не был подменён и получен в том виде, в котором его изначально подготовили разработчики, что особенно полезно при установке пакетов с произвольных зеркал; В установочный образ добавлена поддержка прямой загрузки через PXE; Расширены возможности Archiso, инструментария для формирования сборок.
