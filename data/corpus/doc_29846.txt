Обновление Linux-дистрибутивов для сетевых шлюзов: Alpine 2.1.5 и IPFire 2.9 Core 47

Вышел релиз Alpine Linux 2.1.5, специализированного дистрибутив для построения сетевых шлюзов, межсетевых экранов, VPN-серверов и VoIP-систем. Дистрибутив сформирован с претензией на повышенную безопасность и собран с патчами PaX и SSP (Stack Smashing Protection). Загрузочный iso-образ занимает 195 Мб, в качестве системой библиотеки используется uClibc, а в качестве стандартных утилит - BusyBox. Из улучшений версии Alpine 2.1.5 можно отметить обновление следующих приложений: linux-ядро 2.6.35.11, kamailio 3.1.2, asterisk 1.8.3, freeswitch 1.0.7, samba-3.5.8, dovecot 2.0.11, bind 9.7.3, pmacct 0.12.5 и openssl 1.0.0d. Кроме того, можно отметить выход дистрибутива для создания маршрутизаторов и межсетевых экранов IPFire 2.9 Core 47. IPFire отличается предельно простым процессом установки и организацией настройки через интуитивно понятный web-интерфейс, изобилующий наглядными графиками. Размер установочного iso-образа составляет 65 Мб. Для установки дополнений в IPFire используется специальный пакетный менеджер Pakfire. Система модульная, кроме базовых функций пакетной фильтрации и управления трафиком для IPFire доступны модули с реализацией системы для предотвращения атак на базе Snort, для создания файлового сервера (Samba, FTP, NFS), почтового сервера (Cyrus-IMAPd, Postfix, Spamassassin, ClamAV и Openmailadmin) и сервера печати (CUPS), организации VoIP шлюза на базе Asterisk и Teamspeak, создания беспроводной точки доступа, организации потокового аудио и видео-сервера (MPFire, Videolan, Icecast, Gnump3d, VDR). В новой версии IPFire проведена работа по устранению ошибок, обновлена версия PHP 5.3.5, добавлена возможность конфигурирования VLAN ID для проброса IGMP-трафика, обновлен механизм загрузки правил для Snort, налажен процесс NTLM-аутентификации в proxy.cgi.
