Увидел свет серверный дистрибутив Zentyal 2.2

После года разработки представлен релиз серверного Linux-дистрибутива Zentyal 2.2, развиваемого ранее под именем "eBox Platform". Дистрибутив построен на пакетной базе Ubuntu 10.04 и ориентирован на создание серверов для обслуживания локальной сети предприятий среднего и малого бизнеса. Zentyal поставляется как в виде отдельного установочного Live-дистрибутива (640 Мб), так и в виде набора пакетов для Ubuntu. Управление всеми аспектами работы дистрибутива производится через web-интерфейс, в рамках которого объединено 35 различных открытых платформ для управления сетью. Поддерживается быстрая организация работы шлюза, межсетевого экрана, почтового сервера, VoIP (Asterisk), VPN-сервера, прокси (squid), файлового сервера, системы для организации взаимодействия сотрудников, системы мониторинга, сервера для резервного копирования, системы обеспечения сетевой безопасности (Unified Threat Manager) и т.п. Из ключевых улучшений Zentyal 2.2, добавленных с момента выпуска прошлой стабильной ветки Zentyal 2.0, можно отметить: Улучшена производительность, увеличена отзывчивость интерфейса. Переписан бэкенд конфигурирования, который теперь использует для хранения данных NoSQL БД Redis 2.0; Проведена работа по улучшению юзабилити: выполнена реорганизация меню и интерфейса пользователя. Реализована возможность добавления объектов или сервисов во всплывающем диалоге непосредственно из выпадающих списков. Внесено множество мелких улучшений, делающих в целом работу более удобной. Улучшен процесс инсталляции: увеличена скорость установки, упрощен интерфейс выбора пакетов, добавлена возможность легкого выбора как индивидуальных пакетов, так и комплексных профилей, объединяющих группу пакетов определенной функциональности; Поддержка организации входа пользователей в сеть через аутентификацию на специальной web-странице (Captive portal). При работе через Captive portal) поддерживается индивидуальное ограничение пропускной способности для пользователей (после аутентификации в Captive portal динамически открывается доступ в сеть и устанавливается лимит на пропускную способность); Поддержка организации VPN-соединений с использованием ISSec и PPTP; Ведение лога для аудита изменения конфигурации и прочих действий администратора; Новая система интернационализации интерфейса. Все элементы интерфейса теперь могут быть переведены на другие языки. Изменения языка производится через подключение языковых пакетов; Добавление в интерфейс средств для управления виртуальными машинами KVM и VirtualBox; Улучшение работы модуля управления DNS: поддержка прозрачного режима кэширования, настройка перенаправления через другой DNS-сервер и конфигурирование записей SRV и TXT; Интеграция поддержки системы для организации совместной работы Zarafa 7; Добавление работающего как HTTP Proxy модуля для блокирования рекламы в транзитном трафике; Новый модуль мониторинга пропускной способности, который собирает информацию об интенсивности трафика и при необходимости ограничивает доступ; Поддержка установки и загрузки с USB-накопителей.
