Stanbol и Airavata переведены в категорию первичных проектов Apache

Организация Apache Software Foundation, после успешной проверки в "инкубаторе", объявила о придании проектам Apache Stanbol и Apache Airavata статуса первичных проектов Apache. Перевод в разряд первичных означает, что продукт и развивающее его сообщество подтвердили способность следования принципам разработки Apache и теперь готовы для самостоятельного существования, не требующего дополнительного надзора. Для управления разработкой и координации подготовки релизов новых первичных проектов будет выбран комитет, состоящий из активных участников разработки. Apache Airavata представляет собой тулкит, изначально разработанный Национальным Научным Фондом США как платформа для обеспечения работы инфраструктуры проекта атмосферных исследований. Airavata позволяет организовать создание, управление, выполнение и мониторинг заданий и приложений различного размера в окружениях распределённых вычислений, таких как кластеры, grid-сети или облачные системы. В состав входят компоненты: XBaya Workflow Suite - GUI для построения рабочего процесса и мониторинга; GFac - прослойка для превращения работающих в режиме командной строки научных приложений в сетевые сервисы; XRegistry - сервис регистрации для хранения информации о доступных сервисах и рабочих процессах; WS-Messenger - система обмена сообщениями, основанная на парадигме "публикация-подписка". В рамках проекта Apache Stanbol подготовлен набор повторно используемых компонентов для семантического управления контентом. Stanbol позволяет расширить традиционные системы управления контентом семантическими сервисами для классификации данных и автоматического улучшения контента, при котором представленный для обработки текст автоматически обогащается метаданными, аннотациями и ссылками, полученными из внешних источников, такими как информация о людях, местах и организациях.
