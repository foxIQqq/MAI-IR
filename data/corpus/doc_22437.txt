Обзор новшеств бета версии Red Hat Enterprise Linux 5.4

Компания Red Hat представила бета версию линейки дистрибутивов Red Hat Enterprise Linux 5.4: RHEL 5 Advanced Platform, RHEL Server и RHEL Desktop. Релиз планируется выпустить в середине августа. Основные новшества RHEL 5.4: Виртуализация: Полная поддержка инфраструктуры виртуализации на базе гипервизора KVM (Kernel-based Virtual Machine), завершение адаптации инструментов libvirt (например, virsh, virt-install и virt-manager) для работы с KVM. Новая стратегия Red Hat подразумевает использование в качестве базового механизма виртуализации KVM, вместо используемого в настоящее время Xen, поддержка которого также сохранится в полном объеме. Для повышения производительности сетевой и дисковых подсистемы в комплект включены специальные версии паравиртуализированных драйверов. Обеспечена поддержка широкого спектра 32- и 64-разрядных операционных систем в гостевом режиме, например, Red Hat Enterprise Linux, Windows XP, Windows Server 2003 и Windows Server 2008; Поддержка механизма аппаратной акселерации для системы виртуализации - SRIOV (Single Root I/O for Virtualization), позволяет оптимизировать совместный доступ виртуальных окружений к PCI картам. SRIOV работает совместно с механизмами IOMMU и VT-d на платформах AMD и Intel; Поддержка протокола VDI SPICE, позволяет улучшить время отклика при отображении данных на экране и снизить нагрузку на CPU без задействования специального оборудования. Поддержка Perl интерфейса для libvirt; Кластеры: Подготовлена кластерная сборка Red Hat Enterprise Linux (Cluster Suite GFS/GFS2) для платформы Power Systems в конфигурации LPAR, при которой не допускается смешивание узлов кластера разных архитектур; Новые агенты управления ресурсами для серверов приложений SAP SAP Instance и баз данных SAP; Агент управления ресурсами виртуальных машин (vm.sh) отныне базируется на libvirt; Десктоп: Обновление инфраструктуры звуковой системы ALSA, добавлена поддержка новых HD аудио чипсетов; Новые версии графических драйверов i810, intel, mga, ati и nv. Поддержка IGP чипсета AMD RS880. Системные улучшения: Полноценная поддержка FUSE. Тем не менее в состав дистрибутива пока не включены FUSE реализации файловых систем; Поддержка Fibre Channel поверх Ethernet (FCoE), включая набор утилит для упрощения конфигурации; Поддержка в Linux ядре новых трассировочных вызовов SystemTap (аналог DTrace для Solaris), в подсистемах vm, blktrace, sunrpc, network socket, page cache, scheduler; Поддержка измерения параметров Java приложений в системе профилирования OProfile. Безопасность Включение в состав дистрибутива GnuPG 2, для соответствия требованиям первого уровня сертификации FIPS 140-2; Предварительная реализация требований FIPS 140-2: openssl, openssh, libgcrypt, fipscheck, openswan, hmaccalc. Linux ядро: в ядро, основанное на версии 2.6.18, перенесены многие новые возможности из последних релизов 2.6.x. Например: поддержка T-состояний для ACPI; многочисленные улучшения, связанные с поддержкой архитектур i386, x86_64, PPC64, System Z (s390); поддержка RAID 4/5/10 в dm-raid; 34 новых и 43 обновленных драйвера для поддержки различных аппаратных устройств; Экспериментальные возможности для тестирования (Technology Previews): Новый драйвер be2net для поддержки сетевых адаптеров Emulex Tiger Shark; Поддержка файловой системы ext4 и пакета утилит e4fsprogs; Включение в состав GCC 4.4 с поддержкой сборки с новыми опциями оптимизации и задействованием OpenMP3; Дополнительная реализация функции malloc в glibc, которую можно активировать через установку переменной окружения MALLOC_PER_THREAD; Поддержка CIFS DFS. Пакеты с samba 3.3 и ctdb c поддержкой кластеризируемого TDB бэкенда. Запустив samba3x и ctdb на нескольких узлах кластера с файловой системой GFS, позволит организовать работу кластерного CIFS хранилища; SPICE X11 драйвер (xorg-x11-drv-qxl).
