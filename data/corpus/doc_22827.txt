Цели и задачи для будущего релиза Debian GNU/Linux

Представлен для обсуждения предварительный план подготовки следующего релиза Debian GNU/Linux 6.0 (Squeeze), который планируется выпустить весной или летом следующего года. Определены следующие базовые задачи для будущего релиза: Поддержка многоархитектурных пакетов, позволяющих устанавливать один пакет для 32- и 64-разрядных сборок; Увеличение скорости загрузки: переход на использование для выполнения shell скриптов вместо /bin/sh быстрой и упрощенной оболочки dash; использование возможностей системы инициализации с параллельным запуском сервисов; Увеличение качества пакетов, проведение чистки для прохождения пакетами тестов piuparts на установку из stable и обновления до unstable; тестировании пересборкой пакетов два раза подряд; Подготовка к внедрению новых форматов пакетов, реализация нового алгоритма сжатия пакетов (LZMA); Удаление устаревших библиотек; Добавление архитектур kfreebsd-amd64 и kfreebsd-i386, представляющих собой попытку совместить ядро FreeBSD с пользовательским окружением на базе glibc и GNU-утилит; Полная поддержка IPv6 (поддержка IPv6 во всех приложениях которые могут работать через IPv4); Более тесная интеграция с системой debtags, позволяющей сопоставить пакетам набор ключевых слов, для упрощения поиска нужного пакета; Поддержка работы с файлами большого размера во всех пакетах (некоторые программы не читают файлы больше 2/4 Гб). Под вопросом остается официальная поддержка архитектур alpha и hppa, для которых в настоящее время наблюдается ряд проблем, оказывающих блокирующее влияние на выпуск релиза.
