Вышел Linux-дистрибутив для создания межсетевых экранов IPFire 2.9 Core 45

Доступен релиз специализированного Linux-дистрибутива IPFire 2.9 Core 45, предназначенного для создания межсетевых экранов. IPFire отличается предельно простым процессом установки и организацией настройки через интуитивно понятный web-интерфейс, изобилующий наглядными графиками. Размер установочного iso-образа составляет всего 63 Мб. Для установки дополнений в IPFire используется специальный пакетный менеджер Pakfire. Система модульная, кроме базовых функций пакетной фильтрации и управления трафиком для IPFire доступны модули с реализацией системы для предотвращения атак на базе Snort, для создания файлового сервера (Samba, FTP, NFS), почтового сервера (Cyrus-IMAPd, Postfix, Spamassassin, ClamAV и Openmailadmin) и сервера печати (CUPS), организации VoIP шлюза на базе Asterisk и Teamspeak, создания беспроводной точки доступа, организации потокового аудио и видео-сервера (MPFire, Videolan, Icecast, Gnump3d, VDR). Из изменений в новом выпуске можно отметить: До версии 2.0.4 обновлен код сервиса Fireinfo, который по желанию пользователя производит отправку анонимизированной информации о параметрах текущей системы разработчикам проекта; Прокси-сервер squid обновлен до версии 3.1.10, исправлена недоработка, приводящая к ошибке "proxy unable to handle max download size correctly"; Система для обнаружения и предотвращения атак snort обновлена до версии 2.9.0.3; memtest86+ обновлен до версии 4.20; По умолчанию отключен модуль ядра geode_aes; Исправлены проблемы, связанные с некорректным восстановлением backupiso cd; Улучшена работа vpn-watch; В fcron отключена отправка почты и поправлены некоторые cron-задания; Правила блокирования исходящего трафика теперь запускаются опцией ведения лога.
