Релиз NTFS-3G 2010.8.8 с полной поддержкой сжатия

Компания Tuxera выпустила обновление fuse-драйвера файловой системы NTFS - NTFS-3G 2010.8.8, который позволяет работать с NTFS в любой операционной системе, поддерживающей FUSE, например, Linux, Mac OS X, FreeBSD, NetBSD, OpenSolaris, QNX и Haiku. Новые возможности в версии 2010.8.8: Добавлена полная, прозрачная для пользователя, поддержка сжатия при операциях чтения-записи; Появились две новые опции монтирования, относящиеся к сжатию: compression и nocompression (по умолчанию). Опция говорит драйверу о том, что все файлы, которые создаются в директории, имеющей атрибут сжатия (0x00000800, бит 11), нужно сжимать. Существующие файлы не будут затронуты этой опцией; Добавлена опция монтирования windows_names, которая запрещает использование в именах файлов зарезервированных символов. Другие изменения: Заново разрешено сжатие файлов в корневой директории; Изменены умолчания на противоположные для показа/скрытия скрытых файлов; Переопределено назначение UID/GUID файлов по умолчанию; Файлы, которые имеют первым символом точку, теперь помечаются скрытыми, если используется опция монтирования hide_dot_files; Опция монтирования default_permissions теперь называется просто permissions; Исправлено использование функции utimensat() при использовании fuse библиотеки версии выше 2.8; Драйвер больше не зависает при чтении испорченных сжатых файлов; Предотвращён возможный крах при создании потока; Исправлены все известные, найденные ошибки, связанные с записью сжатых файлов; Исправлена потенциальное затирание разряженных кластеров на сильно фрагментированной файловой системе; Исправлена проверка ошибок записи в функции setxattr(); Исправлена обработка некоторых сжатых файлов; Исправлена установка размера для сжатых файлов; Исправлена проверка состояния устройства при монтировании; Исправлена обработка опций монтирования silent и no_def_opts; Исправлено избыточное использование кластеров внутри больших дыр; Исправлено создание пустых расширенных атрибутов шифрования; Исправлено возвращение ошибок при записи в сжатые файлы; Исправлены проверки на исправность при закрытии сжатых файлов; Исправлено обнуление в конце кластера, бывшего до этого дыркой (ex-hole), для сжатых файлов; Исправлены проверки на исправность дескрипторов безопасности.
