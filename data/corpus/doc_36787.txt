Релиз Linux-дистрибутива Ubuntu 13.04

Официально представлен релиз Linux-дистрибутива Ubuntu 13.04 "Raring Ringtail". Дистрибутив доступен в редакциях для десктопов, серверов и cloud-окружений. Загрузочные iso-образы можно получить на данной странице. Одновременно выпущены релизы смежных проектов: Kubuntu (KDE 4.10), Xubuntu (Xfce 4.10), Lubuntu (LXDE), Mythbuntu, Edubuntu (с подборкой обучающего ПО) и Ubuntu Studio (для обработки мультимедиа информации). Кроме того, впервые выпущены сборки двух новых официальных ответвлений - Ubuntu GNOME, поставляемого со штатным десктоп-окружением GNOME, и UbuntuKylin, специализированная версия для пользователей из Китая. В отличие от прошлых промежуточных выпусков поддержка Ubuntu 13.04 будет осуществляться 9 месяцев (до января 2014 года). Ключевые новшества Ubuntu 13.04: Задействовано ядро Linux 3.8; Система инициализации Upstart обновлена до версии 1.8. В качестве экспериментальной возможности добавлена поддержка управления сеансами пользователей (режим Session Init), появившаяся в Upstart 1.7 и позволяющая использовать Upstart для инициализации графических сеансов. По умолчанию данная возможность будет отключена (для включения нужно раскомментировать строку "ubuntu" в файле /etc/upstart-xsessions); В базовом составе десктоп-редакции дистрибутива теперь поставляется только Python 3.3 (Python 2 остаётся в репозиториях); Пользовательская оболочка Unity обновлена до версии 7, отличающейся заметно возросшей производительностью. Из изменений в интерфейсе можно отметить новую реализацию меню для синхронизации данных с использованием сервиса Ubuntu One, новые диалоги завершения работы, новые визуальные эффекты (например, в процессе перемещения окна за пределы экрана в призрачной форме указывается на область, которую приложение займёт при отпускании кнопки мыши), новые пиктограммы панели и линз, упрощённый режим перебора из панели нескольких открытых окон приложения (для перебора окон можно использовать скроллинг мышью на пиктограмме приложения), новые настройки панели (например, скрытие иконки с виртуальными экранами, добавление кнопки сворачивания всех окон). К сожалению в выпуск Ubuntu 13.04 не вошла поддержка умных областей (Smart Scopes) и средств для управления приватной информацией. Smart Scopes представляют собой серверные обработчики, позволяющие представлять в Dash определённые виды локальной или внешней информации с определением порядка вывода областей с учётом их релевантности и востребованности пользователем. Cредства для управления приватной информацией позволят более точно контролировать какие сервисы использовать, например, определить фильтры для локального и глобального поиска (таким образом, в Ubuntu 13.04 как и в прошлом выпуске будет можно либо разрешить, либо запретить online-поиск целиком); В преддверие выхода версии Ubuntu для смартфонов в процессе подготовки выпуска 13.04 проведена большая работа по увеличению производительности и оптимизации потребления ресурсов на маломощных системах. В итоге удалось заметно увеличить отзывчивость интерфейса и сократить потребление памяти; Предоставлены средства для тестирования экспериментального дисплейного сервера Mir, который будет использоваться в будущих выпусках для запуска оболочки Unity Next, без использования X-сервера и Wayland. Новая оболочка, будет написана на Qt/QML и позволит применять единую кодовую базу при формировании окружений для различных классов устройств, автоматически адаптируясь к широкому диапазону экранных разрешений (от смартфонов до ПК) и систем ввода (клавиатура/мышь, сенсорный экран, пульт ДУ); Представлен новый сервис Friends, который заменил собой Gwibber. Сервис позволяет организовать доступ ко всем аккаунтам пользователя в социальных сетях через единый аккаунт в Ubuntu Online. Вместо клиентского приложения Gwibber предлагается установить пакет friends-app; Обновлены компоненты системы печати - CUPS 1.6.2 и cups-filters 1.0.34. Полностью переработана организация сетевой печати, которая теперь основана на использовании демона cups-browsed, позволяющего как обеспечить поддержку сетевых принтеров с использованием протокола Bonjour, так и сохранить совместимость с ранее поддерживаемым протоколом CUPS; Автоматически активирован сервис отправки уведомлений при обнаружении сбоев; В интерфейсе для установки обновлений представлен упрощённый вариант панели с выводом детальной информации, которая теперь показывает по отдельности только пользовательские приложения и вручную установленные пакеты, все библиотеки и системные пакеты теперь объединены в рамках одной записи; В серверной редакции до релиза Grizzly обновлена облачная платформа OpenStack, также представлены новые версии Ceph 0.56.4, MongoDB 2.2.4 и OpenvSwitch 1.9.0. Включены новые средства для развёртывания и управления работой распределённых приложений, позволяющие разработчику не задумываться где выполняются данные программы, в приватных или публичных облаках, на x86- или ARM-серверах, или на ноутбуке разработчика. Указанные средства подходят для локального тестирования и развёртывания решений для облачных систем. В основе лежит идея запуска локального "микро-облака" на базе облачной инфраструктуры OpenStack. После тестирования продукта в "микро-облаке", разработчик может переместить продукт во внешние облачные системы, такие как Amazon's EC2, или на удалённые серверы. Значительно расширены возможности web-интерфейса к Juju, который дополняет работающий в командной строке инструментарий средствами наглядного моделирования процесса развёртывания сервисов - конфигурация сервисов определяется путем перемещения функциональных блоков и определения связи между ними (например, для запуска MediaWiki можно активировать сервисы LAMP, MediaWiki, MySQL, memcached и определить связи между ними). Juju позволяет выполнять операции по развёртыванию сразу работоспособных сервисов, используя привычную семантику пакетного менеджера APT, но манипулируя готовыми и уже преднастроенными сервисами, которые могут включать в себя несколько пакетов и приложений. Для использования с Juju подготовлено боле 130 сервисов (Juju Charms), готовых к работе сразу после их установки. Среди таких сервисов различные СУБД (MySQL, PostgreSQL, Redis, Cassandra, Mongodb, Membase), web-приложения (wordpress, drupal, Node.js, Django, Ruby on Rails), серверные системы (lamp, tomcat, vsftpd), системы мониторинга, облачные платформы и т.д. В репозитории backports дополнительно доступна для тестирования новая экспериментальная ветка Juju, переписанная на языке Go (изначально Juju был написан на Python); Временно прекращена поставка WUBI, программы для установки Ubuntu из Windows, так как в ней не реализована поддержка Windows 8 и наблюдается ряд серьёзных недоработок; Обновлены входящие в поставку приложения, в том числе задействован офисный пакет LibreOffice 4.0; В Kubuntu отмечается обновление компонентов KDE до версии 4.10, поставка центра управления приложениями Muon Suite 2, обновление браузера Rekonq до версии 2.2.1, интеграция полноэкранного меню приложений Homerun (см. скриншот ниже), новая система настройки многомониторных конфигураций, поставка шрифтов Oxygen Sans, переработка интерфейса инсталлятора, поддержка доступа к мобильным устройствам по протоколу MTP (используется в Android 3+ вместо эмуляции USB-накопителя), поддержка загрузки в режиме UEFI Secure Boot, поставка OpenJDK7 в качестве реализации Java по умолчанию; В Xubuntu произведено увеличение размера iso-образа до 850 Мб, в поставку возвращены Gnumeric и GIMP, произведено обновление визуальной темы, добавлены новые версии приложений Catfish 0.6.1 и Parole 0.5.0; В Ubuntu Studio обновлено окружение Xfce 4.10 и задействован графический инсталлятор Ubiquity. Добавлены новые программы для работы со звуком supercollider, faust, lmms; программы для работы с видео: kdenlive и recordmydesktop; графические пакеты: krita и create-resources; В Lubuntu осуществлён переход на новую версию файлового менеджера pcmanfm 1.10, в который добавлены встроенные средства поиска; В Ubuntu GNOME задействовано десктоп-окружение GNOME 3.6. Выпущенный недавно релиз GNOME 3.8 доступен через PPA-репозиторий, но его решено не включать в релиз, так как данный выпуск недостаточно протестирован и приводит к возникновению ряда проблем в процессе использования. По сравнению с Ubuntu GNOME Remix 12.10 в Ubuntu GNOME по умолчанию задействован Firefox вместо Epiphany, Ubuntu Software Center и Update Manager вместо конфигураторов GNOME, LibreOffice вместо Abiword и Gnumeric. Дополнение: Марк Шаттлворт объявил о присвоении следующему выпуску Ubuntu (13.10) имени "Saucy Salamander".
