Дистрибутив Fedora 11 перешел на стадию бета-тестирования

Вышла первая бета версия Linux дистрибутива Fedora 11. Релиз Fedora 11 намечен на конец мая. Список новшеств: Обновление программ: Python 2.6 (переходные выпуск для упрощения миграции на Python 3.0), GCC 4.4, NetBeans 6.5, Xfce 4.6, Gnome 2.26, KDE 4.2.1, Firefox 3.1-beta3; Значительное ускорение загрузки и завершения работы. До появление окна входа в систему (gnome-session) должно проходить около 20 секунд; Система автоматической установки шрифтов и обработчиков заданных Mime-типов, позволяющая десктоп приложениям автоматически устанавливать необходимые программы, шрифты, мультимедиа кодеки и элементы клипарта; Встроенная возможность аутентификации через датчик отпечатков пальцев. Для настройки разработано специальное GUI приложение; Поддержка переключения режимов на уровне Linux ядра (KMS - kernel modesetting) для видеокарт Intel, ATI и Nvidia. Позволяет избежать скачков и мерцания при переключении видеорежима в процессе загрузки. В качестве драйвера по умолчанию для видеокарт Nvidia будет использован свободный драйвер Nouveau, имеющий поддержку 3D (ранее по умолчанию использовался 2D драйвер xf86-video-nv). Виртуальная консоль - позволяет более точно позиционировать указатель мыши и использовать более высокие разрешения экрана в окне виртуальной машины. Также предоставляет более простой интерфейс для проброса USB устройств в виртуальное окружение; Поддержка средств кросс-компиляции MinGW, при помощи которых можно будет собрать в Linux исполняемый файл для платформы Windows; Улучшение средств для управления громкостью, они будут перестроены на использование PulseAudio, что позволит определять разную громкость для подключаемых аппаратных устройств, таких как Bluetooth гарнитуры, звуковые карты с USB интерфейсом, web-камеры с микрофоном и т.п. Например, можно будет настроить проигрывание музыки только на Bluetooth гарнитуру, а системные звуки выводить через основную звуковую карту. В инсталляторе Anaconda полностью переписан код для работы с устройствами хранения данных, что позволит устанавливать Fedora на новые типы устройств. Также в программу установки добавлено средство для упрощения создания инсталляций минимального размера, с исключением всего лишнего; Поддержка групп контроля, позволяющих системному администратору разделять системные ресурсы на подгруппы, которые можно связывать с определенными приложениями; Интеграция DeviceKit, сервиса для управления работой устройств, слежении за их подключением и отключением. Предназначен для частичной замены HAL и устраняющего основные его ограничения. Дополнительно будут созданы подсистемы DeviceKit-disks и DeviceKit-power, для взаимодействия с блочными устройствами и управления питанием. Для удобства работы будут разработаны графические фронтэнды. В прошлых релизах была реализована поддержка установки медиакодеков при возникновении необходимости проигрывания определенного контента, в Fedora 11 эта возможность будет расширена аналогичной возможностью для установки прошивок (Firmware); Включение в комплект Yum-плагина Presto, реализующего бинарные "delta" обновления, при которых в пакет включаются только изменившиеся бинарные данные. Поддержка механизма DNSSEC по умолчанию; По умолчанию при установке дистрибутива будет предложено использовать файловую систему Ext4. В случае возникновения большого числа жалоб во время тестирования предварительных выпусков Fedora 11, будет осуществлен возврат к использованию по умолчанию файловой системы Ext3 и перемещению Ext4 в разряд опциональных возможностей. Разработка простого инструмента для настройки multi-seat систем для организации независимой работы нескольких пользователей на одном физическом компьютере, к которому подключено несколько мониторов, звуковых карт, клавиатур и манипуляторов "мышь". В комплект будет включен дисплейный менеджер MDM (Multi-Seat Display Manager). Замена RealVNC на TightVNC; В дистрибутив и программу установки интегрирована экспериментальная поддержка файловой системы Btrfs (для включения поддержки в инсталляторе в загрузочном приглашении нужно добавить опцию "icantbelieveitsnotbtr"); Для подписывания пакетов будет осуществлен переход на сигнатуры созданные с использованием SHA-256 хешей; По умолчанию 32-битные пакеты будут собраны с оптимизацией ддля CPU i586, а не i386 как было ранее. По умолчанию на 64-разрядных платформах будет использовано x86-64 ядро, даже при установке 32-разрядной сборки дистрибутива. Для 32-разрядных платформ будет использовано ядро с поддержкой PAE; Поддержка работы в режиме интернет-киоска, позволяющий войти в систему под пользователем guest для организации гостевого входа в систему, без права доступа к файлам других пользователей и без возможности сохранить что-либо на диске (для входа не нужен пароль, права максимально урезаны и все файлы автоматически удаляются после завершения сеанса). Для отключения данной возможности достаточно удалить пакет xguest; В дистрибутив интегрирована поддержка sVirt MAC, позволяющего использовать SELinux Mandatory Access Control (MAC) политики для гостевых виртуальных окружений; Новый набор приложений для управления питанием и новый демон, адаптивно изменяющий параметры энергопотребления в зависимости от нагрузки на систему.
