Фонд Apache вышел из комитета JCP, управляющего развитием Java

Организация Apache Software Foundation заявила о выходе из состава комитета JCP, принимающего решения о дальнейшем развитии Java SE/EE. В заявлении фонда Apache предоставляется объяснение совершенного шага, смысл которого сводится к следующим утверждениям: Недавнее голосование по принятию спецификаций Java SE 7/8 было последним шансом продемонстрировать, что комитет хоть как-то желает защитить статус JCP как открытого процесса, а также что буква и дух закона ещё что-то значат. Фонд Apache считает, что продолжая не выполнять свои обязательства по JSPA (Java Specification Participation Agreement), в частности, по-прежнему отказываясь предоставить проекту Apache Harmony лицензию на TCK для Java SE, Oracle тем не менее предложила на голосование Java Specification Request и лицензию, содержащую внутренние противоречия, строго запрещающую распространение независимых реализаций спецификаций, и, что более важно, запрещающую распространение независимых открытых реализаций спецификаций. Голосование являлось единственным рабочим рычагом комитета JCP как органа управления экосистемой Java-спецификаций, и, как объявлялось ранее, фонд Apache ожидал, что комитет в рамках своих возможностей защитит права Java-разработчиков, а также сохранит целостность структуры лицензирования JCP, закрепив возможность свободных реализаций и свободного распространения спецификаций JCP. Фонд Apache считает эту свою позицию разумной и обоснованной. Apache Software Foundation также считает, что утвердив спецификации Java SE 7, комитет потерпел поражение по обоим направлениям: члены комитета таким образом отказались поддержать права Java-разработчиков, и, утвердив условия лицензирования TCK для Java SE 7, они нарушили целостность структуры лицензирования JCP. Фонд Apache заявляет, что такой JCP не является открытым процессом: Спецификации Java - это проприетарная технология, которая должна лицензироваться напрямую авторами спецификаций на таких условиях, какие пожелают сами авторы спецификаций; Коммерческие интересы единственной компании, Oracle, будут продолжать грубо вмешиваться и искажать процесс прозрачного управления экосистемой; Распространение независимых реализаций JSR под свободными лицензиями, которые бы защищали пользователей от исков по нарушению прав интеллектуальной собственности от членов экспертной группы или авторов спецификаций - невозможно; Комитет не желает или не в состоянии реализовать свои основные полномочия в процессе управления процессом JCP. Учитывая всё вышеизложенное, представители фонда ASF поставили в известность отдел руководства программой JCP об отставке с поста члена комитета JCP. В силу этого фонд Apache удаляет всех своих официальных представителей из участия во всех JSR. Также фонд будет отказываться от возобновления членства в JCP, а также, разумеется, и от поста в комитете JCP. Несколько дней назад, комитет также покинул независимый эксперт Тим Перлс (Tim Peierls), мотивируя свой поступок тем, что комитет не может влиять на намерения Oracle и неспособен добиться отмены ограничений на распространения тестового пакета TCK.
