Увидел свет пакетный менеджер RPM 4.12.0

Компания Red Hat объявила о выходе новой ветки пакетного менеджера RPM 4.12.0. Проект RPM4 развивается компанией Red Hat и используется в таких дистрибутивах, как RHEL (включая производные проекты CentOS, Scientific Linux, AsiaLinux, Red Flag Linux, Oracle Linux), Fedora, SUSE, openSUSE, ALT Linux, Mageia, PCLinuxOS, Tizen и многих других. Параллельно независимой командой разработчиков развивался проект RPM5, который непосредственно не связан с RPM4. Наиболее заметные улучшения в RPM 4.12: Поддержка размещения в пакетах файлов, размером более 4 Гб; Поддержка тегов "suggests" "recommends" "supplements" и "enhances" для определения слабых зависимостей (weak dependency). Возможность автоматической генерации слабых зависимостей в rpmbuild; Увеличена производительность генерации пакетов и формирования цифровых подписей; Добавлена утилита rpm2archive для преобразования пакета в tar-архив; Прекращена поддержка коллекций ("collections"), которые развивались в качестве экспериментальной возможности; Новый программный интерфейс для создания плагинов. В состав выпуска включены плагины: systemd_inhibit, selinux и syslog. Для отключения всех плагинов предусмотрена опция "--noplugins"; Новый API для доступа к данным, находящимся в составе пакета; Поддержка новых архитектур: m68k, Sparc Niagara, ARM v6 и ARM v7 (определение встроенного FPU); В rpmdb добавлены опции "--exportdb" "--importdb" для экспорта и импорта БД; Реализован новый режим "--reinstall", который нормально обрабатывает изменения состава файлов (например, при переустановке с опцией "--excludedocs"); Частичная поддержка многопоточности; Большая порция внутренних изменений, проведение чистки кодовой базы.
