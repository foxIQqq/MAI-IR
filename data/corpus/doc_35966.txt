Стабильный релиз NTFS-3G и ntfsprogs 2013.1.13

Компания Tuxera опубликовала стабильный релиз свободного драйвера NTFS-3G 2013.1.13, работающего в пространстве пользователя с использованием механизма FUSE, а также комплекта утилит ntfsprogs 2013.1.13 для манипуляций с разделами NTFS. Код проекта распространяется в рамках лицензии GPLv2. Драйвер поддерживает чтение и запись данных в NTFS-разделах и может работать в широком спектре операционных систем, поддерживающих FUSE, в том числе в Linux, Android, Mac OS X, FreeBSD, NetBSD, OpenSolaris, QNX и Haiku. Предоставляемая драйвером реализация файловой системы NTFS, полностью совместима с операционными системами Windows XP, Windows Server 2003, Windows 2000, Windows Vista, Windows Server 2008 и Windows 7. Набор утилит ntfsprogs позволяет выполнять такие операции, как создание NTFS-разделов, проверка целостности, клонирование, изменение размера и восстановление удалённых файлов. Общие компоненты для работы c NTFS, используемые в драйвере и утилитах, вынесены в отдельную библиотеку. Изменения: ntfs-3g: Разрешены многократные монтирования раздела в режиме только для чтения; Опция монтирования "ro" включается принудительно при монтировании устройств, доступных только для чтения; При удалении файла его имя теперь сохраняется с целью упрощения восстановления удалённых файлов; Улучшено наследование ACL Windows-типа, в соответствии с требованиями Windows 8; Добавлено наследование флага set-group-id; Отменено выполнение необязательных обновлений списка задач, работающих при добавлении данных в файл; Запрещено монтирование при активном режиме быстрой перезагрузки (fast reboot) Windows 8; Для жёстких ссылок теперь отклоняется получение или выставление DOS-имён. В ntfsclone исправлено затирание пользовательских данных при создании образов метаданных; В ntfsclone и ntfslabel реализована опция для установки нового серийного номера; В ntfsfix добавлена возможность исправления резервной копии загрузочного сектора, а также исправлена ошибка при очистке списка потерянных кластеров; В ntfsinfo добавлено отражение в сводной информации о разделе сведений о размещении данных. Показано количество запусков и число фрагментов для выбранного файла; В ntfswipe добавлена опция затирания концов файлов и удалённых данных; В mkntfs в $UpCase вставлен поток $Info для совместимости с Windows 8; В драйвере и fuse-lite учтены особенности OpenIndiana.
