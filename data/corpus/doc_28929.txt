Компания Oracle будет развивать Oracle Enterprise Linux для платформы SPARC

Ларри Эллисон (Larry Ellison), руководитель компании Oracle, заявил о намерении обеспечения в дистрибутиве Oracle Enterprise Linux поддержки архитектуры SPARC, как одной из первичных платформ для продуктов Oracle. Ввести в промышленную эксплуатацию решения на базе Oracle Enterprise Linux для платформы SPARC планируется к моменту появления процессора Sparc T4 или T5, которые ожидаются в ближайшие 2-3 года. В настоящее время Oracle обеспечивает поддержку решений на базе Oracle Enterprise Linux только для x86- и x86_64-серверов, предоставляя для таких систем выбор установки Linux или Solaris. Для sparc-серверов доступен только Solaris, что по мнению руководства Oracle ограничивает для пользователей возможность выбора. Предоставление нескольких вариантов операционных систем позволит Oracle усилить конкурентные преимущества перед компаниями IBM и Hewlett Packard, которые уже поставляют для своих высокотехнологичных систем как Linux, так и HP-UX и AIX. Для желающих использовать Linux на sparc-серверах в настоящий момент наиболее доступными решениями являются CentOS и Debian, недостаток которых связан с отсутствием предоставления коммерческой технической поддержки. В Red Hat Enterprise Linux, на пакетной базе которого построен Oracle Enterprise Linux, поддержка платформы SPARC была прекращена после выхода релиза Red Hat Linux 6.2 в апреле 2000 года. Дополнительно, Ларри Эллисон представил новую категорию сервиса поддержки - Gold Standard Service, отличающегося использованием только рекомендованных Oracle программных конфигураций. Перед выходом каждого обновления и исправления, заранее определенные в рамках сервиса сочетания программных компонентов и настроек усиленно тестируются в лаборатории Oracle, что дает повышенные гарантии отсутствия сбоя при выполнении обновления.
