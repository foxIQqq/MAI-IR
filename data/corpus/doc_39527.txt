Выпуск web-браузера Chrome 34

Компания Google представила релиз web-браузера Chrome 34. Браузер построен на кодовой базе свободного проекта Chromium и отличается использованием логотипов Google, встроенной поддержкой Flash и PDF, наличием системы отправки уведомлений в случае краха, системой автоматической установки обновлений и передачей при поиске RLZ-параметров. Основные изменения: Интеграция возможности быстро получить ответ на интересующий вопрос с использованием механизма распознавания речи и сервиса Google Now. Для активации сервиса нужно на сайте google.com кликнуть на микрофон и разрешить распознавание команд "Ok Google". В дальнейшем достаточно при открытом в текущей вкладке сайте Google.com сказать "Ok Google" и без лишних кликов и нажатий клавиш задать вопрос, после чего браузер при помощи системы распознавания речи сформируют обращение к поисковой системе и выведет ответ. Через систему голосового ввода также можно добавлять напоминания и устанавливать таймер; Поддержка импорта на новый компьютер уже созданного для других систем списка контролируемых пользователей, в котором выделены работающие с компьютером пользователи для которых требуется дополнительный контроль за их активностью в Сети. Через интерфейс chrome.com/manage имеется возможность проследить за работой таких пользователей и задать для них ограничения доступа к сайтам (например, можно разрешить открытие только конкретных сайтов); Реализация технологии отзывчивых изображений (Responsive Images), дающей возможность определить через новый атрибут "srcset" тега "img src" серию вариантов изображений, оптимизированных для различных типов устройств, экранных разрешений и уровня масштабирования. В будущих выпусках ожидается появление нового блочного тега picture, позволяющего определить более сложный сценарий применения различных вариантов изображения. Новое CSS-свойство font-variant-ligatures, позволяющее управлять лигатурой (соединением букв) в тексте; В движке Blink проведена чистка ряда устаревших и редко используемых возможностей; Обеспечена возможность сохранения и автозаполнения полей ввода паролей, несмотря на установку опции autocomplete=off. Реализовано новое отдельное оформление при запуске браузера в режиме Windows 8 Metro; Множество изменений под капотом, направленных на увеличение стабильности и производительности; В следующем выпуске Chrome для Linux будет удалён код обеспечения поддержки плагинов NPAPI (Netscape Plugin Application Programming Interface), в настоящее время по умолчанию уже применяется блокировка плагинов NPAPI. Для Windows и OS X удаление поддержки NPAPI будет произведено до конца года. Причиной заблаговременного прекращения поддержи NPAPI в версии для Linux является переход с GTK+ на собственный графический стек Aura, который ожидается в следующем выпуске. В качестве альтернативы NPAPI компания Google продвигает PPAPI (Pepper API), который разработан для устранения проблем с переносимостью и производительностью классического NPAPI, а также для упрощения создания плагинов, работающих в виде отдельных процессов; Кроме нововведений и исправления ошибок, в новой версии устранена 31 уязвимость, из которых 21 проблема помечена как опасная. Ни одной уязвимости не присвоен статус критической проблемы, позволяющей обойти все уровни защиты браузера и совершить атаку на систему пользователя. 14 проблем выявлены сотрудниками Google в процессе внутреннего аудита кодовой базы. В рамках программы по выплате денежного вознаграждения за обнаружение уязвимостей для текущего релиза компания Google выплатила 8 премий на сумму 29500 долларов США (две премии по $5000, три по $3000, три по $2000, одна $1500 и три по $1000).
