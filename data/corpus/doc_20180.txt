Фонд свободного ПО выступил против принятия TLS Authorisation в качестве стандарта

Фонд свободного программного обеспечения (Free Software Foundation) призвал бойкотировать принятие комитетом IETF в качестве Интернет стандарта расширений авторизации для протокола TLS (Transport Layer Security), используемого для обеспечения аутентификации и безопасной передачи данных. Проблема заключается в задействовании в TLS расширениях для авторизации ряда запатентованных компанией RedPhone Security технологий, что по мнению FSF является существенным аргументом против стандартизации данного протокола. Комитет IETF будет принимать отзывы по поводу принятия TLS Authorisation в качестве стандарта до 11 февраля. FSF предлагает всем желающим отправить сообщение по адресу ietf@ietf.org с выражением своего несогласия. В будущем компания RedPhone вправе начать сбор лицензионных отчислений за реализацию TLS Authorisation, поэтому Free Software Foundation настаивает на приостановке принятия стандарта до утверждения компанией RedPhone пункта о возможности полностью безвозмездного использования патента.
