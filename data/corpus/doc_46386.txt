Определены основные векторы развития форков Unity 8 от проектов UBports и Yunit

Проекты UBports и Yunit, после заявления компании Canonical о прекращении разработки Uniy 8, выступившие с инициативами по созданию форков данного пользовательского окружения, опубликовали основные планы дальнейшей работы. Возникновение двух разных форков объясняется различиями в целях UBports и Yunit: первый ориентирован на создание оболочки и прошивки для мобильных устройств, а второй сосредоточен на создании решения для рабочего стола. Кроме того, Yunit пытаются создать новое сообщество, открытое для всех желающих, в то время как UBports позиционируется как уже сформировавшееся сообщество, включающее энтузиастов, вовлечённых в портирование Ubuntu Touch для официально не поддерживаемых устройств. При этом форки готовы к сотрудничеству и планируют организовать совместную работу над решением общих задач. Основные планы по разработке форка от UBports: В качестве эталонных устройств, для которых будет развиваться прошивка, отмечаются Nexus 5, Fairphone 2 и Oneplus One. Все официальные устройства, сопровождением которых занималась компания Canonical, за исключеним Meizu PRO 5 и Bq Aquaris M10, будут поддерживаться проектом, но в качестве устаревших ("legacy") платформ, для которых будут переноситься только исправления значительных проблем. Для Meizu PRO 5 и Bq Aquaris M10 возможно будут выпускаться актуальные прошивки UBports. Для распространения приложений вместо Ubuntu Store запущен собственный каталог openstore.ubports.com; Для аутентификации доступа к сервисам возможно будут использоваться наработки Ubuntu One; Несмотря на то, что Canonical продолжит разработку дисплейного менеджера Mir для Интернета вещей (IoT), сопровождение для мобильных и настольных систем не гарантируется. Поэтому разработчики UBports не хотят зависеть от тенденций в разработке Mir и намерены создать прослойку для обеспечения работы API Mir поверх Wayland; На базе проекта Anbox будут подготовлены средства для запуска штатных Android-приложений. Набор Google Play Services поставляться не будет из-за необходимости получения лицензии у Google. В рамках форка также будет продолжена разработка базовых мобильных приложений Ubuntu Touch. Работа будет вестись на GitHub, вместо Launchpad/Bazaar; В качестве формата пакетов будет применяться Snap; Решение о продолжении или прекращении разработки Ubuntu SDK пока не принято. Особенности проекта Yunit: В репозиторий форка Yunit уже начали поступать изменения и исправления. В репозитории UBports пока отcутствует какая-либо активность. Обсуждается перевод Yunit с Mir на Wayland, при этом планируется не переписать все привязанные к Mir компоненты, а реализовать прослойку, позволяющую запускать Mir в качестве Wayland-клиента. Возможно, данная работа будет выполнена совместно с UBPorts, которые также движутся в этом направлении; Стартовал конкурс по разработке логотипа.
