Разработчики GNOME и KDE на пути к созданию альтернативы Dropbox

Разработчики GNOME и KDE начали обсуждение возможности объединения своих усилий в направлении создания свободной альтернативы таким закрытым сервисам централизованного хранения данных, как Dropbox, iDisk, Mandriva Click’n Backup или Ubuntu One, предоставляющим пользователю доступ к удаленному хранилищу данных, которое можно использовать для хранения файлов, для организации совместного доступа или для проведения синхронизации данных между несколькими машинами. В настоящее время компания Canonical активно развивает сервис Ubuntu One, который не лишен ряда недостатков, среди которых закрытый характер разработки и излишняя привязка к Ubuntu и GNOME, хотя тестируются версии клиентов для KDE и Fedora. В настоящее время проекты GNOME и KDE выдвинули в рамках инициативы Google Summer of Code несколько идей, связанных с созданием свободной альтернативы сервису Dropbox. В отличие от закрытых сервисов, новый свободный проект позволит администраторам развернуть полностью подконтрольный сервис хранения на локальных мощностях. В настоящий момент разработчиками обсуждается совместное продолжение развития изначально созданного рамках инициативы KDE Social Desktop проекта ownCloud, представляющего собой серверное решение для организации совместного хранения документов и файлов, а также для обеспечения обмена данными между пользователями. В качестве лицензии предлагается использовать AGPL. Среди идей по развитию ownCloud называется создание системы синхронизации данных между несколькими ownCloud-серверами, что позволит создавать внутри организации приватные ownCloud-серверы, которые будут синхронизированы с публично доступными серверами. Клиентская часть будет максимально переносимой, сможет работать не только в GNOME и KDE во всех популярных Linux-дистрибутивах, но и в Windows, Mac OS X и мобильных платформах. Для быстрого доступа с хранимым данным с чужих машин будет подготовлен web-интерфейс. Из предусмотренных планов возможностей также можно отметить: хранение данных в древовидной структуре; возможность монтирования хранилища как дискового раздела; версионный характер хранения данных, с возможностью просмотра истории операций и доступа к прошлым версиям файлов; поддержка сервером нескольких разных бэкендов для хранения; автоматическое резервное копирование. В заключение стоит отметить, что в прошлом году компания Novell и разработчики openSUSE представили очень похожий по своей сути свободный проект iFolder, направленный на создание безопасного, простого в использовании и доступного из любой точки сети сетевого хранилища для синхронизации файлов между машинами и обмена данными между группами пользователей. После установки iFolder, сохраненные в определенные локальные директории данные автоматически загружаются на сервер хранения и доставлены на других машинах пользователя. Клиентская часть iFolder доступна для Linux, Mac OS X и Windows, поддерживается управление сохраненными файлами через web-браузер.
