Выпуск Red Hat Enterprise Linux 6.8

После десяти месяцев разработки компания Red Hat представила релиз дистрибутива Red Hat Enterprise Linux 6.8, в котором проведена работа по исправлению ошибок, реализации поддержки нового оборудования и добавлению новых возможностей. Установочные образы RHEL 6.8 доступны для загрузки только зарегистрированным пользователям Red Hat Network (RHN). Ветка RHEL 6.x развивается параллельно с веткой RHEL 7.x и будет поддерживаться до 30 ноября 2020 года. Выпуск RHEL 6.8 является последним в рамках первой стадии поддержки, подразумевающей включение функциональных улучшений. RHEL 6.9 перейдёт на вторую стадию сопровождения, на которой приоритеты сместятся в сторону исправления ошибок и безопасности, с внесением незначительных улучшений, связанных с поддержкой важных аппаратных систем. Ключевые новшества Red Hat Enterprise Linux 6.8: В состав включен инструмент резервного копирования Relax-and-Recover, позволяющий создавать бэкапы в форме ISO-образов, позволяющих быстро восстановить систему просто загрузившись с них; Упрощён процесс подключения дополнительных репозиториев, требуемых в процессе возникновения ошибок при поиске зависимостей. В категорию опциональных перенесены пакеты gnome-devel-docs, libstdc++-docs и xorg-x11-docs; Реализация IPsec переведена с Openswan на Libreswan, что дало возможность пользователям RHEL 6 получить доступ к расширенным средствам защиты VPN; В SSSD улучшена переносимость с Active Directory и серверами управления индентификацией: реализовано кэширование initgroups и запросов аутентификации для снижения интенсивности обращений к Active Directory, добавлена утилита adcli для упрощения управления объектами host, user, group в домене Active Directory, реализована поддержка маппинга UID и GID для определённых клиентов, добавлена поддержка аутентификации через смарткарты для системных входов и привилегированных операций (например, вызов sudo); Прекращена поддержка SSLv2. В OpenLDAP, yum, stunnel, vsftpd, Git и Postfix реализована поддержка TLSv1.2. В nss добавлена поддержка сертификатов ECDSA и включен по умолчанию TLSv1.2. В OpenJDK 8 добавлена поддержка ECC (Elliptic Curve Cryptography). В OpenJDK 6 и OpenJDK 7 по умолчанию отключен алгоритм RC4; Добавлена утилита dmstats, предоставляющая средства для отображения статистики по использованию хранилищ и производительности дисковых операций; Максимальный размер файловой системы XFS увеличен со 100 до 300 Тб, реализована возможность просмотра статистики XFS через /sys/fs/; В dmidecode добавлена поддержка SMBIOS 3.0.0. В mcelog добавлена поддержка 6 поколения процессоров Intel Core, Intel Xeon E3 v5 и актуальных моделей CPU под брендами Intel Pentium и Intel Celeron; В OProfile добавлена поддержка 6 поколения процессоров Intel Core и Intel Xeon Processor D-1500; До актуального выпуска обновлён стек Open MPI; Добавлена утилита Omping (Open Multicast Ping) для тестирования IP multicast; Вывод installed-rpms упрощён и приведён к читаемому формату; SystemTap обновлён до версии 2.9; Добавлен пакет java-1.8.0-ibm с реализацией Java 8 от IBM; Обновлены компоненты рабочего стола: LibreOffice 4.3.7.2, Mesa c поддержкой GPU 6 поколения процессоров Intel Core, icedtea-web 1.6.2; Расширена поддержка оборудования: бэкпортирован драйвер rtlwifi для беспроводных чипов RTL8188CE, добавлена поддержка новых USB to serial конвертеров Sealevel, карт-ридеров O2Micro 8520 и Realtek 5229, устройств Wacom Cintiq 27QHD и Wacom Intuos PT, процессора AMD GX-212JC, драйвер ixgbe обновлён до версии 4.2.1 с поддержкой новых контроллеров семейства Intel X550 Ethernet, обновлены драйверы e100, e1000e, bnx2, lpfc; В kickstart добавлена поддержка загрузки файлов по HTTPS; В установщике добавлена поддержка автоматической настройки сетевых устройств при помощи тегов 802.1q VLAN из iSCSI Boot Firmware Table (iBFT); Появилась возможность привязки меток SELinux к блочным устройствам, создаваемым через udev; В планировщик sch_qfq добавлена поддержка более точного и эффективного алгоритма QFQ+ (Quick Fair Queuing Plus); Расширены средства для работы в роли гостевой системы под управлением гипервизора Hyper-V: добавлена поддержка хранилищ с 4096-байтовыми секторами, поддержка TRIM, поддержка 2-го поколения виртуальных машин, поддержка протоколов Windows 10 и Windows Server 2016. Добавлен пакет WALinuxAgent (Microsoft Azure Linux Agent); В virtio-win добавлены драйверы для Windows 10 (можно запускать Windows 10 в KVM); Добавлена утилита guest-set-user-password для установки пароля любого пользователя гостевых систем в QEMU и KVM; В PowerVM virtual Ethernet driver (ibmveth) добавлена поддержка Large Send Offload (LSO) и Large Receive Offload (LRO); Добавлен пакет c системой синхронизации точного времени chrony; Добавлен пакет с кэширующим рекурсивным DNS-сервером unbound; Добавлен пакет squid34 с новым выпуском прокси-сервера Squid 3.4.14; В NetworkManager добавлена поддержка iBFT и возможность прикрепления определённых сетей Wi-Fi к заданному диапазону частот; В конфигураторе system-config-network добавлены новые поля PEERDNS и ONBOOT, в список сетевых интерфейсов добавлена кнопка удаления; Подготовлен обновлённый системный образ, позволяющий переносить традиционные окружения в форму изолированных контейнеров, пригодных для запуска под управлением Red Hat Enterprise Linux 7 и Red Hat Enterprise Linux Atomic Host; В разряд не рекомендуемых к использованию (deprecated) в ветке RHEL 6.x переведены технологии Btrfs, eCryptfs и seabios, которые ранее позиционировались в дистрибутиве RHEL 6 как экспериментальные возможности (Technology Preview), не рекомендованные к применению в промышленных решениях. Это обычная практика досрочного прекращения поддержки возможностей категории "Technology Preview" в старых ветках RHEL, переходящих на следующий уровень сопровождения. В RHEL 7.x поддержка данных систем остаётся без изменений.
