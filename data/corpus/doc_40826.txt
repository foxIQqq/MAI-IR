Выпуск Red Hat Enterprise Linux 6.6

Компания Red Hat представила релиз дистрибутива Red Hat Enterprise Linux 6.6, в котором проведена работа по исправлению ошибок, реализации поддержки нового оборудования и добавлению новых возможностей. Установочные образы RHEL 6.6 доступны для загрузки только зарегистрированным пользователям Red Hat Network (RHN). Ключевые новшества Red Hat Enterprise Linux 6.6: В ядро Linux добавлены улучшения, связанные с работой блокировок, что позволило добиться повышения эффективности работы на крупных системах с архитектурой NUMA; Поддержка технологии RDMA (Remote Direct Memory Access) поверх Ethernet (RoCE), предназначенной для обеспечения прямого доступа к памяти другой машины; Реализация механизма поллинга сокетов (busy-poll), позволяющего снизить задержки в передаче пакетов и повысить однородность сетевого потока; Поддержка новых сетевых адаптеров с пропускной способностью 40GbE от компаний Solarflare, Intel, Emulex, Mellanox, и Chelsio; Добавлен экспериментальный модуль dm-cache, предназначенный для ускорения доступа к жестким дискам через применение кэширования на SSD-накопителях; Добавлен экспериментальный модуль "dm-era", предназначенный для поддержания списка блоков, изменённых за определённый пользователем промежуток времени (можно отслеживать изменённые блоки для систем резервного копирования или частично сбрасывать кэш для восстановления состояния после отката на снапшот); Добавлен фреймворк Performance Co-Pilot (PCP) для мониторинга производительности и организации управления в распределённых окружениях; Улучшена управляемость ресурсами в крупных, высоконадёжных конфигурациях; Полная поддержка технологий балансировки нагрузки HAProxy и keepalived; Улучшение средств для работы Red Hat Enterprise Linux в гостевых окружениях различных платформ виртуализации, в том числе добавлена возможность использования RHEL в виртуальных машинах второго поколения в Microsoft Hyper-V Server 2012 R2; Поддержка передачи данных об энтропии из хост-системы в генератор псевдослучайных чисел гостевой системы; Добавлен модуль ядра Open vSwitch; Увеличена производительность сетевых операций и доступа к хранилищам в виртуальных окружениях; Дополнение High Performance Networking (HPN) больше не поставляется в форме отдельного продукта и интегрировано в базовый состав дистрибутива; В состав включен пакет gdk-pixbuf2 с новой версией библиотеки для загрузки различных типов изображений, используемой в GTK+ и Clutter. Добавлен пакет с OpenJDK 8.
