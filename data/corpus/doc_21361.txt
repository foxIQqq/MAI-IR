Вышел релиз системы управления web-контентом concrete 5.3

Вышел релиз перспективной системы управления web-контентом concrete 5.3, исходные тексты которой написаны на PHP и распространяются в рамках свободной лицензии MIT. Concrete5 концептуально отличается от таких сисетем, как Drupal, Joomla и Wordpress, своей ориентацией не на программиста, а на типичного неквалифицированного пользователя. Все операции по развертыванию и созданию каркаса сайта максимально упрощены. По замыслу авторов проекта, после создания сайта, управление им не должно требовать привлечения специалистов, наполнение должно быть таким же простым как подготовка документа в текстовом процессоре. Некоторые возможности для владельцев сайтов: Редактирование в WYSIWYG режиме; Панель инструментов для изменения сайта на лету; Автоматическое создание элементов навигации; Режим предварительного просмотра изменений перед их активацией; Версионный контроль изменений сайта; Встроенный в панель управления файловый менеджер с редактором фотографий на базе кода Picnik; Гибкая система управления правами доступа: группы редакторов, группы модераторов, группы писателей и т.п. Перемещения контента между разделами в Drag-n-Drop режиме; Возможность копирования блоков и функциональности из одного части сайта в другую; Интегрированная система проверки орфографии; Автоматическая установка дополнений и визуальных тем; Дополнение для создания интернет-магазина; Планировщик размещения контента, позволяет размещать подготовленный контект не сразу, а в определенное время; Интегрирована система организации слайд-шоу, вставки You Tube роликов, гостевая книга, голосования, опросы, средства поиска, RSS и т.д. Возможности для разработчиков: В разработке используется парадигма MVC (Model View Controller) и объектно-ориентированный подход. Полное разделения контента, внешнего вида и функциональности; Расширенная система управления правами доступа на основе пользователей и групп; Гибкая и мощная система создания шаблонов; Встроенная поддержка контроля версий изменений; Поддержка кеширования; Специальный API для разработчиков; Поддержка внешних механизмов аутентификации пользователей (openID); Ведение лога входа пользователей; Возможность формирования итоговой работы по созданию сайта в виде пакета для быстрого развертывания на хостинге клиента; Возможность централизованной обработки группы независимых сайтов одной инсталляцией CMS;
