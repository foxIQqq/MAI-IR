Выпуск облачной платформы Apache CloudStack 4.12

После года разработки представлен релиз облачной платформы Apache CloudStack 4.12, которая позволяет автоматизировать развертывание, настройку и поддержание приватной, гибридной или публичной cloud-инфраструктуры (IaaS, инфраструктура как сервис). Платформа CloudStack была передана Фонду Apache компанией Citrix, которая получила проект после поглощения компании Cloud.com. Установочные пакеты подготовлены для RHEL/CentOS и Ubuntu. CloudStack не зависит от типа гипервизора и позволяет использовать в одной cloud-инфраструктуре одновременно Xen (XenServer и Xen Cloud Platform), KVM, Oracle VM (VirtualBox) и VMware. Для управления пользовательской базой, хранилищем, вычислительными и сетевыми ресурсами предлагается интуитивно понятный web-интерфейс и специальный API. В простейшем случае облачная инфраструктура на базе CloudStack состоит из одного управляющего сервера и набора вычислительных узлов, на которых организуется выполнение гостевых ОС в режиме виртуализации. В более сложных системах поддерживается использование кластера из нескольких управляющих серверов и дополнительных балансировщиков нагрузки. При этом, инфраструктура может быть разбита на сегменты, каждый из которых функционирует в отдельном дата-центре. Основные новшества: Для пользователей всех типов предоставлена возможность создания виртуальных сетей на канальном уровне (L2); Реализована поддержка удалённой отладки управляющих и рабочих серверов, а также агентов KVM; Добавлена поддержка offline-миграции окружений из VMware; В API добавлена команда для вывода списка управляющих серверов; Обновлены библиотеки, используемые для построения web-интерфейса (например, jquery); Расширена поддержка IPv6, предоставлена возможность отправки данных через виртуальный маршрутизатор и вычисления адресов IPv6 вместо выдачи готовых из пула. Для IPv6 добавлен отдельный набор фильтров ipset; Для XenServer реализована поддержка online-миграции неуправляемых хранилищ в управляемые хранилища; Для решений на базе гипервизора KVM переработана поддержка групп безопасности (Security Group), обеспечена передача на управляющий сервер корректных данных о доступной памяти, в сборщик статистики добавлена поддержка БД influxdb, реализовано использование libvirt для ускорения ввода/вывода, переработан скрипт настройки VXLAN, добавлена поддержка IPv6, включена поддержка DPDK, добавлены настройки для запуска в гостевых системах Windows Server 2019, реализована live-миграция виртуальных машин с корневым разделом в файловом хранилище; В клиентском интерфейсе предоставлена возможность редактирования протокола в правилах ACL; Добавлена возможность удаления локального первичного хранилища. В свойствах сетевого адаптера обеспечено отображение MAC-адреса; Прекращена поддержка Ubuntu 14.04 (официальная поддержка LTS выпуска Ubuntu 14.04 прекращается в конце апреля).
