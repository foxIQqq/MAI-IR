Подборка заметок про производительность MySQL

Несколько интересных заметок про MySQL: "MySQL-Memcached or NOSQL Tokyo Tyrant" (Часть 1, Часть 2, Часть 3) - серия статей про использование MySQL для высоконагруженных проектов. Проведена оценка повышения производительности за счет использования связки MySQL + Memcached, а также распределенных хранилищ данных, подобных Tokyo Tyrant и Cassandra; "How number of columns affects performance ?" - оценка влияния числа столбцов в таблице на производительность выборки. Проблемы проявляются при использовании типов не фиксированного размера. Когда результаты сканирования таблицы укладываются в память, то MyISAM на простых таблицах с большим числом записей оказывается значительно быстрее Innodb; "Quick comparison of MyISAM, Infobright, and MonetDB" - сравнение производительности и размера базы MyISAM со специализированными БД Infobright и MonetDB (хранят данные в построчном формате, с группировкой данных по строкам, а не столбцам) при хранении большого объема данных. Во второй части оценивается производительность при выполнении операций анализа данных со статистикой движения воздушных судов. "How (not) to find unused indexes" - заметка про выявление неиспользуемых индексов; "Tuning for heavy writing workloads" - советы по тюнингу конфигураций MySQL с большой интенсивностью записи данных; На сайте percona.tv опубликованы видеозаписи докладов с конференции Percona Performance Conference; Выпущена 19 сборка MySQL с патчами от компании Percona. В новой версии осуществлен переход на MySQL 5.0.86, включение в комплект патча userstatv2 и добавления кода для оптимизации работы больших запросов для InnoDB баз.
