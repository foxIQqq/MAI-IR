В GitHub добавлен инструментарий для обеспечения разработки больших проектов

Сервис GitHub.com, ориентированный на хостинг Git-репозиториев, объявил о реализации возможности создания нового типа проектов "Organizations", в рамках которых поддерживается несколько репозиториев, доступ к которым имеет группа лиц. Новый сервис ориентирован на большие проекты и содержит в себе механизмы по упрощению организации работы над проектом большого числа разработчиков. Полномочия по управлению новым типом репозиториев могут быть распределены среди группы лиц, что очень удобно в ситуации когда у проекта имеется несколько активных разработчиков. Имеющие доступ к репозиторию разработчики образуют своеобразную иерархию, каждому уровню которой могут быть предоставлены отдельные полномочия и права доступа. Система разграничения доступа позволяет делегировать такие права, как выполнение операции pull (только чтение), выполнение push и pull (чтение и запись), выполнение push, pull и команд администратора. Для свободных проектов сервис бесплатен. В качестве примера больших проектов, перешедших на использование "Organizations" называются MongoDB и RubyGems.
