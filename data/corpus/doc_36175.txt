Релиз сетевого конфигуратора NetworkManager 0.9.8

После восьми месяцев разработки представлен релиз интерфейса для упрощения настройки параметров сети - NetworkManager 0.9.8, а также апплета с реализацией индикатора для панели GNOME и модулей для поддержки VPN, OpenConnect, PPTP, OpenVPN и OpenSWAN. Предварительный выпуск NetworkManager 0.9.8 был интегрирован в состав дистрибутива Fedora 18. Апплет из состава NetworkManager 0.9.8 будет включён в релиз GNOME 3.8. Среди улучшений, представленных в NetworkManager 0.9.8: Возможность настройки работы в виде точки доступа к беспроводной сети (AP Hotspot), если драйверы для установленной сетевой карты поддерживают работу в режиме точки доступа; Поддержка настройки сетевых интерфейсов в режиме сетевого моста; Поддержка 4G LTE; Поддержка автоматической активации VPN-соединений для определённых типов соединений; Устранение ряда мешающих комфортной работе проблем с WiFi. Например, NetworkManager теперь не выводит после случайных сбоев повторное предложение ввести пароль, пароль теперь запрашивается только при инициировании нового соединения или при явном сбое аутентификации. Кроме того, NetworkManager не пытается автоматически соединиться с беспроводной сетью, если для данной сети все прошлые попытки были неудачны; Большое число исправлений и улучшений, связанных с поддержкой IPv6. Например, добавлена поддержка DHCPv6 DDNS (Dynamic DNS) и устранены проблемы с дублированием маршрутов IPv6; Обеспечена поддержка интеграции с новыми выпусками демона ModemManager 0.7/0.8, используемым для настройки GSM/CDMA/UMTS-модемов. ModemManager берет на себя непосредственное взаимодействие с модемом или мобильным телефоном, а для внешних приложений предоставляет унифицированный интерфейс, который поддерживает такие операции, как установка соединения, отправка SMS, чтение и изменение адресной книги, вывод данных о качестве сигнала, чтение параметров GPS-приемника; Поддержка упреждающего кэширования ключей (Proactive Key Caching) для WiFi-соединений WPA Enterprise; Переработан и значительно улучшен GTK-интерфейс для редактирования параметров сетевых соединений; Улучшена обработка режимов rfkill/Airplane; Устранены проблемы с заданием DNS-серверов при использовании некоторых типов мобильных адаптеров для широкополосного доступа; В библиотеку libnm-glib добавлена порция изменений для улучшения работы в асинхронном режиме; Поддержка использования systemd для контроля за переходом в спящий и ждущий режимы; Для объединённых сетевых адаптеров (bonding) теперь возможно назначение slave-устройств; Прекращена поддержка dhclient v3.
