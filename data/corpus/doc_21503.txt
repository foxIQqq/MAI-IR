Релиз VirtualBox 2.2.2. Инициатива по созданию открытого стандарта для cloud-систем

Вышел релиз системы виртуализации VirtualBox 2.2.2, содержащий исправления 30 ошибок, накопившихся с момента выхода версии 2.2.0. Например, решены проблемы с буфером обмена при использовании X-сервера на гостевой и хост-системах; исправлены проблемы с использование D-Bus на Linux хосте; устранена ошибка которая приводила к повреждению виртуальных дисковых образов; переработан диалог настройки сети в GUI интерфейсе (с учетом настройки NAT); добавлена начальная поддержка Mac OS X Snow Leopard в роли хост-системы. Дополнительно можно упомянуть запуск проекта "Open Cloud Standards Incubator", созданного организацией Distributed Management Task Force (DMTF) с целью разработки серии открытых спецификаций, стандартизирующих методы переноса приложений и данных между cloud-окружениями, размещенными в одном или нескольких дата-центрах. Cloud-окружения являются одним из ключевых компонентов модели "Программа как сервис" (SaaS - Software as a Service), при которой программа обслуживает запросы клиента, выполняясь на внешнем сервере. Одной из первых задач проекта является продвижение единого универсального формата для оформления и распространения программ для виртуальных машин - Open Virtualization Format (OVF). Поддержка формата OVF реализована в VirtualBox 2.2 и продуктах VMware (ESXi, VMware Player, Workstation и Server), ведется работа по интеграции поддержка OVF в qemu, kvm и xen. Исходные тексты прототипа библиотеки и набора утилит для работы с OVF форматом опубликованы под лицензией Eclipse Public License. Создание Open Cloud Standards Incubator поддержали такие компании, как AMD, Cisco, Citrix, EMC, HP, IBM, Intel, Microsoft, Novell, Red Hat, Savvis, Sun Microsystems и VMware. Лидер рынка cloud-вычислений, компания Amazon, не поддержала инициативу. Ранее компании Cisco, MIT Lincoln Labs и Yahoo для разработки похожих стандартов ранее сформировали консорциум Open Cloud Consortium (OCC).
