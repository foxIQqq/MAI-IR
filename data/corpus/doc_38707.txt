Релиз дистрибутива для создания сетевых хранилищ FreeNAS 9.2

Представлен стабильный релиз FreeNAS 9.2, дистрибутива для быстрого развёртывания сетевого хранилища (NAS, Network-Attached Storage). Дистрибутив основан на кодовой базе FreeBSD, отличается интегрированной поддержкой ZFS. Управление работой FreeNAS осуществляется через веб-интерфейс, построенный с использованием Python-фреймворка Django. Для организации доступа к хранилищу поддерживается FTP, NFS, Samba, AFP, rsync и iSCSI, для повышения надежности хранения может применяться программный RAID (0,1,5), для авторизации клиентов реализована поддержка LDAP/Active Directory. Размер iso-образа составляет 163 Мб. Особенности нового выпуска: Переход на кодовую базу FreeBSD 9.2; Увеличение производительности дистрибутива в условиях высокой нагрузки. При использовании шифрования разделов ZFS для ускорения шифрования задействованы встроенные в современные CPU инструкции AESNI. Для ZFS по умолчанию задействован механизм распределения памяти UMA, который позволяет добиться большей производительности для ZFS; Добавлен REST API для удалённого управления FreeNAS; Через Avahi (multicast DNS или Bonjour) зарегистрированы все доступные сервисы, что позволяет не задумываться об IP-адресе хранилища на системах с поддержкой mDNS, в том числе при выполнении начального конфигурирования FreeNAS. По умолчанию FreeNAS-сервер для систем с поддержкой mDNS доступен по имени freenas.local (если серверов в локальной сети несколько, то freenas-N.local). Имя можно поменять в настройках; По умолчанию FreeNAS теперь рассматривает все диски как диски с 4K-секторами, что подготовит основу для будущего использования 4K-дисков в качестве запасных для прозрачной замены старых дисков с размером сектора 512 байт без потери производительности. При использовании ZFS обеспечен вывод предупреждения для пулов не выровненных по границе 4K; Прекращено использование аккаунта admin. При первом входе в интерфейс теперь запрашивается пароль для пользователя root, который используется в дальнейшем для привилегированных действий. При заведении и редактировании пользователей и групп добавлена настройка для разрешения использования Sudo; Интерфейс со списком плагинов теперь явно разделяет установленные плагины и предлагаемые для установки плагины. Для установленных плагинов выводится информация о наличии обновлений; В интерфейс настройки добавлена возможность выбора нестандартных номеров портов для доступа по HTTP и HTTPS; При выводе состояния раздела теперь выводятся данные с результатами последней проверки целостности раздела ZFS; Расширены средства для работы с изолированными окружениями. Добавлена кнопка для быстрого запуска shell для Jail-окружения. При выводе списка Jail-окружений теперь выводится информация о привязке к IP-адресам. В функции добавления хранилищ для Jail-окружений появилась возможность создания директории для монтирования и возможность монтирования в режиме только для чтения. Добавлена поддержка создания Jail-окружений на основе шаблонов. В том числе подготовлены шаблоны для Linux jail, например, Debian 7.1.0, Gentoo 20130820, Ubuntu 13.04, CentOS 6.4, Fedora 19 и openSUSE 12.3; Добавлен драйвер bxe(4) для адаптеров Broadcom NetXtreme II Ethernet 10Gb PCIe и драйвер padlock(4) для средств ускорения криптографических операций процессоров VIA C3, C7 и Eden; Поддержка USB 3.0 отключена по умолчанию, так как наблюдаются проблемы с совместимостью с некоторыми чипсетами, включая Haswell (для включения USB 3.0 следует установить значение "YES" для загрузочной опции xhci_load). Развитие проекта FreeNAS курирует компания iXsystems, развивающая операционную систему PC-BSD и способствующая улучшению поддержки ZFS во FreeBSD. Одновременно в рамках проекта OpenMediaVault развивается вариант FreeNAS, основанный на пакетной базе Debian GNU/Linux. Оба варианта развиваются параллельно, версия на базе FreeBSD ориентирована в основном на использование возможностей ZFS, в то время как версия на базе Linux разрабатывается в направлении расширенной поддержки встраиваемых устройств и гибкой системы установки дополнений. Кроме того, в рамках проекта NAS4Free ведётся разработка ответвления, продолжающего развитие монолитной архитектуры FreeNAS 7 и классического интерфейса WebGUI на языке PHP. Основателем дистрибутива FreeNAS создан проект BSD Router Project, нацеленный на предоставление инструментов для создания программных маршрутизаторов для сетей предприятий и небольших интернет-провайдеров (поддержка протоколов маршрутизации OSPFv2, OSPFv3, RIP, RIPng, PIM и BGP основана на базе пакетов Quagga, BIRD и mrouted, управление производится через CLI-интерфейс в стиле Cisco).
