Начало тестирования VirtualBox 4.0.0 и GTK-клиента для протокола SPICE

Компания Oracle выпустила первую бета-версию системы виртуализации VirtualBox 4.0, в которой представлено несколько существенных улучшений. Отдельно отмечается, что начиная с версии 4.0 некоторые из ранее доступных в базовом комплекте возможностей будут поставляться в виде внешних пакетов. В настоящий момент подготовлен только один внешний пакет PUEL, обеспечивающий поддержку USB 2.0, RDP-сервера и PXE-загрузчика с поддержкой сетевых карт E1000. Установка подобных пакетов производится из секции "Расширения" в меню глобальных настроек VirtualBox. Ключевые улучшения VirtualBox 4.0.0: Новый формат для файлов с настройками и дисковыми образами, измененный с целью повышения переносимости виртуальных машин; Поддержка формата архивов OVA (Open Virtualization Format Archive), созданного с целью стандартизации поставки образов виртуальных машин и формирования пакетов конечных приложений, предназначенных для запуска в различных виртуальных окружениях. Пакеты OVA универсальны и не привязаны к определенной архитектуре и системе виртуализации. Пакеты представляют собой обычный TAR-архив с набором дополнительной мета-информации, хранимой в подкаталоге OVF; Расширены возможности контроля за ресурсами: добавлена возможность установки ограничения на потребляемое виртуальной машиной процессорное время и на пропускную способность ввода/вывода; Обеспечена возможность изменения размера VDI и VHD образов; Обеспечена возможность выделения гостевым окружениям более 2 Гб ОЗУ на 32-разрядных хост-системах; Добавлена поддержка эмуляции для гостевых систем чипсета Intel ICH9 с тремя PCI-шинами, PCI express и системой MSI (Message Signaled Interrupts); Добавлена поддержка эмуляции для гостевых систем звуковой подсистемы Intel HD Audio; Произведен редизайн графического интерфейса, добавлен предпросмотр окон гостевых систем, реализована возможность уменьшения масштаба экрана гостевой системы; Для iSCSI, VMDK, VHD и дисковых образов Parallels добавлена поддержка асинхронного ввода/вывода; Подготовлен унифицированный набор Java-биндингов, поддерживающих как локальный API COM/XPCOM, так и удаленный API SOAP; Для Windows подготовлена функция автоматического обновления пакета паравиртуальных драйверов и дополнений Guest Additions Guest Additions добавлена поддержка копирования файлов в гостевую файловую систему. Отдельно можно отметить выпуск компанией Red Hat первой версии нового GTK-клиента Spicy для протокола SPICE. Пакет с GTK-клиентом пока собирается только в Fedora 14, так как для Fedora 13 недоступны некоторые необходимые для его функционирования пакеты. Протокол SPICE позволяет организовать удаленную работу с рабочим столом, функционирующим в виртуальном окружении под управлением QEMU/KVM. В отличие от таких протоколов как VNC (Virtual Network Computing), ICA (Citrix Independent Computing Architecture) и RDP (Microsoft Remote Desktop Protocol), в Spice рендеринг содержимого экрана и обработка аудиопотоков производится на стороне клиента, а не на сервере, что, например, позволяет без лишней нагрузки на сервер просматривать видео или осуществлять VoIP звонки, делая для пользователя выполнение приложения на удаленном сервере максимально приближенным к локальному запуску программы.
