Релиз PostGIS 2.0.0, дополнения к СУБД PostgreSQL для хранения географических данных

Представлен релиз PostGIS 2.0.0, дополнения к СУБД PostgreSQL, предназначенного для организации хранения географических данных в соответствии с требованиями стандарта "Simple Features Specification for SQL". PostGIS позволяет использовать PostgreSQL в качестве хранилища пространственных данных для географических информационных систем, выступая аналогом таких проприетарных продуктов, как ESRI SDE и Oracle Spatial. Среди добавленных улучшений: Возможность хранения растровых данных и средства для анализа растровых и векторных данных; Топологическая модель работы с объектами, имеющими общие границы; Интеграция с PostgreSQL typmod для автоматического формирования таблицы geometry_columns; Релизация 3D- и 4D-индексации; Основанная на использовании индексов высокопроизводительная система поиска ближайших соседей; Новые векторные функции: ST_Split, ST_Node, ST_MakeValid, ST_OffsetCurve, ST_ConcaveHull, ST_AsX3D, ST_GeomFromGeoJSON, ST_3DDistance; Интеграция с подсистемой разработки расширений из состава PostgreSQL 9.1; Улучшение в работе утилиты работы с дампами базы и файлов с определением очертаний объектов (shapefile); Поддержка одновременной загрузки нескольких файлов и экспорта нескольких таблиц в GUI для работы с shapefile; Geo-coder оптимизирован для работы с данными US Census TIGER (2010).
