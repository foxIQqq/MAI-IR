Выпуск VirtualBox 5.1.8

Компания Oracle выпустила корректирующий релиз системы виртуализации VirtualBox 5.1.8, в котором отмечено 24 исправления. Одновременно сформировано обновление VirtualBox 5.0.28. Кроме исправлений, направленных на увеличение стабильности и устранение регрессивных изменений, в новых версиях устранена порция уязвимостей, детальная информация о которых пока не раскрывается. Среди изменений: В компонентах для гостевых систем и хостов на базе Linux обеспечена поддержка ядер, собранных с опцией CONFIG_CPUMASK_OFFSTACK; В дополнениях для гостевых систем с Linux снято ограничение на работу всех виртуальных консолей в текстовом режиме, что позволило избавиться от проблем при загрузке систем с графической заставкой; В дополнениях для гостевых систем с Linux для depmod добавлены отдельные параметры загрузки модулей ядра vboxguest и vboxsf для обхода конфликтов с вариантами данных модулей, поставляемыми в составе дистрибутивов; Добавлена возможность сборки на платформах с Python 3; В дополнениях для X11 осуществлено отключение 3D для гостевых систем, запускаемых в хост-окружениях без необходимых графических возможностей; Решены проблемы с работой горячих клавиш в macOS и Windows; Большая порция исправлений, связанных со звуковой подсистемой. В том числе, в macOS для работы со звуком задействован API Audio Queues, а в Linux устранена проблема с записью звука при использовании бэкенда PulseAudio; Решены внесённые в 5.1.4 регрессивные изменения, проявляющиеся при удалении снапшотов на некоторых типах дисков. Также устранён крах в ситуации невозможности удалить снапшот; Решены проблемы с инициализацией контроллеров SAS; Многочисленные исправления, связанные с работой на платформе Windows, в том числе проведена адаптация для поддержки Windows 10 build 14901, решена проблема с зависанием совместных папок и улучшена поддержка групп процессоров.
