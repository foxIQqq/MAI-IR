Увидел свет Red Hat Enterprise Linux 5.9

Компания Red Hat анонсировала релиз дистрибутива Red Hat Enterprise Linux 5.9, в котором реализована поддержка нового оборудования и бэкпортированы некоторые новые возможности, при полном сохранении совместимости с прошлыми выпусками на уровне программных интерфейсов ядра и пользовательских приложений. Несмотря на параллельное развитие ветки RHEL 6, которая рекомендована для установки на новое оборудование, поддержка ветки RHEL 5 продлится до марта 2017 года. RHEL 5.9 является последним выпуском, в котором проведена работа по наращиванию функциональности, начиная с выпуска 5.10 в течение полутора лет будут выпускаться обновления с ограниченным бэкпортированием поддержки нового оборудования, исправлением серьёзных ошибок и проблем безопасности. После этого в течение трёх лет будут выпускаться только исправления уязвимостей и серьёзных ошибок. Дистрибутив доступен в трех редакциях: Red Hat Enterprise Linux 5 Server (x86, AMD64/Intel 64, Itanium, System p, System z); Red Hat Enterprise Linux 5 Advanced Platform - серверная редакция с расширенными возможностями виртуализации; Red Hat Enterprise Linux 5 Desktop (x86 и AMD64/Intel). Основные новшества Red Hat Enterprise Linux 5.9: Поддержка нового оборудования и обновление драйверов устройств; Приведение системы проверки надёжности выбираемых пользователями паролей в соответствии с новыми требованиями для правительственных учреждений, добавление в pam_cracklib поддержки опций maxclassrepeat и gecoscheck; Для корневых разделов, работающих на базе dmraid, реализован режим соответствия требованиям стандартов безопасности FIPS. FIPS-режим поддерживает определение RAID, активацию RAID, создание, удаление и перестроение томов, отображение свойств; Добавлены новые инструменты для разработчиков, в том числе полная поддержка OpenJDK 7 в дополнение к OpenJDK 6 и интеграция новой версии системы динамической трассировки SystemTap 1.8 с поддержкой IPv6 и возможностью параллельной компиляции сценариев. Добавление в ядро точек трассировки для системных вызовов sys_enter и sys_exit; В дополнение к rsyslogd 3 в состав дистрибутива включён rsyslogd 5 (пакет rsyslog5) с реализацией нового формата спула; Осуществлён переход на ветку Samba 3.6 с поддержкой SMB2 и переработанной реализацией сервера печати; Включение новой версии пакета smartmontools с поддержкой мониторинга работы контроллеров HP Smart Array и MegaRAID; Добавление команды delloem с поддержкой специфичных для оборудования Dell расширений IPMI; Включение в поставку драйверов для запуска Red Hat Enterprise Linux 5 в режиме паравиртуализированной гостевой системы под управлением системы виртуализации Microsoft Hyper-V. Интеграция в ядро системного вызова prlimit64 для гибкого ограничения ресурсов для работающих процессов, в том числе с использованием интерфейса /proc/PID/limits; Поддержка ограничения доступа к компонентам иерархии /proc/PID/; Поддержка VDSO (Virtual Dynamically-linked Shared Object) - разделяемой библиотеки, предоставляющей таблицу системных вызовов.
