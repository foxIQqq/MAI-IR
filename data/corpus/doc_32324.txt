Adobe изменил планы по открытию Flex SDK и решил передать проект фонду Apache

Компания Adobe опубликовала обновлённый план перевода Flex SDK на открытую модель управления. Вместо заявленного несколько дней назад создания новой управляющей инфраструктуры на базе организации Open Spoon Foundation решено поступить по аналогии с проектом PhoneGap и отдать компоненты Flex SDK под попечительство фонда Apache. Тем не менее, пока не ясно примет ли Apache под своё крыло новый проект, так как технологии Flex сильно привязаны к проприетарным компонентам, например, ориентированы на выполнение под управлением Flash и AIR, которые Adobe открывать не собирается. Фонд Apache достаточно щепетильно относится к подобным вопросам и требует полного открытия кода и передачи всех связанных с ним имущественных прав. Официальная заявка на передачу кода Flex SDK в инкубатор Apache, по словам Adobe, будет сформирована в течение нескольких недель. Кроме Flex SDK и связанных с ним компонентов по визуализации и автоматизации обработки данных, фонду Apache также планируется передать код следующих проектов: BlazeDS - технология удалённого обмена сообщениями в режиме реального времени между серверными компонентами и web-приложениями на базе Adobe Flex и Adobe AIR; Компоненты Spark с базовыми шаблонами различных подходов к формированию интерфейса, включая такие контейнеры с реализацией элементов интерфейса, как ViewStack, Accordion, DateField, DateChooser и DataGrid; Falcon - компилятор MXML и ActionScript. Работа над проектом ещё не завершена и код планируется открыть только в следующем году; Falcon JS - экспериментальный кросс-компилятор из MXML и ActionScript в HTML и JavaScript. Falcon JS потенциально может быть использован для автоматической трансляции Flex-приложений в представление на базе технологий HTML5/JavaScript; Инструментарий для тестирования Flex, ранее используемый для внутренних нужд компании Adobe и контроля за высоким качеством кода. Для поддержки переданных в руки Фонда Apache проектов, компания Adobe намерена выделить команду разработчиков, которые будут продолжать заниматься развитием Flex SDK в режиме полного рабочего дня. Flex SDK является открытым проектом начиная с третьей версии, но ранее открытость была ограничена возможностью отладки и анализа кода, а теперь представители сообщества смогут непосредственно принимать решения по доработке продукта, добавлять свои улучшения и без посредников исправлять ошибки. Разработка Flex SDK будет вестись в соответствии с принципами меритократии, т.е. управление будет в руках наиболее активных участников проекта. В совет, управляющий развитием проекта, планируется включить ключевых разработчиков Flex из компании Adobe, наиболее активных представителей сообщества и компаний, активно использующих Flex. Adobe считает, что несмотря на повышение интереса к HTML5 технологию Flex еще рано списывать со счетов. В секторе разработки приложений для корпоративного использования у Flex сохраняется ряд преимуществ: обеспечение целостной и полнофункциональной поддержки работы на различных платформах; компоненты Flex и модель программирования более эффективны при создании сложных интерфейсов пользователя; ActionScript является зрелым языком, подходящим для разработки больших приложений; накоплен большой багаж вспомогательных инструментов для редактирования кода, отладки и профилирования. Тем временем, компания Google выпустила расширение Swiffy для пакета Flash Professional, позволяющее не выходя из среды разработки экспортировать подготовленные SWF-файлы в представление на базе технологий HTML5. Отдельно функционирует online-сервис Swiffy, поддерживающий SWF 8 и ActionScript 2.0.
