Для Firefox OS развивается новая концепция интерфейса пользователя

На проходящей в эти дни конференции FOSDEM разработчики из проекта Mozilla продемонстрировали первую реализацию проекта Haida, в рамках которого развивается новая концепция интерфейса пользователя для платформы Firefox OS. Новый интерфейс планируется задействовать в выпуске Firefox OS 1.4, который ожидается летом этого года. Наиболее заметным изменением в Haida является переход к концепции сменяющих друг друга страниц, управляемых при помощи экранных жестов. При этом страницы будут связаны с отдельными приложениями. Например, перейти от экрана одного приложения к другому можно будет через жест перелистывания страницы (касание со сдвигом вправо или влево) без необходимости вызова менеджера задач или интерфейса для запуска приложений. Функция перелистывания экранов с режимами одной программы будет заменена на ниспадающее меню. Для централизованного поиска в адресной книге, закладках и других данных будет предложена новая панель Rocketbar. Также будет обновлено общее оформление, подготовлены новые реализации домашнего экрана, системы вывода уведомлений, системного лотка и экрана блокировки системы. Появится возможность гибкой настройки экрана блокировки системы и замены базового домашнего экрана на сторонние реализации. Кроме изменения организации взаимодействия с пользователем в Haida предусмотрены существенные внутренние изменения, в частности, браузер не будет выступать в качестве отдельного приложения, а станет неотъемлемой частью операционной системы. Приложения смогут сохранять данные при помощи нового Datastore API, позволяющего приложению предоставить доступ к своим данным для других приложений или задействовать в нескольких приложениях общее хранилище. Через Datastore API также будут доступны информационные БД, такие как список контактов в Facebook, история посещений и закладки.
