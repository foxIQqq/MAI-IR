Linux Foundation будет поддерживать UEFI Secure Boot только для систем x86

Джеймс Боттомли (James Bottomley), известный разработчик ядра Linux, входящий в координационный технический комитет Linux Foundation, пояснил позицию организации Linux Foundation в области распространения универсального решения для обеспечения поддержки работы дистрибутивов Linux на системах с активным по умолчанию режимом безопасной загрузки UEFI. Несмотря на то, что Microsoft рассчитывает привязать к себе пользователей ARM-систем, поставляемых с Windows 8, запретив изменение ключей и отключение UEFI Secure Boot на данной платформе, организация Linux Foundation ограничится распространением заверенного ключом Microsoft загрузчика только для систем x86 и не будет добиваться предоставления возможности загрузки сторонних систем на поставляемых с Windows 8 ARM-устройствах. В качестве мотива такого решения указывается, что на рынке настольных систем Microsoft является монополистом, в то время как доля продуктов данной компании на рынке устройств ARM ничтожна, по сравнению с доминирующими позициями платформ Android и iOS. При этом компания Apple и многие производители устройств на базе платформы Android изначально практикуют технику привязки своих прошивок к оборудованию, поэтому было бы не честно ополчаться против Microsoft, при отсутствии минимальных шансов переубедить Apple. В данном случае следует решать проблему более комплексно, например, добиться запрета привязки на законодательном уровне. Кроме того, в последнее время многие производители, среди которых Samsung, HTC и Sony, добровольно отказываются от тивоизации и предоставляют штатные средства для замены и модификации прошивок. Практика также показывает, что даже несмотря на привязку, энтузиасты достаточно оперативно находят способы обхода ограничений по перепрошивке устройств. В ситуации устройств на базе Windows 8 также можно гарантировать, что прошивки на базе Linux для таких устройств появятся достаточно быстро и UEFI Secure Boot не сможет помешать их установке.
