В GitHub появился новый интерфейс для приема изменений

Хостинг свободных проектов GitHub.com, базирующийся на системе управления исходными текстами Git и позволяющий разработчикам общаться формате единой социальной сети, анонсировал новый интерфейс для операций по слияний веток и синхронизации изменений (git pull). При поступлении от разработчиков заявки на внесение изменений в основную ветку, поддерживающая проект команда теперь имеет возможность обсуждения и рецензирования предлагаемых новшеств, перед их непосредственным слиянием с основной кодовой базой. Кроме того, предлагаемые для добавления изменений более не привязываются к конкретному состоянию ветки: при внесении изменений в код с момента его предложения и до момента слияния, все сторонние модификации наглядно отображаются в web-интерфейсе. В случая нахождения проблем в коде, разработчики могут сразу привязать к предлагаемому изменению тикет в системе GitHub Issues.
