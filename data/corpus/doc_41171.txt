GitHub снова оказался в реестре запрещенных в РФ сайтов

Спустя всего два месяца с момента прошлой блокировки популярный хостинг исходных текстов GitHub снова внесен в реестр запрещенных в РФ сайтов. Блокировка осуществлена на основаниях, предусмотренных статьей 15.1 Федерального закона от 27 июля 2006 года No 149-ФЗ, за размещение текстов, описывающих способы суицида (типовой абсурдный текст, который недоброжелатели пытаются распространять по сайтам, провоцируя их блокировку). В настоящее время в реестр занесены IP 192.30.252.129, 192.30.252.130 и 192.30.252.131. Так как GitHub доступен только по HTTPS, провайдеры могут применить только полную блокировку всего сайта, без возможности ограничить доступ к отдельным страницам. Дополнение 1: 3 декабря GitHub не был убран из чёрного списка. Более того, Роскомнадзор не намерен снимать блокировку: по информации от Максима Ксензова, заместителя главы Роскомнадзора, GitHub счёл требования удаления проблемных страниц необоснованными и игнорировал предписание по их удалению. В связи с этим GitHub остаётся в чёрных списках и уже недоступен в сетях многих крупных российских провайдеров, включая Билайн и Ростелеком: Дополнение 2: 5 декабря Максим Ксензов, заместитель главы Роскомнадзора, сообщил об исключении Github из реестра запрещенных в РФ сайтов после удаления вызвавшей блокировку информации из репоизиториев. Фактически, никакого удаления не произошло - указанные файлы и репозитории просто закрыли для российских IP. Так как система черных списков достаточно инертная и многие провайдеры продолжают блокировать GitHub до следующего цикла обновления списка блокировки. Например, GitHub продолжает оставаться недоступным для пользователей Ростелекома.
