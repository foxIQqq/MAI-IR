Консорциум W3C намерен отложить выпуск стандарта HTML5 до 2014 года

Несмотря на переход разрабатывающего HTML5 сообщества WHATWG на непрерывный цикл обновления спецификаций, консорциум W3C не отказался от намерения зафиксировать технологии HTML5 в виде стандарта. К сожалению из-за незавершенности стандарта объявлено о публикации в мае текущего года последнего призыва отправки заявок на расширение спецификаций, после чего три года уйдет на формирование тестового набора и учет всех замечаний. Таким образом, стандарт HTML5 удастся выпустить не раньше 2014 года. Стандарт HTML5 будет состоять из базовых спецификаций: HTML 5 - описание базового API; Список отличий HTML5 от HTML4; HTML Canvas 2D Context - в отдельный черновик выделено описание html-элмента Canvas, предназначенного для формирования векторных 2D-изображений и анимации средствами JavaScript; HTML Microdata - в отдельный черновик выделен механизм Microdata, позволяющий авторам добавить на свои страницы дополнительные семантические элементы для которых нет соответствующих HTML тэгов или атрибутов. Например, HTML не предусматривает отдельные тэги для определения списка контактов в адресной книге, номера телефона, дня рождения и т.п., но при помощи Microdata-атрибутов можно добавить данные свойства. Для определения отдельного элемента введен атрибут "item", а для перечисления свойств данного элемента - атрибут "itemprop". "HTML: The Markup Language" - первый черновик с детальным описанием HTML-разметки без привязки к API и принципов формирования HTML-документа как единого целого (синтакс, атрибуты, типы данных). "HTML+RDFa" - правила и директивы для формирования RDF и интеграции с HTML5 и XHTML5; "Polyglot Markup" - разметка для документов, которые смогут быть обработаны как парсерами XML, так и HTML; "HTML5: Techniques for providing useful text alternatives" - техника снабжения наглядных элементов страниц элементами, которые увеличат их доступность для людей с ограниченными возможностями; Дополнительно в стандарт могут быть включены спецификации, определяющие API для локальных хранилищ (IndexedDB и WebSimpleDB) и API для организации двунаправленного обмена данными (WebSockets).
