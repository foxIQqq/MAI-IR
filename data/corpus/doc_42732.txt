Релиз офисного пакета LibreOffice 5.0

Организация The Document Foundation после семи месяцев разработки представила релиз офисного пакета LibreOffice 5.0. Новый выпуск примечателен переработкой интерфейса пользователя, увеличением эффективности использования экранного пространства, улучшением совместимости с проприетарными пакетами Microsoft Office и Apple iWork, началом формирования 64-разрядных сборок для платформы Windows, поддержкой использования OpenGL для отрисовки. Это десятый выпуск проекта после ответвления от кодовой базы OpenOffice.org. Готовые установочные пакеты подготовлены для различных дистрибутивов Linux, Windows и OS X. Ключевые улучшения: Подготовка урезанных мобильных версий для платформ Android и Ubuntu Touch; Существенная модернизация оформления интерфейса Impress и Draw. Предоставление большего экранного пространства для контента за счёт автоматического скрытия контекстных панелей. Улучшена видимость кнопки для раскрытия свёрнутых боковых панелей. Переработаны инструменты выбора цвета для заливки и фонового цвета. Добавлен новый набор пиктограмм KDE Breeze. Добавлены новые пиктограммы в основной набор Tango. Изменено оформление меню и проведена реструктуризация панелей. Панели адаптированы для отображения на небольших экранах с низким разрешением. Кнопки, индикаторы, вкладки и прочие элементы интерфейса оптимизированы для комфортного просмотра на HiDPI-экранах. Произведена перегруппировка элементов в контекстных меню, выводимых для таблиц и изображений. Из контекстных меню удалены дублируемые с панелью опции форматирования; Во Writer реализована поддержка предпросмотра стилей непосредственно в боковой панели. Изменено оформление полос прокрутки и элементов перемещения между электронными таблицами в Calc; Повышено удобство настройки в экспертном режиме. Добавлена поддержка поиска параметров конфигурации. Реализовано выделение чередующихся строк разными цветами; В систему автоматической замены добавлены средства для вставки спецсимволов Unicode, в том числе значков Emoji. Для ввода используются типовые кодовые имена, обрамлённые двоеточием, например, ":keyboard:" будет заменён на символ клавиатуры (⌨), ":inch:" на символ дюйма (″), ":---:" на длинное тире и т.п. Реализован интерфейс (Format ▸ Image ▸ Crop) для удобной обрезки краёв изображений, интегрированных в таблицы и документы, с использовнием мыши; Расширена функциональность табличного процессора: поддержка новых функций (FLOOR.MATH, CEILING.MATH, ENCODEURL, ERROR.TYPE), средства для составления сложных формул, табличная адресация, поддержка ссылок вида A:A или 1:1 на целую строку или столбец, в список научных форматов добавлен ##0.00E+00 и т.п. В средствах условного форматирования реализована сплошная заливка, возможность скрытия значений, средства для изменения длины столбцов на графике, улучшен импорт условного форматирования из файлов XLSX. Улучшена работа фильтров импорта и экспорта, особенно в области поддержки форматов Microsoft Office и Apple iWork. Реализована поддержка импорта файлов в форматах Apple Pages, Apple Numbers, ClarisDraw, MacDraft, Lotus 123 (wk3, wk4) и Quattro Pro (wq1, wq2). Улучшен фильтр экспорта в MediaWiki. Значительно улучшен фильтр импорта из Freehand. Добавлены средства сохранения подсветки и затенения текста при импорте/экспорте документов в формате Microsoft Word, решены проблемы с импортом аннотаций из DOC-файлов, улучшен импорт проектов из MS Works, для OOXML реализована отрисовка с использованием DrawingML-команды arcTo, реализован импорт старого формата математических выражений, интегрированных в RTF-файлы, добавлен импорт таблиц OOXML со структурированными ссылками в формулах. В системе экспорта PDF реализована возможность использования криптографического протокола Time-Stamp для заверения факта создания PDF-документа в указанное время. Поддержка загрузки цветовых палитр в формате ASE (Adobe Swatch Exchange); Многочисленные внутренние изменения, в том числе усовершенствование графического тулкита VCL, готовность к скорому включению по умолчанию системы отрисовки через OpenGL и поддержка экспериментального бэкенда для работы поверх Wayland. В LibreOfficeKit значительно улучшены средства обособленного рендеринга документа, без отрисовки на экран. Повышено качество кодовой базы и расширена система автоматизированного тестирования для обеспечения уровня стабильности, пригодного для промышленного применения.
