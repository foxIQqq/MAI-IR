Выпуск Chrome OS 66 с поддержкой виртуальной машины для приложений Linux

Компания Google представила релиз операционной системы Chrome OS 66, основанной на ядре Linux, системном менеджере upstart, сборочном инструментарии ebuild/portage, открытых компонентах и web-браузере Chrome 66. Пользовательское окружение Chrome OS ограничивается web-браузером, а вместо стандартных программ задействованы web-приложения, тем не менее, Chrome OS включает в себя полноценный многооконный интерфейс, рабочий стол и панель задач. Сборка Chrome OS 66 доступна для большинства актуальных моделей Chromebook. Энтузиастами сформированы неофициальные сборки для обычных компьютеров с процессорами x86, x86_64 и ARM. Исходные тексты распространяются под свободной лицензией Apache 2.0. Основные изменения в Chrome OS 66: Представлен новый интерфейс управления горячими клавишами (Keyboard Shortcut Helper, Ctrl+Alt+?). Добавлена горячая клавиша (Alt+Поиск+m) для перемещения активного окна между разными экранами; Улучшены возможности по настройке подключения внешнего монитора. Добавлена настройка для корректировки размера экрана внешнего монитора и изменения уровня масштабирования контента; В приложении для работы с web-камерой (Camera) появилась поддержка записи видео; Улучшена анимация при манипуляции с окнами; Расширено число устройств для которых поддерживается режим Magic Tether (возможность быстрой организации совместного доступа к сети через рядом расположенный смартфон или планшет); Появилась возможность увеличения содержимого произвольных областей экрана в режиме "картинка в картинке" (1/3 экрана выделяется под увеличение изображения рядом с курсором). При масштабировании возможно увеличение от 2 до 20 раз; Добавлена возможность изменения уровня масштабирования в полноэкранном режиме через жест "щипок"; В ARC++ (App Runtime for Chrome, прослойка для запуска Android-приложений) для Android-приложений, установленных из Google Play, добавлена поддержка совместного доступа к экрану. Улучшены средства раскрытия окон Android-приложений на весь экран. В Google Play обеспечено соблюдение требовний GDPR (General Data Protection Regulation), добавлены настройки, связанные с персональными данными; Кнопка для установки программ из Google Play добавлена на экран первого входа в систему; Для приложений, установленных из Google Play, реализована поддержка вывода на печать; Реализована автоматическая передача параметров учётной записи Chrome OS для сетевой аутентификации через 802.1x; При первом входе добавлено уведомление о проведении синхронизации; Добавлена экспериментальная поддержка системы Crostini (chrome://flags#enable-experimental-crostini-ui), позволяющей запустить виртуальную машину с Linux. Для запуска виртуальной машины задействованы наработки проекта CrosVM, основанного на использовании гипервизора KVM. Внутри базовой виртуальной машины запускаются отдельные контейнеры с приложениями, которые планируют распространять в виде пакетов. Образы контейнеров с приложениями можно устанавливать как обычные приложения для Chrome OS и использовать между разными пользователями. Для запуска графических приложений CrosVM предоставляет встроенную поддержку Wayland-клиентов (virtio-wayland) с выполнением композитного сервера на стороне основного хоста и возможностью эффективного использования GPU из гостевых систем. Функция доступна не для всех моделей Chromebook (работоспособность подтверждена в Pixelbook). После активации Crostini в списке приложений появляется пиктограмма 'Terminal', после запуска которой будет предложена опция 'Install' для установки и настройки базового образа виртуальной машины с классическим Linux-терминалом. При установке графических приложений в виртуальной машине, они могут быть запущены по аналогии с Android-приложениями в Chrome OS (отображение Linux-программ в ланчере пока не работает). В виртуальной машине можно вручную создавать дополнительные контейнеры (используется LXC) и запускать такие приложения как Steam и Firefox.
