Релиз версии 5.39 утилит для работы с SMART информацией Smartmontools

После больше чем годовой паузы http://sourceforge.net/mailarchive/forum.php?thread_name=4B201D17.8090603%40t-online.de&forum_name=smartmontools-support вышла]] новая версия пакета smartmontools. Основные изменения, реализованные с момента выхода версии 5.38: Репозиторий исходных текстов проекта мигрировал на SVN (с CVS) Поддержка USB устройств на базе мостов Cypress, JMicron и Sunplus Автоопределение типа USB устройств на FreeBSD, Linux и Win32 платформах Поддержка контроллера Areca для Linux Поддержка адаптеров MegaRAID Поддержка контроллеров HighPoint RocketRAID для FreeBSD Поддержка команд CHECK_POWER_MODE и WRITE_LOG для FreeBSD Поддержка более 16 устройств (до 128) подключенных к 3ware контроллеру Опция smartctl '-l xerror' для вывода расширенного журнала ошибок SMART Опция smartctl '-l xselftest' для вывода расширенной информации о результатах самотестирования Опция smartctl '-l sataphy' для вывода SATA Phy счетчиков событий Опция smartctl '-l sasphy' для вывода SAS счетчиков событий Опции smartctl '-l gplog,...' и '-l smartlog,...' для вывода любых журналов smartctl опция '-x' для вывода расширенной информации (если доступна) smartctl выводит статистику load/unload для SCSI Улучшено отображение raw значений атрибутов Новая опция '-d sat+TYPE' для SAT устройств, которые требуют '-d TYPE' Опция '-v ID,RAW_FORMAT,ATTR_NAME' для добавления своих атрибутов Поддержка SSD устройств с 64-bit атрибутами Значительно обновлена база дисков Новый, упрощенный синтаксис базы дисков Опция '-B FILE' для загрузки базы дисков из файла smartd теперь корректно отрабатывает атрибуты 197 и 198 с увеличивающимся raw значением smartd отслеживает изменения статуса self test логов Директива smartd '-n powermode,N' для ограничения числа пропускаемых проверок Флаг smartd '!' для директив '-r' и '-R' позволяющий записывать любые изменения как критичные smartd поддерживает планированное выборочное самотестирование (selective self tests) Опция smartd '-s PREFIX' для хранения внутренних данных smartd при перезапусках Опция smartd '-A PREFIX' для записи атрибутов при каждой проверке Использование ООП в интерфейсах доступа к дисковой подсистеме Linux, Win32 и FreeBSD код переписан с использованием нового интерфейса Используется getaddrinfo() вместо gethostbyname() для поддержки IPv6 Дополнительная поддержка для flash дисков Samsung MLC Новый алгоритм определения типа устройства для FreeBSD, не основанный на имени файла устройства Реализована поддержка adaX устройств (ATA на CAM шине) для FreeBSD 8.0 и выше
