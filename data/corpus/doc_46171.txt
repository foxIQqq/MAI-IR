Релиз клиента для мгновенного обмена сообщениями Pidgin 2.12.0

Доступен релиз клиента для мгновенного обмена сообщениями Pidgin 2.12.0, поддерживающего работу с такими сетями как AIM, ICQ, XMPP, Google Talk, Gadu-Gadu и IRC. Графический интерфейс Pidgin написан с использованием библиотеки GTK+ и поддерживает такие возможности как единая адресная книга, одновременная работа в нескольких сетях, интерфейс на базе вкладок, работа с аватарами и интеграция с областью уведомлений Windows, GNOME и KDE. Поддержка подключения плагинов позволяет легко расширять функциональность Pidgin, а реализация базовой поддержки протоколов в отдельной библиотеке libpurple дает возможность создавать на базе технологий Pidgin собственные реализации (например, Adium для macOS). Выпуск Pidgin 2.12.0 примечателен удалением поддержки протоколов Facebook XMPP, MSN, MySpace, Mxit и Yahoo, которые остались без сопровождения: Компания Microsoft прекратила поддержку реализованного в Pidgin протокола MSNP18 и для возвращения работы с MSN требуется большая работа по созданию плагина для нового протокола. При этом, для работы c MSN можно воспользоваться неофициальным плагином SkypeWeb. Службы Mxit и MySpaceIM прекратили своё существование. Протокол Yahoo был существенно переработан, старый вариант протокола не поддерживается с августа 2016 года. В качестве замены предлагается неофициальный плагин с поддержкой нового варианта протокола Yahoo. Facebook прекратил поддержку XMPP Chat API в апреле 2015 года. Новый протокол реализован в альтернативном плагине purple-facebook. Поддержка AIM сохранена и приведена к работоспособному состоянию. Из других изменений отмечается устранение уязвимости (CVE-2017-2640), которая может привести к записи за границу буфера при получении от сервера специально оформленной порции данных в формате XML. Уязвимость присутствует в libpurple и позволяет инициировать выполнение кода на стороне клиента через отправку со стороны сервера специально оформленного сообщения.
