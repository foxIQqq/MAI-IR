Выпуск Chrome OS 85

Состоялся релиз операционной системы Chrome OS 85, основанной на ядре Linux, системном менеджере upstart, сборочном инструментарии ebuild/portage, открытых компонентах и web-браузере Chrome 85. Пользовательское окружение Chrome OS ограничивается web-браузером, а вместо стандартных программ задействованы web-приложения, тем не менее, Chrome OS включает в себя полноценный многооконный интерфейс, рабочий стол и панель задач. Сборка Chrome OS 85 доступна для большинства актуальных моделей Chromebook. Энтузиастами сформированы неофициальные сборки для обычных компьютеров с процессорами x86, x86_64 и ARM. Исходные тексты распространяются под свободной лицензией Apache 2.0. Основные изменения в Chrome OS 85: Добавлена возможность независимой настройки разрешения экрана и частоты обновления картинки для внешних мониторов. Переработана секция настройки экрана в конфигураторе. Предоставлена функция Wi-Fi Sync для синхронизации настроек беспроводной сети между несколькими устройствами. При вводе пароля к Wi-Fi он теперь запоминается в профиле пользователя и автоматически применяется при входе данного пользователя с других устройств, без необходимости повторного ручного набора пароля подключения к Wi-Fi на новом устройстве. Добавлена возможность использования строки поиска в конфигураторе для ввода запросов и определения необходимых настроек. Кроме прямых совпадения выводятся и рекомендуемые настройки, косвенно связанные с указанным запросом. В диалог быстрой настройки добавлен ползунок для изменения уровня чувствительности микрофона. В камеру добавлены дополнительные функции управления записью видео: запись теперь можно приостанавливать и возобновлять, а также сохранять фотографии во время записи видео. Видео по умолчанию записывается в более распространённом формате MP4 (H.264). В режиме голосового чтения выделенных областей (Select to Speak) появилась опция для затенения части экрана вне выделенной области. Добавлена поддержка типовых экранных жестов (удаление текста, добавление пробела и т.п.) в режиме рукописного ввода. Улучшен интерфейс вывода на печать, в котором появилась возможность управления очередью ожидающих печати документов и просмотра завершённых заданий. Для принтеров Hewlett-Packard, Ricoh и Sharp добавлена поддержка ограничения доступа к печати при помощи PIN-кода. Дополнительно можно отметить статью Эмиля Великова (Emil Velikov), отвечающего за подготовку стабильных выпусков Mesa, посвящённую устройству графического стека Linux, его использованию в Chrome OS и работе, проводимой для улучшения качества программного рендеринга. Для избавления от привязки к X11 в прослойке Ozone применяется OpenGL/GLES и EGL. В частности, в Chrome OS используется EGL-расширение EGL_MESA_platform_surfaceless, позволяющее использовать OpenGL или GLES и выполнять отрисовку в область памяти, без необходимости задействования компонентов для интеграции с экранной системой и без привлечения кода Wayland, X11 и KMS.
