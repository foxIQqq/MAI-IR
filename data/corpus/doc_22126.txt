Альтернативная реализация поддержки мультитач режима для Linux

Организация Interactive Computing Lab представила реализацию поддержки мультитач режима для Linux, которая требует для свой работы Linux ядро 2.6.30 и использует механизм трансляции стандартных событий ввода (/dev/input/eventX) в MT ("мультитач") события без задействования X сервера. Затем MT-события через D-Bus сообщения передаются в Compiz, в котором интерпретируются в виде multi-touch эффектов. В текущем состоянии система поддерживает распознавание лишь наиболее простых и типичных жестов, в которых используется одновременное касание двух точек тачпада или планшета (например, эффект вращения). Код системы уже включен в состав драйвера NTrig, который включен в тестовый git-репозиторий "linux-next", а также драйверов Stantum, DiamondTouch и Broadcom 5974, которые имеют шанс войти в экспериментальную часть следующего релиза Linux ядра. Разработка Interactive Computing Lab в настоящий момент является лишь первым демонстрационным прототипом, являющимся альтернативой технологии Multi-Pointer X, которая запланирована для включения в состав X.Org 7.5 / X Server 1.7 и позволит организовать работу нескольких независимых устройств ввода, например, несколько управляемых разными мышами курсоров на экране или ввод в разные окна с разных клавиатур.
