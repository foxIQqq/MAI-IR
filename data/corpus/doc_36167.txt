Релиз дистрибутива для создания межсетевых экранов IPFire 2.13, с поддержкой платформы ARM

Представлена новая версия дистрибутива для создания маршрутизаторов и межсетевых экранов - IPFire 2.13 Core 66. IPFire отличается предельно простым процессом установки и организацией настройки через интуитивно понятный web-интерфейс, изобилующий наглядными графиками. Размер установочного iso-образа составляет всего 93 Мб. Система модульная, кроме базовых функций пакетной фильтрации и управления трафиком для IPFire доступны модули с реализацией системы для предотвращения атак на базе Snort, для создания файлового сервера (Samba, FTP, NFS), почтового сервера (Cyrus-IMAPd, Postfix, Spamassassin, ClamAV и Openmailadmin) и сервера печати (CUPS), организации VoIP шлюза на базе Asterisk и Teamspeak, создания беспроводной точки доступа, организации потокового аудио и видео-сервера (MPFire, Videolan, Icecast, Gnump3d, VDR). Для установки дополнений в IPFire используется специальный пакетный менеджер Pakfire. Ключевым новшеством IPFire 2.13 является объявление стабильной версии дистрибутива для платформы ARM. Возможна как установка дистрибутива на встроенный накопитель, так и загрузка с USB Flash или SD-карты. Из поддерживаемых устройств отмечены Raspberry Pi, платформы на базе Marvell Kirkwood и Texas Instruments OMAP4, а также компьютеры SheevaPlug, DreamPlug и GuruPlug+, выпускаемые компанией Globalscales на базе CPU Marvell Feroceon 1.2 GHz. Размер загрузочного образа для архитектуры ARMv5tel составляет 159 Мб. Из других изменений отмечается переход на использование ядра Linux 3.2 (ранее использовалось ядро 2.6.32), Glibc 2.12.1 (ранее 2.3.6), GCC 4.4.7 и других системных компонентов. Добавлена поддержка протокола SMB2. По умолчанию задействвоан алгоритм управления очередями ожидающих отправки пакетов CoDel (Сontrolled Delay - управляемая задержка), разработанный в рамках инициативы по борьбе с негативным влиянием промежуточной буферизации пакетов (Bufferbloat) сетевым оборудованием. Задействована новая реализация IPsec, с поддержкой AES-192. Обеспечена поддержка частоты 5 GHz для беспроводных карт, соответствующих спецификациям 802.11a/n.
