Релиз системы виртуализации VirtualBox 4.1.2

Компания Oracle выпустила корректирующий релиз системы виртуализации VirtualBox 4.1.2, в котором отмечено 32 исправления. Из изменений можно отметить: Налажена работа 64-разрядных гостевых систем на процессорах AMD Fusion; Исправлена поддержка машинных инструкций sysenter/sysexit для 64-разрядных гостевых систем при их работе на хосте с процессорами Intel; В пакете Linux Additions исправлена проблема с использованием llseek() на Linux-ядрах начиная с версии 2.6.37; В GUI добавлена поддержка связанного клонирования; При экспорте через графический интерфейс исправлена проблема, приводившая к невключению дополнительной информации, такой как лицензия и название поставщика; При создании снапшотов через переключатель виртуальных машин больше не создаются live-снапшоты; Устранен крах при запуске в определенных программных окружениях, из-за конфликта версий библиотек; В пакете Guest control execution реализовано копирование отдельных файлов из гостевой системы в хост-окружение (директории и фильтры пока не поддерживаются); В утилите VBoxManage добавлена поддержка команд copyfrom и stat для копирования файлов из гостевой системы в хост-окружение и для проверки наличия файлов; В VBoxSVC в процессе регистрации теперь используется сравнение путей, независимо от типа ОС. В VBoxSVC также увеличена скорость клонирования VM; При создании снапшотов устранены проблемы с сохранением флага "Solid-State drive"; В VBoxManage добавлена возможность запуска сразу нескольких VM через команду startvm; В VBoxManage добавлена экспериментальная поддержка горячего подключения устройств SATA CD/DVD; Устранены проблемы с мышью при работе с гостевыми системами с Windows; В VRDP решены проблемы с некорректной перерисовкой экрана в некоторых ситуациях; Решены проблемы с определением USB-устройств. Устранена ошибка, приводившая к невозможности применения удаленных фильтров к локальным USB-устройствам; Решена проблема с импортом переименованных OVA-файлов; В хост-системах на базе Solaris hosts устранена ошибка, из-за которой невозможно было использовать параллельно несколько сетевых плат, для которых были настроены сетевые мосты через систему Crossbow; Для хостов на базе Linux устранены случайные крахи ядра в процессе завершения работы и перехода в спящий режим.
