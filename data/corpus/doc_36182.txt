Увидел свет дистрибутив Red Hat Enterprise Linux 6.4

Компания Red Hat представила релиз дистрибутива Red Hat Enterprise Linux 6.4, в котором проведена работа по исправлению ошибок, реализации поддержки нового оборудования и добавлению новых возможностей. Установочные образы RHEL 6.4 доступны для загрузки только зарегистрированным пользователям Red Hat Network (RHN). RHEL 6.4 поставляется в следующих вариантах: Red Hat Enterprise Linux Desktop 6 для архитектур x86 и AMD64/Intel 64 Red Hat Enterprise Linux High Performance Compute Node 6 для архитектуры AMD64/Intel 64 Red Hat Enterprise Linux Server для архитектур x86, AMD64/Intel 64, IBM System z и IBM POWER; Red Hat Enterprise Linux Workstation для архитектур x86 и AMD64/Intel 64. Ключевые новшества Red Hat Enterprise Linux 6.4: Виртуализация: Включение в поставку драйверов для запуска Red Hat Enterprise Linux 6 в режиме паравиртуализированной гостевой системы под управлением системы виртуализации Microsoft Hyper-V. Увеличение производительности гостевых окружений Red Hat Enterprise Linux 6, работающих под управлением Hyper-V; Поддержка установки паравиртуальных драйверов для VMware и Hyper-V на этапе инсталляции, что упрощает развёртывание гостевых систем с RHEL в виртуальных окружениях на базе VMware и Hyper-V; Задействование virtio-scsi c KVM, нового стека для работы с устройствами хранения, который развивается в качестве альтернативы virtio-blk. В virtio-scsi снято ограничение на одно PCI-устройство на диск, добавлена поддержка использования нескольких целевых устройств (qemu и lio), функция горячего подключения/отключения накопителей и возможность привязки к одному PCI-устройству тысяч дисков, которые внутри виртуального окружения не будут ничем отличаться от реальных SCSI-устройств; Включение в поставку модуля ядра с реализацией виртуального коммутатора Open vSwitch; Поддержка live-миграции гостевых систем с обеспечением автоматического перенаправления проброшенных в окружения USB-устройств. Обновление пакета qemu-guest-agent до версии 1.1; Системные изменения: Поддержка размещения раздела подкачки (swap) поверх NFS, что позволяет создавать большие бездисковые серверы, полностью лишённые локальных накопителей; Улучшение реализации c-groups, обеспечение поддержки корректной миграции многопоточных приложений; Обновление утилиты для мониторинга производительности perf с добавлением поддержки особенностей новых процессоров Intel; В состав дистрибутива включен балансировщик нагрузки HAProxy и сервер с реализацией протокола PTP - linuxptp; Поддержка установки c обеспечением работы сети с использованием VLAN (добавлена загрузочная опция vlanid); Система динамической трассировки SystemTap обновлена до версии 1.8; Пакеты с Samba обновлены до ветки 3.6, в которой обеспечена полноценная поддержка SMB2 и добавлен переработанный сервер печати. Добавлен пакет со свежей версией Samba4; В NSS добавлена поддержка TLS 1.1; Вместо библиотеки libjpeg задействован альтернативный более высокопроизводительный аналог - libjpeg-turbo; Добавлена поддержка задействования запасной конфигурации межсетевого экрана (/etc/sysconfig/iptables.fallback) в случае если при загрузке базового набора правил возникла ошибка; Приведение системы проверки надёжности выбираемых пользователями паролей в соответствии с новыми требованиями для правительственных учреждений, добавление в pam_cracklib поддержки опций maxclassrepeat и gecoscheck; Добавлена поддержка нового оборудования, в том числе новых моделей планшетов Wacom. Обновлены драйверы для сетевых устройств, систем хранения. Системы хранения: Добавлена возможность ведения системного лога, отражающего информацию о маппинге логических блочных устройств и физических идентификаторов устройств, что позволяет администратору легко выявить нужный физический диск; Обеспечена полноценная поддержка pNFS (Parallel NFS). Улучшена производительность клиента pNFS, добавлена поддержка прямого ввода/вывода (Direct I/O), позволяющая обеспечить более быстрый доступ к данным в конфигурациях с высокой интенсивностью ввода/вывода, например, при работе СУБД; В файловую систему XFS добавлена поддержка высвобождения неиспользуемых блоков на лету (online discard), без отмонтирования раздела; В LVM добавлена поддержка RAID10. Управление идентификацией: в демоне SSSD (System Security Services Daemon) улучшена совместимость с Microsoft Active Directory, что позволило обеспечить возможность централизованного управления доступом для Linux/Unix клиентов, работающих в гетерогенных окружениях; Пользовательские компоненты: В почтовом клиенте Evolution улучшена совместимость с Microsoft Exchange, улучшена работа таких функций календаря-планировщика, как уведомление о наступлении событий и планирования встреч;
