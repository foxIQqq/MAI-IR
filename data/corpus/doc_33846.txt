Началась подготовка Mandriva 2012

На wiki-проекта Mandriva Linux отмечается активность, связанная с началом подготовки нового релиза Mandriva 2012. Официально о планах компании будет известно только после публикации доклада с изложением новой стратегии развития, который планировалось опубликовать в середине мая. Из появившейся информации опубликован начальный план подготовки релиза, в соответствии с которым первый тестовый выпуск будет выпущен 31 мая, а первая альфа версия в течение июня. Дата финального релиза пока не определена. Из намеченных для Mandriva 2012 новшеств можно выделить: Переход на использование системной библиотеки eglibc; Использование для пересборки пакетов gcc 4.7.x, интеграция в GCC патчей от проекта Linaro для улучшения поддержки архитектуры ARM; Возвращение загрузчика Aurora или портироваение возможностей Aurora в plymouth; Задействование в качестве клиента для синхронизации точного времени Chrony NTP, который отличается низким потреблением памяти (около 1 Мб), отсутствием активации фонового процесса без необходимости и соответственно более высокой экономией энергии. Кроме того, Chrony NTP обладает лучшими характеристиками по поддержанию точного времени для систем без постоянного соединения с Интернетом, не работающих 24 часа в сутки или содержащих низкокачественные/нестабильные часы (например, в виртуальных машинах); Включение в поставку поддержки msec (Multi-Security Tool) и msec-gui, интерфейса для управления настройками, связанными с безопасностью. Msec также может использоваться для формирования отчетов на основе выполненения периодических проверок, например для контроля за правами доступа к файлам; Одновременная поддержка urpmi и smart для управления пакетами; Поддержка в инсталляторе файловых систем btrfs, jfs и xfs; Улучшение поддержки настройки соединения к беспроводным сетям в drakx-net; Включение в репозиторий пакетов с пользовательским окружением Razor-Qt; Создание инструмента для восстановления системы в случае проблем на этапе загрузки (например, при невозможности примонтировать ФС); Интеграция в дистрибутив пакета Bumblebee с поддержкой технологии Nvidia Optimus; Поддержка x32 ABI (гибридного x86_64 Linux ABI с 32-битной адресацией памяти); Задействование по умолчанию технологию prelink, которая позволит ускорить загрузку программ, связанных с большим количеством библиотек; Добавление обособленного GUI-интерфейса для решения проблем с загрузкой X-сервера (предлагается написать такой интерфейс на Qt и запускать поверх framebuffer). Пока точно не сообщается о будущем взаимодействии с компанией ROSA в процессе подготовки релиза Mandriva 2012, скорее всего как при подготовке Mandriva Linux 2011 и ROSA Desktop 2011 новые версии данных дистрибутивов будут развиваться синхронно, объединённой командой разработчиков и на основе одной пакетной базы.
