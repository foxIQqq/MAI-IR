Опубликован план расширения функциональности дистрибутива Mandriva 2011

Представлен обзор новых возможностей, запланированных для включения в релиз Linux-дистрибутива Mandriva 2011, выход которого ожидается 30 мая. Напомню, что разработчики Mandriva приняли решение перейти на годичный цикл подготовки релизов, что позволит выпускать более значительные по числу улучшений выпуски, даст возможность сосредоточить больше внимания на тестировании, проверке и улучшению качества. В новой версии ожидается: Переход на пакетный менеджер RPM5, развиваемый независимо от компании Red Hat; Изначально для повышения интерактивности выполнения десктоп-ориентированных задач планировалось использовать планировщик BFS, но потом разработчики склонились к интеграции CONFIG_SCHED_AUTOGROUP-патча с реализацией идеи автоматической группировки задач; В качестве системы инициализации планируется использовать системный менеджер systemd. Systemd нацелен на более интенсивную параллелизацию выполнения сервисов на этапе загрузки системы, вобрав в себя лучшие черты таких систем, как launchd (Mac OS X), SMF (Solaris) и Upstart (Ubuntu, Fedora). Кроме запуска сервисов, systemd обеспечивает функции контроля за выполнением фоновых процессов. Подробнее о systemd можно прочитать в ранее опубликованных обзорах (здесь, здесь, здесь и здесь). Для настройки параметров сети будет использован NetworkManager с включенным по умолчанию плагином ifcfg-mdv. Конфигуратор net_applet можно будет использовать в качестве альтернативного и совместимого с прошлыми выпусками решения; Будет разработано новое высокоуровневое приложение для управления пакетами, отличающееся более современным и простым в использовании GUI-интерфейсом; По умолчанию будет задействовано только одно десктоп-окружение (KDE), стандартизирующее базовый набор приложений и визуальное оформление дистрибутива; Подготовка при участии сообщества отдельных сборок дистрибутива с десктоп-окружениями на базе GNOME, XFCE и LXDE ; Упрощение работы инсталлятора: поддержка быстрой установки, при которой число выполняемых в инсталляторе шагов сведено к минимуму. В упрощенном режиме выбор десктоп-окружения и типа ядра будет выполняться автоматически, на основе наименования ISO-образа (для mandriva-2011.iso будет по умолчанию ставиться KDE, для mandriva-gnome-2011.iso - GNOME, для mandriva-xfce-2011.iso - xfce. Для продвинутых пользователей останется возможность изменения принятых по умолчанию настроек; Обеспечение унифицированного оформления и поведения для диалогов всех базовых приложений, например, диалогов записи и чтения файлов, вывода на печать; Новый пользовательский интерфейс для входа в систему и для десктоп-окружения; Новый пользовательский интерфейс для нетбуков и машин с пониженным потреблением энергии; Новая графическая тема для пользовательского интерфейса и набора виджетов; Интеграция семантического поиска (работа с файлами на основе мета-данных, а не не только через файловый путь) в почтовый клиент, файловый менеджер, адресную книгу, графический и текстовый редакторы; Интеграция оригинальных конфигурационных инструментов Mandriva в центр управления KDE; Оптимизация потребления ресурсов для десктопа Mandriva; Чистка основного репозитория пакетов (Main) и закрепление майнтейнеров для всех пакетов в основном репозитории; Обновление зависимостей базовой системы и интеграция пакета busybox для обеспечения возможности создания минималистичных установок, занимающих минимум дискового пространства; Чистка и переработка списка всегда устанавливаемых пакетов: mandi, shorewall, acpi и т.п. Улучшение совместимости с zeroconf-приложениями; Обеспечение параллельной разработки стабильного релиза и ветки "Сooker" с непрерывным циклом обновления версий программ. Вместо ранее применявшейся практики заморозки "Сooker" перед релизом, для стабилизации нового выпуска будет создаваться отдельное ответвление, а "Сooker" продолжит обновление без заморозки.
