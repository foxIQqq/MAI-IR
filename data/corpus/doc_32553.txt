Релиз Novell Open Enterprise Server 11

Анонсирован релиз сетевой операционной системы Novell Open Enterprise Server 11 (OES), продолжающей развитие системы NetWare. Продукт базируется на SUSE Linux Enterprise Server 11 SP1 и предлагает на базе Linux-сервера полную поддержку всех сервисов NetWare. Напомним, что серия OES была запущена Novell в 2005 году, чтобы помочь безболезненно мигрировать многочисленным пользователям NetWare на современную ОС. Данный релиз интересен тем, что после большого перерыва в выпусках, это первый релиз после поглощения компании Novell со стороны Attachmate, после чего Novell и SUSE стали отдельными и независимыми подразделениями, в связи с чем некоторые эксперты сомневались в продолжении выпуска OES. Глэн Дэвис (Glen Davis), менеджер OES в Novell, заявил, что уже скоро последует обновление этого релиза до SUSE Linux Enterprise Server 11 SP2, а в будущем планируется синхронизировать выход OES с обновлениями SUSE. В последней версии стали доступны следующие новшества и улучшения: Реализована "бесшовная" кроссплатформенная поддержка клиентов через собственную реализацию протокола AFP (Apple Filing Protocol), которая позволяет пользователям Mac OS X чувствовать себя на сервере OES как в родной среде. Кроме интеграции с Novell eDirectory, служба Novell AFP поддерживает согласованную блокировку файлов при работе c протоколами AFP, CIFS и NCP, начальную поддержку протокола Bonjour, поддержку имён файлов с Unicode-символами, а также пароли, длина которых может превышать 8 символов; Централизованное управление сервером – инновационная система серверных обновлений, позволяющая управлять всеми обновлениями в системе из одного места, существенно экономящая время и силы администратора; Dynamic Storage Technology – система управления и связывания пользовательских данных. Она создаёт два независимых раздела на диске (называемых "tiers") для всех пользовательских данных. Система сохраняет все последние данные с которыми работал пользователь на основном разделе, обычно размещаемом на высокопроизводительном хранилище, например SAN. Затем, согласно установленным администратором политикам, менее активно используемые данные, которые давно не читались или к которым редко обращаются, прозрачно перемещаются на второй связанный с пользователем раздел, который размещается, как правило, на менее дорогих и быстрых носителях. Несмотря на то, что эти два раздела физически два совершенно разных устройства, с точки зрения пользователя это один единственный общий диск; Встроенная аутентификация через Microsoft Active Directory – поддержка бизнес-приложений на базе Microsoft Active Directory для их аутентификации напрямую из Novell eDirectory.
