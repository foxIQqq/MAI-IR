Релиз дистрибутива openSUSE Leap 15.2

После более года разработки состоялся релиз дистрибутива openSUSE Leap 15.2. Выпуск сформирован с использованием базового набора пакетов из находящегося в разработке дистрибутива SUSE Linux Enterprise 15 SP2, поверх которого поставляются более новые выпуски пользовательских приложений из репозитория openSUSE Tumbleweed. Для загрузки доступна универсальная DVD-сборка, размером 4 ГБ, урезанный образ для установки с загрузкой пакетов по сети (138 Мб) и Live-сборки с KDE (910 МБ) и GNOME (820 МБ). Выпуск сформирован для архитектур x86_64, ARM (aarch64, armv7) и POWER (ppc64le). Основные новшества: Обновлены компоненты дистрибутива. Как и в SUSE Linux Enterprise 15 SP2 продолжает поставляться базовое ядро Linux, подготовленное на основе версии 5.3.18 (в прошлом выпуске использовалось ядро 4.12). Ядро аналогично используемому в дистрибутиве SUSE Linux Enterprise 15 Service Pack 2 и сопровождается компанией SUSE. Из изменений отмечается поддержка GPU AMD Navi и обеспечение совместимости с технологией Intel Speed Select, используемой в серверах на базе CPU Intel Xeon. Предоставлен вариант ядра с Real-Time патчами для систем реального времени. Как и в двух предыдущих выпусках поставляется systemd версии 234. Помимо GCC 7 (Leap 15.0) и GCC 8 (Leap 15.1) добавлены пакеты с набором компиляторов GCC 9. В дистрибутиве также предложены новые выпуски PHP 7.4.6, Python 3.6.10, Perl 5.26, Clang 9, Ruby 2.5, CUPS 2.2.7, DNF 4.2.19. Из пользовательских приложений обновлены Xfce 4.14 (в прошлом выпуске был 4.12), GNOME 3.34 (был 3.26), KDE Plasma 5.18 (был 5.12), LXQt 0.14.1, Cinnamon 4.4, Sway 1.4, LibreOffice 6.4, Qt 5.12, Mesa 19.3, X.org Server 1.20.3, Wayland 1.18, VLC 3.0.7, GNU Health 3.6.4, OnionShare 2.2, Syncthing 1.3.4. Как и в прошлом выпуске для настройки сети настольных систем и ноутбуков по умолчанию предложен Network Manager. В серверных сборках по умолчанию продолжено использование Wicked. Для генерации сертификатов Let’s Encrypt применяется скрипт dehydrated. Обновлена утилита Snapper, отвечающая за создание снапшотов Btrfs и LVM со срезами состояния файловой системы и откат изменений (например, можно вернуть случайно перезаписанный файл или восстановить состояние системы после установки пакетов). В Snapper реализована возможность вывода в новом формате, оптимизированном для машинного разбора и упрощающего использование в скриптах. Переработан плагин для libzypp, который избавлен от привязки к языку Python и может использоваться в окружениях с сокращённым набором пакетов. В инсталляторе реализован боле простой диалог для выбора системной роли. Улучшен вывод информации о ходе установки. Улучшено управление устройствами хранения при установке на платы Raspberry Pi. Обеспечено более точное определение разделов Windows, зашифрованных при помощи BitLocker. В конфигураторе YaST реализовано разделение системных настроек между каталогами /usr/etc и /etc. Улучшена совместимость YaST Firstboot с подсистемой WSL (Windows Subsystem for Linux) в Windows. Переработан модуль настройки сети. Повышено удобство интерфейса разбивки дисковых разделов и добавлена возможность создания и управления разделами Btrfs, охватывающими несколько накопителей. Повышена производительность интерфейса установки приложений Software Manager. Расширена функциональность модуля NFS. В систему автоматизированной массовой установки AutoYaST добавлены дополнительные настройки и улучшено информирование о возможных ошибках в установочных профилях. Предоставлена возможность обновления серверных установок openSUSE Leap до SUSE Linux Enterprise, что позволяет вести разработку проекта на openSUSE, а после готовности мигрировать на SLE при необходимости получения коммерческой поддержки, сертификации и расширенного цикла доставки обновлений. В репозиторий включены пакеты с фреймворками и приложениями, связанными с машинным обучением. В том числе для быстрой установки теперь доступны Tensorflow и PyTorch, обеспечена поддержка формата ONNX для распространения моделей машинного обучения. Добавлены пакеты Grafana и Prometheus, позволяющие организовать наглядный мониторинг и анализ изменения метрик на графиках. Предоставлены официально поддерживаемые пакеты для развёртывания инфраструктуры контейнерной изоляции на базе платформы Kubernetes. Добавлен пакетный менеджер Helm для установки компонентов Kubernetes. Добавлены пакеты с runtime CRI-O (легковесная альтернатива Docker), соответствующим спецификации Container Runtime Interface (CRI) от Open Container Initiative (OCI). Для организации безопасного сетевого взаимодействия между контейнерами добавлен пакет с сетевой подсистемой Cilium. Обеспечена поддержка системных ролей Server и Transactional Server. В Server применяется традиционный набор пакетов для создания минимального серверного окружения, а в Transactional Server предложена конфигурация для серверных систем, использующая механизм транзакционных обновлений и монтируемый в режиме только для чтения корневой раздел.
