Релиз дистрибутива для создания межсетевых экранов IPFire 2.19 Core 104

Доступен выпуск дистрибутива для создания маршрутизаторов и межсетевых экранов - IPFire 2.19 Core 104. IPFire отличается предельно простым процессом установки и организацией настройки через интуитивно понятный web-интерфейс, изобилующий наглядными графиками. Размер установочного iso-образа составляет 160 Мб (x86_64, i586, ARM). Система модульная, кроме базовых функций пакетной фильтрации и управления трафиком для IPFire доступны модули с реализацией системы для предотвращения атак на базе Snort, для создания файлового сервера (Samba, FTP, NFS), почтового сервера (Cyrus-IMAPd, Postfix, Spamassassin, ClamAV и Openmailadmin) и сервера печати (CUPS), организации VoIP шлюза на базе Asterisk и Teamspeak, создания беспроводной точки доступа, организации потокового аудио и видео-сервера (MPFire, Videolan, Icecast, Gnump3d, VDR). Для установки дополнений в IPFire используется специальный пакетный менеджер Pakfire. Новый выпуск примечателен добавлением переписанной системы предотвращения вторжений Guardian, работающей в связке со Snort и осуществляющей автоматическое обновление правил пакетного фильтра на основании предупреждений, генерируемых Snort. Guardian позволяет использовать IPFire в качестве динамического межсетевого экрана, автоматически блокирующего хосты с которых зафиксирована нетипичная сетевая активность. Из других улучшений отмечается добавление в Dynamic DNS Updater поддержки сервисов DuckDNS и spdyn, а также обновление версий программ: Snort 2.9.8.2, OpenSSH 7.3p1, shadow 4.2.1, libarchive 3.2.1, libcap 2.25, acl 2.2.52, iputils s20160308, curl 7.49.1, popt 1.16, pcre 8.39, acpid 2.0.26, which 2.21, libtiff 4.0.6, ntp 4.2.8p8, wget 1.18. Добавлены пакеты с libvirt и freeradius.
