Релиз видеодрайвера AMD Catalyst 9.11. Создан Gallium3D драйвер для VMware

Компания AMD выпустила новую версию проприетарного видеодрайвера для видеокарт ATI - AMD Catalyst 9.11. Это первый релиз после ухода из компании Matthew Tippett, руководителя подразделения Linux-разработок и основного разработчика Linux версии драйверов. В AMD Catalyst 9.11 представлены следующие улучшения: Добавлена поддержка Linux дистрибутивов RHEL 5.4 и openSUSE 11.2; Добавлена поддержка видеокарт Radeon HD 5750 и Radeon HD 5770; Исправлены ошибки: Неприятное мерцание курсора мыши в многомониторном Clone-режиме, проявляющееся в Ubuntu 9.04; Повреждение картинки на экране после подключения дисплея к работающей системе и переключения на виртуальный терминал; Зависание при включенном режиме CrossFire и работе полноэкранного приложения в момент перевода ноутбука с сетевого питания на работу от аккумулятора; Нарушение отображения на экране в SUSE 11.1 при отключении второго монитора; Нарушение отображения при проигрывания Flash-ролика в полноэкранном режиме на втором мониторе; Повреждение картинки при включении базирующегося на OpenGL хранителя экрана в сочетании с включенными десктоп-эффектами. Неисправленными остаются 5 известных проблем, среди которых: Крах при загрузке X-ов, если до этого использовался многомониторный режим; Зависание при вызове некоторых функций Xrandr; Проблемы с установкой нужного разрешения экрана в Catalyst Control Center; Прекращение отображения при передергивании питания у монитора. Также можно отметить появление в git-репозитории проекта Mesa 3D нового Gallium3D драйвера VMware SVGA, предназначенного для использования внутри гостевых Linux окружений VMware. Новый драйвер содержит приблизительно 30 тыс. строк кода, позволяет ускорять выполнение графических операций и получить прямой доступ к функциям X11, OpenGL, OpenCL, X-Video и XvMC управляющей системы, на которой установлен X Server 1.7.x и модуль ядра vmwgfx. По утверждению VMware Gallium3D стек уже готов к промышленному использованию, но драйвер ядра и код для DRI/X.Org еще находится на стадии бета-тестирования. Окончательная стабилизация кода ожидается к моменту выхода Mesa 7.7. По оптимистичным прогнозам драйвер vmwgfx может войти в состав основной ветки Linux ядра 2.6.33. Gallium3D является реализацией новой архитектуры для разработки 3D драйверов, представляет собой попытку адаптировать OpenGL реализацию Mesa 3D к современным аппаратным архитектурам, создав единый промежуточный интерфейс для упрощенной разработки драйверов. В Gallium3D, для оптимизации работы, используется два модуля ядра - DRM2 (Direct Rendering Manager) для управления памятью, и DRI (Direct Rendering Interface) для взаимодействия с GPU.
