Представлен wlterm, эмулятор терминала для Wayland

Представлен wlterm, созданный c нуля эмулятор терминала, реализованный в роли клиента дисплейного сервера Wayland. Проект развивается в одном репозитории с kmscon, эмулятором терминала, использующим возможности KMS-модулей (Kernel Mode Setting) ядра Linux для создания работающего в пространстве пользователя аналога поставляемой в ядре системы виртуальных терминалов. Wlterm использует единую с kmscon подсистему TSM (Terminal-emulator State Machine) для обработки управляющих последовательностей vt220/xterm. Wlterm не зависит от кода демонстрационных приложений композитного сервера Weston и обеспечивает декорирование окон собственными силами, что может быть использовано для выявления ошибок в клиентском API Wayland, которые были пропущены в процессе тестирования стандартных демонстрационных приложений. Из обязательных зависимостей присутствуют только libxkbcommon и libwayland-client/cursor.
