Релиз системы для организации мониторинга и централизованного управления RHQ 4.2

Представлен релиз RHQ 4.2, открытой системы централизованного управления, конфигурирования, инвентаризации и мониторинга за работой программного обеспечения на машинах в сети предприятия. Код RHQ написан на языке Java (для работы требуется Java 6, функционирование под управлением Java 7 пока полностью не протестировано) и распространяется в рамках лицензии GPL. Платформа построена по модульному принципу и позволяет организовать управление практически всеми аспектами работы операционной системы и пользовательских приложений. Например, RHQ позволяет контролировать работу различных дистрибутивов Linux и ОС Windows, web-серверов Apache, контейнеров Apache Tomcat, сервера приложений JBoss, СУБД MySQL, PostgreSQL и других популярных открытых проектов. Служба отправки уведомлений также базируется на плагинах и расширяется в зависимости от нужд администратора, например, можно организовать запуск произвольных скриптов при возникновении событий и легко создать модуль для интеграции RHQ с корпоративными системами обработки жалоб клиентов. В состав также входит набор инструментов для упрощения развёртывания программных пакетов на группе обслуживаемых компьютеров. Управление может производиться как через web-интерфейс, так и через специальное клиентское приложение, работающее в режиме командной строки. Новшества, добавленные в RHQ 4.2 Поддержка PostgreSQL 9.1 в качестве СУБД для хранения данных RHQ (ранее поддерживались только версии 8.x и 9.0); Начальная поддержка нового REST API для доступа к данным и функциональности сервера из внешних приложений. Например, данный API можно использовать для создания расширений, написанных не на языке Java; Улучшено выполнение операций синхронизации конфигураций rhq-сервера (например, поддерживается наложение части конфигурации одного сервера для других серверов), упрощены настройки для создания реплицированного сервера; Улучшена работа плагина для отправки уведомлений через Twitter; Проведена работа по улучшению юзабилити web-интерфейса; Завершена реализация drift-системы, позволяющей определять изменения файлов и отправлять соответствующие уведомления.
