Вышел PelicanHPC 2.0, Linux дистрибутив для развертывания кластеров

Вышел PelicanHPC 2.0, Linux дистрибутив для быстрого развертывания кластеров. PelicanHPC построен на пакетной базе Debian и позволяет превратить обычный офисный компьютерный парк в вычислительный кластер, загрузив один управляющий компьютер с LiveCD/LiveUSB, а остальные машины использовав в качестве загружаемых через PXE узлов. Графический интерфейс управляющей станции построен на основе Xfce4. Из особенностей новой версии отмечено: Переход на использование Debian Squeeze (testing) вместо Debian Lenny (stable), что позволило включить в состав дистрибутива новейшие версии кластерных приложений, например, пакет Open MPI 1.3.3. Также можно отметить, что в дистрибутиве используется Linux ядро 2.6.30. Добавление новых MPI биндингов для GNU Octave, которые позволяют использовать Octave 3.2.x вместо версий 3.0.x, которые значительно отстают по производительности; Из реализаций интерфейса MPI отныне в состав дистрибутива входит только Open MPI, ранее присутствовавший LAM-MPI удален из поставки; Интерфейс мониторинга работы кластера Ganglia установлен и преднастроен по умолчанию для 4 узлов и легко расширяется для более крупных кластеров (достаточно открыть http://localhost/ganglia после инициализации кластера через pelican_setup). ПО для мониторинга ksysguard также присутствует в составе дистрибутива; Обновлена версия входящего в комплект теста производительности кластеров Linpack.
