Выход проприетарного драйвера NVIDIA 325.15 и библиотеки libvdpau 0.7

Компания NVIDIA опубликовала первый стабильный выпуск новой серии проприетарного драйвера NVIDIA 325.15. Драйвер доступен для Linux, FreeBSD и Solaris. В новой версии добавлена поддержка GPU GeForce GT 625, GRID K340, GRID K350, NVS 315 и Quadro K500M, а также прекращена поддержка ядер Linux серии 2.4. Для работы драйвера теперь требуется ядро Linux 2.6.9 или более новая версия. В прошлых LTS-сериях 319.x, 173.14.xx и 304.xx поддержка ядер 2.4.x будет сохранена. Из других изменений, не связанных с исправлением ошибок, можно отметить добавление в nvidia-settings настройки поворота экрана для каждого дисплея в отдельности. В разделе "X Server Display Configuration" конфигуратора nvidia-settings добавлена поддержка настройки режимов SLI Mosaic и Base Mosaic. Увеличена производительность сложных градиентов. Одновременно представлен релиз libvdpau 0.7, открытой библиотеки с поддержкой API VDPAU (Video Decode and Presentation) для Unix. Библиотека позволяет обеспечить работу реализации VDPAU API на базе сторонних технологий аппаратного ускорения, например, через использование Intel VA-API. VDPAU API предназначен для задействования возможностей GPU по аппаратному ускорению обработки различных форматов видео, например, на плечи GPU могут быть вынесены функции пост-процессинга, слияния (compositing), отображения и декодирования. В новой версии устранена утечка файловых дескрипторов и добавлена поддержка переменной окружения DRI_PRIME для выбора корректного устройства DRI2 в конфигурациях с несколькими видеокартами. Также опубликован новый плагин к libvdpau, позволяющий организовать работу VDPAU API без проприетарного драйвера NVIDIA на системах, поддерживающих OpenGL и VAAPI. Например для аппаратного ускорения видео в Adobe Flash Player на видеокартах AMD и Intel.
