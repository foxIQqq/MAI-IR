Компания Oracle выпустила ядро Unbreakable Enterprise Kernel R5U4

Компания Oracle выпустила четвёртое функциональное обновление для ядра Unbreakable Enterprise Kernel R5, позиционируемого для использования в дистрибутиве Oracle Linux в качестве альтернативы штатному пакету с ядром из Red Hat Enterprise Linux. Ядро доступно для архитектур x86_64 и ARM64 (aarch64). Исходные тексты ядра, включая разбивку на отдельные патчи, опубликованы в публичном Git-репозитории Oracle. Пакет Unbreakable Enterprise Kernel 5 основан на ядре Linux 4.14 (UEK R4 базировался на ядре 4.1, а UEK R6 на 5.4), которое дополнено новыми возможностями, оптимизациями и исправлениями, а также проверено на совместимость с большинством приложений, работающих в RHEL, и специально оптимизировано для работы с промышленным программным обеспечением и оборудованием Oracle. Установочные и src-пакеты с ядром UEK R5U4 подготовлены для Oracle Linux 7 (нет никаких препятствий по использованию данного ядра в аналогичных версиях RHEL, CentOS и Scientific Linux). Ключевые улучшения: Добавлена поддержка резервирования диапазонов виртуального адресного пространства процесса (Process Virtual Address Space Reservation), позволяющего повысить стабильность СУБД Oracle при включенной рандомизации раскладки адресного пространства (ASLR). Из свежих выпусков основного ядра перенесены исправления и оптимизации для NFS, связанные с доступом к страничному кэшу, обработкой вызовов RPC и поддержкой клиентов NFSv4. Решены проблемы с работой NFS поверх OCSF2. Расширены средства диагностики TCP-стека, добавлена поддержка точек трассировки на базе eBPF и снижены накладные расходы при трассировке. Из ядра 5.6 перенесены новые патчи для защиты от уязвимостей класса Spectre v1. Обновлены драйверы устройств, в том числе задействованы новые версии драйверов для BCM573xx (bnxt_en), Intel Ethernet Switch Host Interface (fm10k), Intel Ethernet Connection XL710 (i40e), Broadcom MegaRAID SAS (megaraid_sas), LSI MPT Fusion SAS 3.0 (mpt3sas), QLogic Fibre Channel HBA (qla2xxx), Microsemi Smart Family Controller (smartpqi), Intel Volume Management Device (vmd) и VMware Virtual Machine Communication Interface (vmw_vmci).
