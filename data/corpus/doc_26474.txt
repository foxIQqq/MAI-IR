Релиз Spacewalk 1.0, свободной платформы для управления Linux-инфраструктурой

Вышел релиз свободной платформы для управления инфраструктурой Linux-серверов - Spacewalk 1.0, построенной на одной кодовой базе с продуктом Red Hat Network Satellite. Исходные тексты Spacewalk распространяются в рамках лицензии GPLv2, основная масса кода написана на языке Python. Управление инфраструктурой производится через web-интерфейс. Серверная и клиентская часть системы поддерживает RHEL 5, Fedora 11/12 и производные от них дистрибутивы, например, CentOS и Scientific Linux. Базовые возможности Spacewalk: Организация инвентаризации оборудования и программного обеспечения; Автоматизация установки и обновления ПО; Распространение собственных пакетов и приложений по группам машин; Автоматизация установки типовых конфигураций системы в режиме kickstart; Централизованное управление конфигурационными файлами; Выполнение функции мониторинга; Управление работой виртуальных машин, выполнение операций по запуску, остановке и конфигурированию гостевых систем; Эффективное распространение контента по нескольким географически распределенным сайтам. Из главных новшеств выпуска Spacewalk 1.0 можно отметить: Улучшение кода клиента для работы с сервисом Red Hat Network (RHN), который теперь базируется на python-ugdev и больше не поддерживает HAL; Добавлен скрипт для переименования имен хостов при реконфигурации сервера Spacewalk со сменой IP или имени хоста. Расширено число дополнительных API-вызовов (ApiAdditions); RHN Proxy теперь использует mod_wsgi; Добавлена поддержка контейнера сервлетов/JSP Tomcat 6 для Fedora 12; Добавлен скрипт для удаления систем; Проведена большая работа по чистке кода и устранению ошибок для соответствия качеству релиза 1.0. К сожалению разработчикам не удалось довести до приемлемого состояния поддержку использования PostgreSQL в качестве базовой СУБД, поэтому пользователи по прежнему вынуждены использовать СУБД Oracle или мириться с проблемами при использовании экспериментальной поддержки PostgreSQL.
