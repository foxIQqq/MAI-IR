Сервис Google Code расширил выбор открытых лицензий

Кристофер ДиБона (Chris DiBona), управляющий Open Source проектами в компании Google, анонсировал изменение политики в отношении выбора лицензий для проектов, размещаемых в хостинг-сервисе Google Code. Отныне сервис допускает выбор любой открытой лицензии, одобренной организацией Open Source Initiative (OSI). Кроме того, проведена работа для предоставления проектам возможности распространения сразу под несколькими открытыми лицензиями (двойное лицензирование). Некоторые ограничения оставлены для проектов, переведенных в разряд общественного достояния. Как показывает практика, законодательство некоторых стран не допускает отречения от авторских прав, кроме того часто авторы не до конца осознают суть перевода проектов в разряд общественного достояния и подразумевают нечто иное, пользуясь отсутствием четкой формулировки. Хостинг Google Code допускает поддержку подобных проектов только после специального согласования. Из недопустимых проектов, которые подлежат удалению из сервиса, называются любые закрытые и условно-открытые разработки, а также попытки использования Google Code для хостинга файлов, не имеющих отношения к коду. По словам Кристофера ДиБона, руководителям проекта Google Code на самом деле никогда не нравилось отвергать проекты, которые были под свободными, но не допускавшимися Google лицензиями (как, например, лицензия zlib) или под лицензиями, созданными с особыми, совершенно новыми целями (как, например, AGPL). Также новая политика позволит устранить разногласия, связанные с допуском проектов, состоящих из кода с разными лицензиями (например, если проект под Apache License содержит компоненты под лицензией zlib). Ранее сервис Google Code придерживался компромиссного варианта, при котором допускалась регистрация проектов, распространяемых в рамках базового набора популярных открытых лицензий: Artistic license, GPL v2 и v3, LGPL, Apache 2.0, MIT, MPL, EPL и BSD. Этим сервис отличался с одной стороны от Sourceforge, допускающего регистрацию проектов под любой одобренной OSI лицензией, а с другой от Savannah, принимающего только лицензированные под GPL проекты.
