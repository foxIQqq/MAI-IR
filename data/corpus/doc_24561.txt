Результаты анализа предпочтения пользователей графической подсистемы Linux

Опубликован отчет, в котором отражены предпочтения пользователей и тенденции, связанные с графической подсистемой Linux. Отчет составлен на основании опроса около 14 тыс. пользователей ресурса phoronix.com. Итоги (в скобках указаны данные прошлогоднего опроса, но показатели отличаются из-за возможности множественного выбора в текущем опросе): Наиболее интересные технологии: 44% - заинтересованы в системе переключения видеорежимов на уровне ядра (Kernel-based Mode-Setting); 67% - отдали предпочтение улучшениям, связанным с акселерацией декодирования видео (XvMC, VA-API, XvBA, VDPAU); 43% - за качественный рендеринг, без артефактов и мерцания; 39% - считают важными вопросы энергосбережения; 29% - Gallium 3D. Популярность видеокарт: 33% (21.88%) - Intel; 43.4% (32.24%) - ATI; 43% (41.11%) - NVIDIA; 1.7% - VIA; 0.7% - SiS / XGI. Используемые видеодрайверы: 31% (18.16%) - xf86-video-intel; 22.5% (11.69%) - xf86-video-ati; 8.5% (6.04%) - xf86-video-radeonhd; 20% (17.84%) - ATI Catalyst; 50% (41.56%) - NVIDIA; 3.5% - Nouveau; 2.6% - xf86-video-nv; 1.5% - VESA. Источник получения видеодрайверов: 23.8% (23.81%) пользователей загрузили видеодрайвер с сайта производителя карты, 77.5% (61.52%) - скачали готовый пакет для своего дистрибутива; 6% собрали драйвер из исходных текстов, загрузив код из Git репозитория; Используемые версии X.Org: 7.4% (27.51%) - 7.3; 54.5% (53.99%) - 7.4; 31% - 7.5; 1.1% - Git снапшот X.Org. Задействование дополнительных функций: 17% (19.96%) - TV-Out; 40% (38.55%) - подключено несколько мониторов; 22% (19.56%) - горячее подключение устройств вывода информации; 25% (21.91%) -энергосберегающий режим работы GPU (PowerPLAY, DynamicClocks). Метод настройки X.Org: 47% (40.38%) - пользователей вручную изменяли содержимое xorg.conf, 53% (31.63%) - воспользовались GUI конфигуратором, 25% (14.60%) - положились на XRandR, 15% (11.49%) - воспользовались консольными программами из состава драйверов (nvidia-xconfig и aticonfig); Использование нескольких дисплеев: 50% - используется один дисплей; 19% - используется несколько экранов, для настройки которых используется XRandr; 8% - используется несколько экранов, для настройки которых используется Xinerama; 12% - используется несколько экранов, для настройки которых используется NVIDIA TwinView или ATI BigDesktop. Используемое разрешение экрана: 44% - начиная с 1280x1024, но ниже 1600x1200; 38% - начиная с 1600x1200, но ниже 2560x1600; 20% - менее 1280x1024; 5% - 2560x1600 и выше. Композитный режим: 22% (15.08%) - KWin; 37% (43.04%) - Compiz; 36.8% (36.61%) - не пользуются композитным оконным менеджером. Наиболее важные области, требующие особого внимания: 70% - стабильность; 73% (17.28%) - производительность 3D графики; 37% (15.09%) - акселерация просмотра видео; 43% (11.50%) - лицензирование и открытие исходных текстов; 55% (10.84%)- производительность 2D графики; 40% (10.31%) - качество вывода изображений; 39% (9.66%) - поддержка режима временного засыпания (Suspend / Hibernate); 31% (8.09%) - упрощение процесса установки и настройки; 18% (4.81%) - возможности, связанные с управлением дисплеем; 8% (2.33%) - задействование нескольких GPU для реднеринга данных. Области применения графики: 43% (22.54%) - игры; 27% (13.45%) - визуализация данных; 84% (37.90%) - просмотр видео; 52% (23.11%) - визуальные эффекты на десктопе; 11% - вычисления с задействованием GPU.
