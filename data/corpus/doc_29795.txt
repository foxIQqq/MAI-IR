Релиз Jabber-сервера Openfire 3.7.0

Вышел релиз многоплатформенного XMPP-сервера Openfire 3.7.0. Сервер поддерживает разработку дополнений в виде плагинов, снабжен web-интерфейсом для администраторов и пользователей, поддерживает шифрование SSL/TLS, поддерживает взаимодействие с LDAP, дает возможность подключения реализации дополнительных транспортных модулей, например, для организации шлюза с ICQ. Начиная с данного выпуска код проекта, написанный на языке Java, перелицензирован и вместо GPLv2 будет распространяться под лицензией Apache 2.0. Среди причин изменения лицензии называются проблемы с использованием Openfire в некоторых компаниях, из-за особенностей корпоративной политики, и надежда, что переход на более лояльную для коммерческого использования лицензию положительно скажется на участии в развитии проекта коммерческих компаний. Из изменений в Openfire 3.7.0 можно отметить: Улучшена обработка неактивных соединений (idle), таймаут для которых можно установить через переменную xmpp.client.idle (-1 - старый метод); Переработана система кэширования, минимизирующая объем необходимой для кэша памяти; Встроенный интерфейс для ведения логов заменен на бэкендны SLF4J и Log4J; Устранена утечка памяти в реализации Personal Eventing Protocol (PEP); Добавлено несколько незначительных улучшений и исправлений в блок конфигурации Multi-User Chat (MUC); Улучшена работа некоторых плагинов, добавлен новый плагин Jingle Nodes; Обновлены файлы с переводами элементов интерфейса на русский язык.
