Началось альфа-тестирование дистрибутива Fedora Linux 15

Вышла первая альфа-версия дистрибутива Fedora Linux 15. Для загрузки доступны Live-сборки с GNOME 3 (567 Мб) и KDE 4.6 (667 Мб). Релиз запланирован на 10 мая. Основные улучшения: В состав дистрибутива включены последние версии десктоп-окружений Gnome 3.0 (с оболочкой Gnome Shell), KDE 4.6 и Xfce 4.8; Интеграция системного менеджера systemd в качестве системы инициализации по умолчанию. Systemd нацелен на более интенсивную параллелизацию выполнения сервисов на этапе загрузки системы, вобрав в себя лучшие черты таких систем, как launchd (Mac OS X), SMF (Solaris) и Upstart (Ubuntu, Fedora). Кроме запуска сервисов, systemd обеспечивает функции контроля за выполнением фоновых процессов. Подробнее о systemd можно прочитать в ранее опубликованных обзорах (здесь, здесь, здесь, здесь и здесь); Изменение наименования сетевых интерфейсов. Вместо ethN имена устройств будут формироваться в зависимости от типа подключения сетевого адаптера: "em{port}" - для встроенных в материнскую плату сетевых карт и "pci{slot}#{port}" - для карт с PCI-интерфейсом. Переход на GCC 4.6 для сборки пакетов. Интеграция в дистрибутив Python 3.2 и новых версий Rails и OCaml; Использование вместо OpenOffice.org офисного пакета LibreOffice; Интеграция BoxGrinder, инструмента для упрощения формирования сборок для развертывания внутри различных систем виртуализации (KVM, Xen, EC2); Добавление фонового процесса для динамического управления правилами пакетного фильтра через DBus; Улучшение поддержки шифрованной файловой системы eCryptfs. Использование authconfig для автоматического монтирования зашифрованных частей домашней директории в момент входа пользователя в систему; Использование по умолчанию DNSSEC в конфигурациях рабочих станций. Для преобразования имен в NetworkManager использует DNS-сервер BIND; Улучшение работы системы управления питанием, поддержка автоматического тюнинга параметров работы системы, с целью обеспечения оптимального баланса между производительностью и энергопотреблением. Интеграция утилиты PowerTOP для выявление узких мест, приводящих к повышенному энергопотреблению; Добавление в дистрибутив ERP-платформы Tryton, предназначенной для разработки приложений для управления ресурсами предприятия (финансы, управление продажами и закупками, аналитика, инвентаризация и т.п.); Новые группы пакетов: Graphics suite переименован в Design group, добавлена новая группа Robotics Package Suite, в которую вошли инструменты для проектирования роботов, симуляции их работы и создания программного обеспечения; Задействование метода сжатия LZMA для формирования Live-образов; C целью повышения безопасности все setuid-приложения будут переведены на использование "capabilities"; Разделы /var/run и /var/lock смонтированы в виде RAM-диска (tmpfs); Добавление LessFS, FUSE-модуля для автоматического объединения дубликатов в файлах; Обновление пакетного менеджера RPM до версии 4.9; Поддержка технологии Spice в virt-manager; GUI-интерфейс Setroubleshoot переработан с целью упрощения диагностики проблем с SELinux. Дополнение: 10 марта состоится день тестирования GNOME3 в Fedora Linux. Цель данного мероприятия протестировать как можно больше функциональных возможностей GNOME 3, убедиться в их корректной работе, а также в том, что GNOME 3 обеспечивает необходимый для ежедневного использования основной функционал среды рабочего стола.
