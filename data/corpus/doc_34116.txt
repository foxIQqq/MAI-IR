Первый тестовый выпуск Mandriva 2012

Представлен первый тестовый выпуск Mandriva 2012, имеющий статус "Technology Preview". Для загрузки подготовлены iso-образы DVD- и CD-сборок, размером 4.1 Гб и 649 Мб. Представленная сборка позиционируется как начальная отправная точка для наращивания функциональности, созданная на базе текущего среза репозитория Cooker. Дата релиза пока не определена. Среди намеченных для Mandriva 2012 новшеств: Переход на использование системной библиотеки eglibc; Использование для пересборки пакетов gcc 4.7.x, интеграция в GCC патчей от проекта Linaro для улучшения поддержки архитектуры ARM; Возвращение загрузчика Aurora или портироваение возможностей Aurora в plymouth; Задействование в качестве клиента для синхронизации точного времени Chrony NTP, который отличается низким потреблением памяти (около 1 Мб), отсутствием активации фонового процесса без необходимости и соответственно более высокой экономией энергии. Кроме того, Chrony NTP обладает лучшими характеристиками по поддержанию точного времени для систем без постоянного соединения с Интернетом, не работающих 24 часа в сутки или содержащих низкокачественные/нестабильные часы (например, в виртуальных машинах); Включение в поставку поддержки msec (Multi-Security Tool) и msec-gui, интерфейса для управления настройками, связанными с безопасностью. Msec также может использоваться для формирования отчетов на основе выполненения периодических проверок, например для контроля за правами доступа к файлам; Одновременная поддержка urpmi и smart для управления пакетами; Поддержка в инсталляторе файловых систем btrfs, jfs и xfs; Улучшение поддержки настройки соединения к беспроводным сетям в drakx-net; Включение в репозиторий пакетов с пользовательским окружением Razor-Qt; Создание инструмента для восстановления системы в случае проблем на этапе загрузки (например, при невозможности примонтировать ФС); Интеграция в дистрибутив пакета Bumblebee с поддержкой технологии Nvidia Optimus; Поддержка x32 ABI (гибридного x86_64 Linux ABI с 32-битной адресацией памяти); Задействование по умолчанию технологию prelink, которая позволит ускорить загрузку программ, связанных с большим количеством библиотек; Добавление обособленного GUI-интерфейса для решения проблем с загрузкой X-сервера (предлагается написать такой интерфейс на Qt и запускать поверх framebuffer).
