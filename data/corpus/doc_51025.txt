Amazon опубликовал Open Distro for Elasticsearch 1.0.0

Компания Amazon представила первый выпуск продукта Open Distro for Elasticsearch, в рамках которого подготовлена полностью открытая редакция платформы поиска, анализа и хранения данных Elasticsearch. Опубликованная редакция пригодна для применения на предприятиях и включает расширенные возможности, доступные только в коммерческой версии оригинального Elasticsearch. Все компоненты проекта распространяются под лицензией Apache 2.0. Готовые сборки подготовлены в форматах DEB и RPM, а также в виде образов Docker и набора отдельных плагинов. Выпуск примечателен синхронизацией компонентов платформы с ветками Elasticsearch 7.0 и Kibana UI 7.0, и обеспечением полной совместимости с ними. В том числе доступны такие возможности, как поддержка SQL, генерация уведомлений, механизмы диагностики производительности кластера и дополнительные средства для обеспечения безопасности ( аутентификация через Active Directory, Kerberos, SAML и OpenID, реализация единой точки входа (SSO), поддержка шифрования трафика, система разграничения доступа на основе ролей (RBAC), ведение детального лога для аудита). Основные изменения по сравнению предварительными выпусками: В модуль для отслеживания событий и генерации предупреждений, позволяющий осуществлять мониторинг за состоянием данных и автоматически отправлять уведомления при срабатывании определённых проверок, добавлена возможность регулирования интенсивности отправки уведомлений через пользовательский интерфейс Kibana. Также добавлены визуальные индикаторы, позволяющие использовать SQL-запросы в качестве критериев для мониторинга; В средствах обеспечения безопасности реализована поддержка нового синтаксиса конфигурации и возможность задания настроек в формате YAML. Обновлена версия модуля для аутентификации в LDAP/Active Directory, в котором появилась поддержка запроса нескольких баз ролей и реализован более функциональный пул соединений; В модуль для использования SQL добавлены дополнительные coverage-тесты и реализована поддержка выборочного включения и выключения тех или иных возможностей SQL. В драйвер SQL JDBC добавлена поддержка преобразования полей с типом datetime с учётом часовых поясов; В состав включён модуль Job Scheduler, предоставляющий SPI-интерфейс для выполнения другими плагинами периодических работ. Планирование выполнения работ может производиться как через задание интервала периодичности их вызова, так и в стиле Сron. Поддерживается выставление блокировок для защиты от наложения выполнения длительно работающих работ.
