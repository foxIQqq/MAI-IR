Началось альфа-тестирование Fedora 22

Доступна для тестирования альфа-версия дистрибутива Fedora 22. Альфа-выпуск содержит все базовые возможности будущего релиза, пригодные для оценки и тестирования. Релиз запланирован на 19 мая. Выпуск охватывает продукты Fedora Workstation, Fedora Server, Fedora Cloud, а также Live-сборки, поставляемые в форме спинов c десктоп-окружениями KDE, Xfce, MATE и LXDE. Наиболее заметные изменения в Fedora 22: Экран входа в систему переведён по умолчанию на использование Wayland. Экран входа является достаточно обособленной системой, поэтому переведя его на Wayland в первую очередь, у разработчиков появится возможность окончательно убедиться в отсутствии подводных камней и незамеченных ошибок, без негативного влияния на работу приложений и рабочего стола. Пользовательское окружение GNOME по умолчанию по-прежнему будет работать с использованием X-сервера, при том, что полноценный сеанс на базе Wayland предоставляется в качестве штатной опции. Переход на Wayland по умолчанию запланирован на Fedora 23; Для управления устройствами ввода в X11 и Wayland задействована библиотека libinput; В дистрибутиве задействован предварительный выпуск рабочего стола GNOME 3.16, релиз которого запланирован на 25 марта. Из улучшений можно отметить новую тему оформления GNOME Shell, переработанную систему вывода уведомлений, стабилизированную и доработанную поддержку Wayland, модернизированный интерфейс файлового менеджера Nautilus, поддержку Google Drive; Включены наработки по снижению энергопотребления. Добавлен инструмент для тестирования автономной работы от аккумулятора; В интерфейс для управления установкой приложений GNOME Software добавлена поддержка работы со сторонними компонентами, такими как мультимедийные кодеки для GStreamer и шрифты; Усовершенствованы темы оформления для Qt4 и Qt5, позволяющие обеспечить бесшовную интеграцию с GNOME Qt-приложений и программ из KDE, интерфейс которых будет унифицирован с приложениями GTK. Усовершенствован GNOME Terminal, в том числе обеспечена возможность вывода уведомлений при завершении длительно работающих заданий, таких как компиляция, что позволит не отвлекаться, постоянно заглядывая в терминал для оценки статуса завершения работы; В инструменте для автоматической отправки сообщений о проблемах ABRT (Automatic Bug Reporting Tool) улучшена организация вывода уведомлений и предоставлены более полные средства обеспечения соблюдения приватности и контроля за отправляемой вовне информации; В спинах представлены свежие версии KDE Plasma 5.2 и Xfce 4.12; В инструментарий Rolekit добавлена поддержка развёртывания и управления серверных "ролей" на базе СУБД PostgreSQL. Под ролью подразумевается определённая серверная функциональность, работу которой обеспечивает набор пакетов (например, почтовый сервер может потребовать установки пакетов с SMTP и IMAP серверами). Rolekit позволяет разом установить и настроить все пакеты, требуемые для реализации сервиса. Обновлён Cockpit, web-интерфейс для настройки и мониторинга одного или нескольких серверов. Позволяет администратору управлять хранилищами, запуском сервисов, анализировать логи, инспектировать и администрировать контейнеры Docker, в том числе отслеживать потребление ресурсов и задавать ограничения. В редакцию для облачных систем включены последние версии пакетов rpm-ostree и rpm-ostree-toolbox, при помощи которых можно сгенерировать собственную начинку базового окружения Atomic, содержащую свой набор пакетов. Напомним, что в рамках проекта Atomic развивается концепция минималистичной ОС для обеспечения запуска изолированных контейнеров, подготовленных и управляемых при помощи инструментария Docker. Предоставляемое Atomic базовое окружение (Atomic Host) содержит только минимальный набор компонентов (systemd, journald, docker, geard и т.п.), необходимых для запуска и управления изолированными контейнерами. Для обновления системного окружения Atomic используется технология OSTree, при которой системный образ атомарно обновляется из Git-подобного хранилища, позволяющего применять методы версионного контроля к компонентам дистрибутива (например, можно быстро откатить систему к прошлому состоянию). RPM-пакеты транслируются в репозиторий OSTree при помощи специальной прослойки rpm-ostree. Сформирован образы Fedora Atomic Host и Fedora Cloud для использования с инструментарием Vagrant, который может использоваться для быстрого развёртывания виртуальных окружений в системах виртуализации на базе KVM и VirtualBox; Представлен Tunir, новый легковесный инструмент непрерывной интеграции, нацеленный на ускорение тестирования образов для облачных систем; Ожидавшийся в Fedora 22 переход по умолчанию на Python 3, а также связанная с этим замена пакетного менеджера Yum на DNF, откладываются до выпуска Fedora 23 из-за нерешённых проблем с портированием инсталлятора Anaconda.
