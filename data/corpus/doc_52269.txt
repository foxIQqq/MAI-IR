Выпуск дистрибутива для создания сетевых хранилищ FreeNAS 11.3

Компания iXsystems представила релиз FreeNAS 11.3, дистрибутива для быстрого развёртывания сетевого хранилища (NAS, Network-Attached Storage). Дистрибутив основан на кодовой базе FreeBSD, отличается интегрированной поддержкой ZFS и возможностью управления через веб-интерфейс, построенный с использованием Python-фреймворка Django. Для организации доступа к хранилищу поддерживается FTP, NFS, Samba, AFP, rsync и iSCSI, для повышения надежности хранения может применяться программный RAID (0,1,5), для авторизации клиентов реализована поддержка LDAP/Active Directory. Установочный iso-образ (780 Мб) подготовлен для архитектуры x86_64. Основные изменения: Переработан движок репликации данных в ZFS. Производительность репликации увеличена в 8 раз. Добавлена поддержка автоматического возобновления прерванных сеансов передачи данных, параллельного выполнения задач и локальной репликации; Добавлен ACL Manager - web-интерфейс для настройки и управления списками контроля доступа в SMB-разделах; Для новых SMB-разделов активирован по умолчанию модуль SMB Shadow Copy, создающий резервные копии при помощи системы снапшотов ZFS. Созданные копии файлов можно посмотреть во вкладке "Previous Versions" в файловом менеджере; Добавлена поддержка использования квот для SMB, определённых в Active Directory; Введён в строй репозиторий "Community plugins" с плагинами, подготовленными представителями сообщества и официально не поддерживаемыми iXsystems; Добавлен iSCSI Wizard, упрощающий создание новых iSCSI target; Переработан интерфейс мониторинга, в котором обеспечена группировка предупреждений по типу, а не алфавиту. Добавлена возможность установки пороговых значений для генерации предупреждений. Реализован новый тип постоянных критических предупреждений, остающихся активными до ручного закрытия. Для периодического запуска обработчиков задействован новый фреймворк Alert; Переписан интерфейс Dashboard, который теперь предлагает сводный отчёт о текущем состоянии системы, сетевой активности, нагрузки на CPU и потреблении памяти; Предоставлена возможность использования транслятора адресов для запуска плагинов, без необходимости присвоения каждому плагину своего IP-адреса (каждый плагин запускается в отдельном изолированном окружении); Предложен новый REST API, поддерживающий Websocket и использующий единые с web-интерфейсом обработчики. API можно использовать для управления FreeNAS из скриптов и интеграции с внешними платформами. Добавлен API для сохранения и аудита файлов конфигурации; Добавлен мастер настройки больших пулов ZFS, охватывающих большое число дисков. Предложенный интерфейс позволяет автоматизировать клонирование раскладки VDEV на все диски; Проведена оптимизация производительности ZFS для разных видов нагрузки; Предложены упрощённые мастера настройки для SMB, параметров сети и репликации; Реализована поддержка VPN WireGuard; Обновлены переводы на чешский, французский, японский, русский и упрощенный китайский. Кроме того, процесс добавления дополнительных переводов был значительно улучшен.
