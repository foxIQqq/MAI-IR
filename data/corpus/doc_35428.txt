Новая версия программы ДЭНСИ:КАССА 1.2.7

Доступна для загрузки новая версия ДЭНСИ:КАССА 1.2.7, программного комплекса для автоматизации рабочего места кассира-операциониста (ПО класса «Front-office»). Программа поддерживает широкий спектр торгового оборудования и работает под управлением различных дистрибутивов Linux. Все графические пользовательские интерфейсы и фоновые сервисы распространяются под лицензией GPLv2. Сервер торгового оборудования распространяется под проприетарной лицензией, но поставляется с открытым API и может быть использован беcплатно при условии печати рекламы продукта на кассовых чеках. В новой версии: Добавлена функция автоматического восстановления текущего чека при входе в режим регистрации после сбоев; Добавлена операция продажи/возврата по свободной цене (в журнале продаж отражается транзакцией с типом 11, как и обычная продажа/возврат, но без указания кода/штрих-кода товара); Новая комбинация клавиш: [Cntrl]+[W]; Добавлена настройка права продажи/возврата по свободной цене в интерфейсе настройки прав пользователей; * Добавлена возможность работы с менеджерами-консультантами на позицию; Для ФР добавлена настройка «Не использовать принудительный отрез чека»; В профили пользователей добавлена настройка права на открытие окна визуального подбора товаров. В соответствии с этим, расширен протокол обмена «ДЭНСИ»; Добавлена настройка максимальной суммы чека и способа реакции на её превышение (блокировка или предупреждение); В окно комбинированных оплат добавлено сочетание клавиш [Cntrl]+[Enter] – закрытие чека и его печать без печати позиций; - добавлена возможность оставлять комментарий (в том числе с помощью сканера штрих-кода) на каждую позицию чека и на весь чек; Новая комбинация клавиш: [Cntrl]+[K] (для комментария на чек – после нажатия [ПРОМИТОГ](Insert)) Новое свойство в тэге DENCI_CHECKTABLE: «comment» – заменяется на комментарий к позиции; (Позволяет, например, заполнять серийные номера для гарантийного талона с помощью сканера штрих-кода); Новый тэг DENCI_COMMENT – заменяется на комментарий к чеку; В очет по товарам добавлена печать внутренних кодов товаров; Добавлен тэг DENCI_NUMPOS для использования во внешних html-шаблонах документов. Данный тэг заменяется на целочисленное значение кол-ва позиций в чеке за вычетом сторнированных позиций; Добавлено свойство «printname» в тэге DENCI_CHECKTABLE – для вывода на печать в документах, формируемых по внешним html-шаблонам, полного наименования товаров (наименования для печати); Добавлен тэг DENCI_DISCOUNTTYPE, который заменяется на строку «Скидка» или «Надбавка», в зависимости от итоговой суммы скидок/надбавок на чек; При этом тэг DENCI_DISCOUNTSUM теперь заменяется модуль суммы всех скидок/надбавок на чек; добавлено свойство «sum_with_dscnt» в тэг DENCI_CHECKTABLE – заменяется на сумму текущей позиции с учетом скидок/надбавок на позицию (ранее так работало свойство «result_sum»; Свойство «result_sum» в тэге DENCI_CHECKTABLE изменено – теперь оно заменяется на итоговую сумму текущей позиции с учетом всех скидок/надбавок на позицию и чек и правил их комбинирования; Добавлено свойство «res_dscnt_percent» в тэг DENCI_CHECKTABLE – заменяется на итоговый реальный процент скидки/надбавки, примененный к данной позиции в результате комбинирования всех скидок/надбавок на позицию/чек по установленным в настройках программы правилам их комбинирования; Добавлена поддержка печати отчета сверки итогов в INPAS SmartSale; Добавлена функция автоматического отреза банковских слипов в INPAS SmartSale; Восстановлена возможность начислять/отменять нулевые фиксированые скидки на чек (кроме автоматических и произвольных); Увеличена скорость работы визуального поиска; Увеличена скорость загрузки окна фиксированных скидок из режима регистрации продаж; Реализована работа с платежной системой СБЕРБАНКа. Утилиты СБЕРБАНКА для работы с ПИН-падами работают только на 32-битной версии ОС Linux и устанавливаются в каталог /usr/local/bin/SB/. Для доступа к приёму банковских платежей в ДЭНСИ: КАССе необходим специальный ключ активации, который можно запросить в отделе продаж.
