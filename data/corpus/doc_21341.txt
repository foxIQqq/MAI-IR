Обновление FreeNAS 0.69.1, дистрибутива для создания сетевых хранилищ

Вышел новый стабильный релиз дистрибутива FreeNAS 0.69.1, основанного на FreeBSD 6.4 и предназначенного для быстрого развёртывания сетевого хранилища (NAS, Network-Attached Storage) и сопутствующих сетевых сервисов (SMB/CIFS, FTP, NFS, rsync и т.п.). В выпуске 0.69.1 присутствуют следующие улучшения: Обновлены версии программ: Samba 3.0.34; ProFTPD 1.3.2; mDNSResponder 1.08.6; lighttpd 1.4.22; cdialog 1.1.20080819; e2fsprogs 1.41.4; nut 2.4.1 (для контроля доступа задействован TCP wrapper, вместо встроенного механизма ACL); BitTorrent клиент transmission 1.51; ntfs-3g/fusefs-ntfs 2009.2.1; Обновлен комплект драйверов для SATA RAID контроллеров 3Ware - 9.5.1; bash 4.0.10. Улучшения: В интерфейс для настройки FTP ('Services|FTP') добавлены пункты 'SSL/TLS only' (работа только по TLS/SSL) и 'Reverse DNS lookup' (выполнять обратное преобразование IP, по умолчанию отключено); В интерфейс для настройки BitTorrent ('Services|BitTorrent') добавлен пункт 'Authentication' для включения или выключения аутентификации в TransmissionBT WebGUI ; Возможность включения режима Wake-On-LAN для сетевых драйверов nfe, nve, sis, vr, xl (пункт WOL в 'Network|Interfaces|LAN') В интерфейс управления бесперебойным источником питания добавлена возможность удаленного мониторинга ('Remote monitoring' в 'Services|UPS'); Добавлена возможность монтирования UDF ISO образов; В интерфейс добавлен блок 'System|Advanced|sysctl.conf' для установки sysctl переменных; Добавлена возможность управления через XMLRPC сервисы, доступные как http://xxx.xxx.xxx.xxx/xmlrpc.php; Возможность настройки ProFTPD модуля mod_ban через WebGUI; Поддержка просмотра статистики ввода/вывода через 'Status|Disks'; В состав включена команда traceroute6, утилита msntp заменена на ntpdate. Исправлено 12 ошибок. Известные ограничения: Невозможно создать программный RAID с ФС FAT16/32; Невозможно создать отдельный шифрованный раздел, шифровать можно только диск целиком; Невозможно просмотреть состояние отдельного CPU в привязке к процессам на SMP системах; Включение режима 'polling' не работает на сетевых интерфейсах, используемых в LAGG интерфейсе (объединение линков); Не поддерживается монтирование EXT2 дисков с размером inode 256 байт, работа возможна только с 128 байтовым форматом. Уже больше года параллельно с веткой 0.6x в разработке находится версия FreeNAS 0.7, основанная на FreeBSD 7 и поддерживающая систему журналирования UFS2 gjournal, файловую систему ZFS и GEOM-класс gvistor для создания виртуальных блочных устройств (предоставляет возможность создать блочное устройство размером больше связанного с ним физического накопителя, позволяет по мере заполнения подключать новые устройства, заменять или удалять уже подключенные). В будущем выпуске также будет представлена новая систему управления дисками, обладающая возможностью выполнения таких операций, как монтирование через WebGUI.
