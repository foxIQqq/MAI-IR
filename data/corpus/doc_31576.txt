Релиз сетевого конфигуратора NetworkManager 0.9

Доступен релиз интерфейса для упрощения настройки параметров сети - NetworkManager 0.9, а также апплета с реализацией индикатора для панели GNOME. Наиболее важным изменением называется значительное упрощение модели конфигурирования, полезной как для конечных пользователей, так и для разработчиков приложений. Пользователи получили возможность более полного и удобного управления сетевыми параметрами, в конфигурационном апплете добавлена поддержка GNOME Shell и новой системы индикаторов Ubuntu. Обеспечена полная поддержка использования NetworkManager в многопользовательских системах - реализована поддержка быстрого переключения между пользовательскими сеансами и добавлена система разграничения доступа. В каждом сеансе пользователя теперь доступен отдельный управляющий апплет, через который можно осуществлять изменений, на которые пользователь имеет полномочия. Например, какому-то пользователю можно запретить выход в сеть 3G-модем, если в момент переключение сеанса для этого пользователя сеть будет активна, то соединение будет разорвано. Улучшена поддержка руминга WiFi, при котором пользователь части использует большое число различных точек доступа (на работе, дома, в гостинице и т.п.). Добавлена возможность использования добавленной в wpa_supplicant 0.7 поддержки nl80211 и фонового сканирования беспроводных сетей. Более точно выводится информация об уровне сигнала и решены проблемы с разрывами соединений. Добавлена поддержка настройки соединений WiMAX. Файлы конфигурации унифицированы и перемещены в единое место, ранее они были размещены разрозненно и в разных форматах. Для разработчиков подготовлен новый гибкий API для создания сетевых апплетов. Представлен переработанный упрощенный D-Bus API, позволяющий облегчить создание пользовательских приложений, которым необходим доступ к параметрам конфигурации и требуется возможность оценки состояния сети. API теперь предоставляет единые методы для типичных клиентских операций, таких как создание нового соединения для обнаруженных точек доступа и фильтрация существующих соединений, привязанных к заданным устройствам. Прекращена поддержка интерфейса User Settings Service: сервисы пользовательских и системных настроек теперь реализованы в NetworkManager через единый механизм. Консолидированы методы для управления доступом.
