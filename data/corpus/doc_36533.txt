Релиз дистрибутива Scientific Linux 6.4

Увидел свет релиз дистрибутива Scientific Linux 6.4, построенного на пакетной базе Red Hat Enterprise Linux 6.4 и дополненного средствами, ориентированными на использование в научных учреждениях. Дистрибутив поставляется для архитектур i386 и x86_64, для загрузки доступно несколько сборок: установочный DVD (3.4 Гб), полный комплект из двух DVD (4.2 Гб + 420 Мб) и сокращенный образ для установки по сети (159 Мб). Live-версия Scientific Linux 6.4 будет подготовлена позднее. Из отличий Scientific Linux 6.4 от версии 6.3, не связанных с заимствованием изменений из RHEL, можно отметить: Изменён метод формирования пакетов с модулем ядра для поддержки клиента OpenAFS, открытой реализации распределенной ФС Andrew File System. Основное различие заключается в методе сборки: если раньше один модуль ядра собирался сразу для ядер всех выпусков Scientific Linux 6.x, то сейчас для каждого выпуска формируется свой отдельный вариант модуля. Осуществлена установка по умолчанию пакета yum-conf-sl6x, который содержит настройки для обновления Yum-репозитория по мере выхода новых релизов Scientific Linux 6.x, что позволяет автоматически переходить на свежую версию Scientific Linux 6.x сразу после её выхода, без дополнительных ручных манипуляций для инициирования обновления дистрибутива; В состав пакета yum-conf-sl-other добавлена запись для поддержки репозитория 'sl-addons', в котором размещены пакеты, претендующие на включение в следующие выпуски, пакеты с неопределённым будущим или которые невозможно добавить в репозиторий main; Из-за устранения уязвимости в пакете xorg-x11-server изменился ABI. Так как изменение связано с безопасностью, обновление пакета с изменённым ABI будет представлено и для прошлых выпусков Scientific Linux 6.x; Прекращена поддержка устаревшего Matahari API, используемого для организации управления системой. Общие отличия Scientific Linux 6.x от RHEL 6.x: Оконный менеджер IceWM 1.2.37; OpenAFS - открытая реализация распределенной ФС Andrew File System; Набор развиваемых в рамках проекта Fedora инструментов для создания Live-дисков и производных сборок. Для создания собственного Live-диска можно использовать пакет livecd-tools, для записи образа на USB-накопитель нужно использовать liveusb-creator; Пакет yum-autoupdate для автоматизации ежедневного обновления системы; Поддержка дополнительных репозиториев: adobe, atrpms, elrepo, epel и rpmforge; Добавлено несколько скриптов для изменения принятых по умолчанию настроек: SL_desktop_tweaks (изменение оформления меню GNOME и KDE), SL_enable_serialconsole (включение входа через последовательный порт), SL_no_colorls (выключение цветного вывода для команды ls), SL_password_for_singleuser (включение необходимости ввода root-пароля для входа в однопользовательский режим (single mode)); Пакет визуализации Graphviz; Библиотека Numpy, для организации научных расчетов; Для драйвера iwlwifi включены дополнительные прошивки (firmware для беспроводных карт Intel серий 5150, 3945, 4945, 5000 и, начиная с текущей версии, 6000; Поддержка файловой системы XFS для архитектуры x86_64. В программе установки пока нельзя создать XFS раздел, но установить Scientific Linux на ранее созданный XFS раздел можно; FUSE-модули fuse-smb и fuse-sshfs, для монтирования SMB и SSH; Стоит напомнить, что большинство ранее доступных в Scientific Linux 5.x научных приложений не включены в базовые репозитории Scientific Linux 6.x, но их можно найти в дополнительных репозиториях, таких как EPEL, elrepo.org и atrpms.net. Кроме того, прекращена поддержка репозитория contrib, в котором размещались дополнительные драйверы устройств - аналоги пакетов из contrib также можно найти во внешних репозиториях elrepo и atrpms. Ранее присутствующие в SL5 пакеты, которые можно найти во внешнем репозитории EPEL: Библиотека cfitsio с реализацией поддержки формата FITS (Flexible Image Transport System), широко используемого в астрономических исследованиях, для языков Си и FORTRAN; Технология DKMS (Dynamic Kernel Module Support), позволяющая автоматически перестраивать текущие модули ядра с драйверами устройств после обновления версии ядра. Внедрение DKMS позволит автоматически устанавливать драйверы, которые необходимы для работы определенных аппаратных компонент текущей системы, но отсутствуют в стандартном Linux ядре. Для разработчиков, DKMS предоставляет средства создания драйверов без оглядки на особенности различных версий Linux ядра. Консольный почтовый клиент Alpine; Дополнительные Perl-модули: DBD-XBase, MailTools, Parse-RecDescent, SQL-Statement, TermReadKey, Text-CSV_XS, Text-Template, Tk; Ранее присутствующие в SL5 пакеты, которые можно найти во внешнем репозитории atrpms: Математическая библиотека FFTW (Fastest Fourier Transform); Утилита 915resolution для модификации видео BIOS для графических карт на базе GPU Intel 845G, 855G, 865G, 915G, 915GM и 945G, что позволяет добиться установки дополнительных разрешений экрана; GNU R (язык для статистических расчетов); Дополнительные мультимедиа кодеки (gstreamer-plugins-extras из SL4 и k3b-extras из репозитория livna); Библиотека матричных вычислений SuiteSparse; Ранее присутствующие в SL5 пакеты, которые придется загрузить с сайта разработчика: Научная библиотека SciPy, которую можно использовать для научных, инженерных и математических рассчетов, в также для построения графиков и диаграмм, на основе набора статистических данных; Sun Java/JDK - в составе теперь поставляется только OpenJDK; Набор обучающих приложений для KDE - KDEEDU.
