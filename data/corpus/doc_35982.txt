В Fedora 19 одобрена замена MySQL на MariaDB и поддержка CRIU, но не принят переход на Btrfs

На состоявшемся вчера заседании комитета FESCo (Fedora Engineering Steering Committee), отвечающего за техническую часть разработки дистрибутива Fedora Linux, были приняты решения, касающиеся ряда новшеств, предложенных для реализации в Fedora 19. В частности, было утверждено предложение по замене MySQL на СУБД MariaDB, которая будет предлагаться по умолчанию при желании установить пакет с MySQL, при этом пакеты с MySQL пока останутся доступны в качестве дополнительной опции. Вторым интересным изменением, одобренным для включения в Fedora 19, является поддержка средств для манипуляции snapshot-ами приложений в пространстве пользователя с использованием системы CRIU и инструментария CRtools, развиваемых отечественной компанией Parallels. Указанный инструментарий позволяет организовать заморозку состояния для одного или группы процессов, с последующим восстановлением работы с сохранённой позиции, в том числе на другом компьютере или после перезагрузки без разрыва уже установленных сетевых соединений. Среди других одобренных предложений можно отметить поставку JRuby 1.7 и Ruby 2.0.0; включение библиотеки MEMSTOMP для выявления некорректного использования функций подобных memcpy и strcpy; задействования новой предсказуемой схемы наименования сетевых интерфейсов, реализованной в systemd 197. Предложение по обеспечению поддержки обновления дистрибутива до новой версии при помощи команды "yum upgrade" отклонено. Что касается многократно отклоняемого плана по задействованию по умолчанию файловой системы Btrfs, то на этот раз данная идея не была принята ещё на этапе начального обсуждения, поэтому не была оформлена в качестве предложения для Fedora 19 (последний срок приёма предложений истек 30 января). Из предложенных, но ещё не рассмотренных изменений, можно отметить намерение предоставления возможности установки Apache OpenOffice, наряду с ныне используемым LibreOffice. Дополнение: разработчики openSUSE также приняли решение использовать MariaDB в качестве предлагаемой по умолчанию реализации MySQL, начиная с выпуска openSUSE 12.3. Поддержка MySQL будет сохранена, пользователи смогут установить из штатных репозиториев, но все зависимости требующих MySQL сторонних пакетов будут теперь связаны с MariaDB.
