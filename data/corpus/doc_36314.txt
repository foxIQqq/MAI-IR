Вышел X.Org Server 1.14

После 6 месяцев разработки анонсирован релиз X.Org Server 1.14, который содержит в основном большую порцию незаметных для конечного пользователя внутренних оптимизаций и исправлений. Наиболее заметным новшеством является реализация расширения X Input 2.3, в котором появилась поддержка определения границ для указателя (pointer barriers) и генерации событий ввода при их пересечении (Pointer Barrier Events). Границы указателя позволяют приложению определять дополнительные ограничения или привязать действия к перемещению курсора в определённую область, что может быть использовано композитными менеджерами и десктоп-окружениями при выводе фиксированных элементов интерфейса в определённой области экрана или для реализации "горячих" углов и краёв экрана, реагирующих на движение (вход или выход) указателя через границы области. Например, В GNOME через определение границ указателя в левом верхнем углу обеспечена возможность быстрого вызова меню, даже при использовании многомониторных конфигураций. В будущем выпуске GNOME 3.8 через использование событий пересечения границ указателя будет обеспечена возможность вызова области сообщений через скользящее касание от края сенсорного экрана, без перемещения курсора в угол. Вторым важным улучшением является поддержка расширения RandR 1.4, в котором добавлены свойства для регулировки края области вывода (например, можно задать область вывода меньше фактического размера экрана для компенсации забегания развёртки). В RandR 1.4 также реализованы объекты для выноса на дополнительные GPU операций рендеринга и вывода. Например, обеспечена возможность горячего перенаправления вывода на другое устройство (например, на подключенный видеоадаптер с интерфейсом USB), или в конфигурациях с несколькими GPU возможно использование первого GPU для вывода, а второго для рендеринга, или может быть организован совместный рендеринг на нескольких GPU. Из других улучшений можно отметить заметное ускорение программного рендеринга, благодаря использованию библиотеки pixman. К сожалению, в состав X.Org Server 1.14 не вошел компонент XWayland, позволяющий обеспечить запуск обычных X11-приложений поверх Wayland. Интеграция XWayland отложена до выпуска 1.15, который ожидается в сентябре.
