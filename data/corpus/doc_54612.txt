Компания Oracle выпустила ядро Unbreakable Enterprise Kernel R5U5

Компания Oracle выпустила пятое функциональное обновление для ядра Unbreakable Enterprise Kernel R5, позиционируемого для использования в дистрибутиве Oracle Linux в качестве альтернативы штатному пакету с ядром из Red Hat Enterprise Linux. Ядро доступно для архитектур x86_64 и ARM64 (aarch64). Исходные тексты ядра, включая разбивку на отдельные патчи, опубликованы в публичном Git-репозитории Oracle. Пакет Unbreakable Enterprise Kernel 5 основан на ядре Linux 4.14 (UEK R4 базировался на ядре 4.1, а UEK R6 на 5.4), которое дополнено новыми возможностями, оптимизациями и исправлениями, а также проверено на совместимость с большинством приложений, работающих в RHEL, и специально оптимизировано для работы с промышленным программным обеспечением и оборудованием Oracle. Установочные и src-пакеты с ядром UEK R5U5 подготовлены для Oracle Linux 7 (нет никаких препятствий по использованию данного ядра в аналогичных версиях RHEL, CentOS и Scientific Linux). Ключевые улучшения: Проведена оптимизация кода, отвечающего за чистку кэша страниц памяти в гипервизоре KVM, которая позволила повысить производительность крупных гостевых систем и сократить время их запуска. Устранены ошибки и внесены улучшения в код файловых систем btrfs, CIFS, ext4, NFS, OCFS2 и XFS. В RDMA повышена производительность failover/failback-переключений RDS (Reliable Datagram Sockets) в случае сбоев. Добавлены новые средства отладки RDS, выполняющие трассировку при помощи eBPF и DTrace. В securityfs добавлен интерфейс /sys/kernel/security/lockdown для управления lockdown-режимом Secure Boot, который ограничивает доступ пользователя root к ядру и блокирует пути обхода UEFI Secure Boot. Обновлены драйверы устройств, в том числе задействованы новые версии драйверов для LSI MPT Fusion SAS 3.0, BCM573xx, Intel QuickData, Intel i10nm EDAC, Marvell PHY, Microsoft Hyper-V и QLogic Fibre Channel HBA.
