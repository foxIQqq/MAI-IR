Компания Oracle убрала ограничение по использованию JDK в коммерческих целях

Компания Oracle изменила лицензионное соглашение на пакет JDK 17 (Java SE Development Kit), в рамках которого предоставляются эталонные сборки инструментов для разработки и выполнения приложений на языке Java (утилиты, компилятор, библиотека классов и среда исполнения JRE). Начиная с JDK 17 пакет поставляется под новой лицензией NFTC (Oracle No-Fee Terms and Conditions), которая разрешает бесплатное использование в персональных и коммерческих проектах, а также допускает применение в рабочих окружениях коммерческих систем. Более того, сняты ограничения по подтверждению операций загрузки на сайте, что позволяет загружать JDK автоматически из скриптов. Лицензия NFTC также подразумевает возможность бесплатного ежеквартального получения обновлений с устранением ошибок и уязвимостей, но данные обновления для LTS веток будут выпускаться не весь срок сопровождения, а лишь ещё год после выпуска следующей LTS-версии. Например, Java SE 17 будет поддерживаться до 2029 года, но бесплатный доступ к обновлениям прекратится в сентябре 2024 года, спустя год после выпуска Java SE 21 LTS. Что касается распространения JDK сторонними поставщиками, то оно разрешено, но если пакет поставляется не для получения прибыли. Свободный пакет OpenJDK, на базе которого Oracle формирует JDK, продолжит развиваться на прежних условиях под лицензией GPLv2 с исключениями GNU ClassPath, разрешающими динамическое связывание с коммерческими продуктами. Напомним, что с 2019 года для JDK применялось лицензионное соглашение OTN (Oracle Technology Network), которое допускало бесплатное применение только в процессе разработки ПО, для персонального использования, тестирования, создания прототипов и демонстрации приложений. При использовании в коммерческих проектах требовалась покупка отдельной лицензии.
