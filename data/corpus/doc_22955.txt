Увидел свет дистрибутив Arch Linux 2009.08

После шести месяцев разработки вышел релиз дистрибутива Arch Linux 2009.08. Установочные сборки дистрибутива доступны для архитектур i686 и x86_64 в двух вариантах: для установки с задействованием сети (170 Мб) и для автономной установки (350 Мб). Основные новшества представленного выпуска: Дистрибутив перешел на использование нового инсталлятора, основанного на ранее включенном в состав дистрибутива фреймворке AIF (Arch Linux Installation Framework). В новой программе установки представлены следующие улучшения: Переработка редактора дисковых разделов, в котором теперь можно осуществлять настройку шифрованных разделов (dm_crypt) и управлять LVM томами; Поддержка автоматизированного режима инсталляции, не требующего участия человека, на основе заранее подготовленного сценария установки (файла ответов). Разработан инструментарий для создания собственных процедур инсталляции; Улучшена функция установки даты/времени с задействованием NTP сервиса; Проведена работа в плане улучшения удобства работы (юзабилити); Переработаны компоненты, связанные с установкой загрузчика grub. Переход на использование Linux ядра 2.6.30.4 и пакета udev 141, с соответствующим обновлением драйверов и поддержкой нового оборудования; Обновление пакетного менеджера Pacman до версии 3.3.0, пакеты в дистрибутиве распространяются в виде tar.gz архивы с привязкой к внешней базе зависимостей; Переход на aufs2 реализацию файловой системы unionfs. В Aufs2 произведен полный редизайн внутренней структуры ФС, добавлен ряд дополнительных возможностей, исключены некоторые значительно усложняющие код функции. Усовершенствован процесс определения загрузочных устройств на начальном этапе загрузки; Установочный ftp-образ переименован в netinstall для избежания неоднозначных толкований; Проведена большая работа по исправлению ошибок, например, исправлена проблема с зависанием при работе с XFS разделами, налажена настройка параметров сети и зеркалирования.
