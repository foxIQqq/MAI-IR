Фонд Apache перевёл свои Git-репозитории на GitHub

Фонд Apache сообщил о завершении работы по интеграции своей инфраструктуры с GitHub и переносе на GitHub всех своих git-сервисов. Изначально для разработки проектов Apache предлагались две системы контроля версий: централизованная система управления версиями Subversion и децентрализованная система Git. C 2014 года на GitHub были запущены зеркала репозиториев Apache, доступные в режиме только для чтения. Теперь репозитории на GitHub переведены в разряд первичных и могут использоваться для внесения и рецензирования изменений. Собственные git-сервисы Apache переведены на работу в качестве запасных зеркал. Под эгидой Фонда Apache развивается более 350 проектов, суммарный размер активной кодовой базы которых превышает 200 млн строк, а общий накопленный за 20 лет архив изменений включает более миллиарда строк кода, охватывающих более трёх миллионов коммитов. Использование GitHub вместо собственной Git-инфраструктуры упростит работу над проектами и позволит использовать уже привычные многим новым разработчикам инструменты для передачи изменений, обсуждения и рецензирования кода, а также даст возможность организовать взаимодействие с разработчиками других проектов.
