Новые версии систем мониторинга Icinga 1.10 и Pandora FMS 5.0

Увидел свет релиз системы мониторинга Icinga 1.10, которая представляет собой форк системы мониторинга Nagios, отделившийся в начале мая 2009 года вследствие конфликта независимых разработчиков с компанией Nagios Enterprises. Исходные тексты Icinga распространяются в рамках лицензии GPL. Для использования подготовлены два варианта Icinga - с классическим интерфейсом, написанным на языке С++, и новым интерфейсом на языке PHP. Дополнительно поставляется пакет icinga-reporting, с реализацией аналитического модуля для генерации отчетов, написанного на языке Java и работающего вкупе с системой JasperServer. От Nagios проект Icinga отличается переработанной архитектурой, переведенной на использование прослоек IDOMOD и IDO2DB, позволяющих организовать хранение данных мониторинга и конфигурационной информации в СУБД MySQL, Oracle или PostgreSQL. Структура Icinga оптимизирована для развертывания распределенных систем мониторинга, при которой возможно создание нескольких агентов мониторинга, осуществляющих проверки и направляющих результаты на основной узел. В состав Icinga входит более гибкий и удобный модульный web-интерфейс, переписанный на языке PHP, активно использующий AJAX и представляющий статистику в виде графиков. Для обеспечения интеграции с внешними сервисами предусмотрено несколько API: XML, JSON, SOAP. Ключевые улучшения, добавленные в Icinga 1.10.0: В классическом web-интерфейсе, написанном на языке С++ и продолжающем развитие интерфейса Nagios, реализованы средства поиска по регулярным выражениям с динамическим выводом подсвеченных результатов по мере набора запроса. Кроме того, появилась поддержка кликабельных фильтров, добавлена возможность задания частичных групп сервисов, обеспечено отображение заданных пользователем переменных в областях просмотра информации о хосте и статусе. В новом web-интерфейсе, написанном на PHP, изменён стиль меню и разбивка на категории, обеспечена возможность удаления всех комментариев и отключения всех уведомлений сразу для нескольких элементов с истекшим временем жизни, добавлен виджет для отображения новых проблем с наглядной подсветкой типов состояния для хостов и сервисов. В модуле IDOUtils, предназначенном для организации хранения данных мониторинга в СУБД, добавлена возможность отключения обновления состояния для заданных пользователем переменных, реализованы средства для автоматической чистки таблиц с историей проблем. В ядре добавлена поддержка использования нескольких IDO-модулей. Одновременно доступен релиз системы мониторинга Pandora FMS 5.0, предназначенной для отслеживания производительности и проблем с работой сервисов в крупных сетях. Поддерживается как удалённый мониторинг с использованием SNMP, сетевых запросов и WMI-проверок, так и сбор информации через локально устанавливаемые агенты, которые доступны для Linux, BSD-систем, коммерческих Unix и Windows. Система обладает достаточно неплохой производительности и масштабируемостью - один сервер мониторинга способен контролировать работу нескольких тысяч хостов. Из особенностей Pandora FMS можно отметить: гибкие средства организации доступа к web-интерфейсу (отдельным пользователям можно делегировать доступ для отслеживания работы отдельных хостов), наглядные графические отчёты для контроля динамики изменения параметров, поддержка WMI (Windows Management Instrumentation), учёт показателей эффективности (KPI) по метрикам SLA и ITIL, возможность привязки к GIS, наличие средств для управления инвентаризацией, гибкие средства компоновки сводных страниц с обзором состояния. Система написана на языке Perl и распространяется в виде community-версии под лицензией GPLv2 и в виде расширенной коммерческой редакции. Основные новшества Pandora FMS 5.0: Metaconsole - портал самообслуживания (self-service), позволяющий пользователям самостоятельно управлять своими конфигурациями; Поддержка анализа трафика в режиме реального времени с использованием протокола Netflow. Предоставлены средства как для выявления аномалий и проблем, так и для формирования отчётов по IP-адресам и протоколам; Модуль BPM (Business Process Monitoring) с поддержкой визуального построения дерева сервисов с определением иерархической связи и зависимостей. Возможность задания пользовательских полей с параметрами событий и привязки выполнения заданных пользователем действий, таких как запуск ping или подсоединение по SSH к агенту, сгенерировавшему событие; Полностью переработанное оформление интерфейса, в котором вместо Flash для обеспечения интерактивности и динамического масштабирования применяются технологии HTML5, а для построения диаграмм используется библиотека Flot; Система централизованного сбора логов с использованием для доставки данных уже используемых агентов мониторинга; Новые плагины для мониторинга Informix, Hyper-V и XEN.
