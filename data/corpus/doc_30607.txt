Увидел свет дистрибутив Red Hat Enterprise Linux 6.1

Спустя полгода с момента выхода RHEL 6.0 компания Red Hat анонсировала выход дистрибутива Red Hat Enterprise Linux 6.1, в котором представлено исправление ошибок, реализация поддержки нового оборудования и добавление новых возможностей, при сохранении полной совместимости с программным окружением ветки RHEL 6.x. В анонсе сообщается, что в недавно проведенном тестировании инфраструктура виртуализации на базе KVM, построенная на базе Red Hat Enterprise Linux 6.1 и 20-ядерного сервера HP ProLiant BL620c G7, побила рекорд производительности в тестах SPECvirt_sc2010. Кроме того, в сотрудничестве с компанией IBM было проведено соответствующее требованиям SPEC тестирование, которое показало, что решение на базе Red Hat Enterprise Linux, Red Hat Enterprise Virtualization и оборудования IBM на 45% опережает конкурентов в тестах SPECvirt_sc2010 (7067 (432 VM) у KVM на IBM x3850 X5 против 3723 (228 VM) у VMware ESX 4.1 на ProLiant DL580 G7). RHEL 6.1 поставляется в следующих вариантах: Red Hat Enterprise Linux Client 6 для архитектур x86 и AMD64/Intel 64 Red Hat Enterprise Linux High Performance Compute Node 6 для архитектуры AMD64/Intel 64 Red Hat Enterprise Linux Server для архитектур x86, AMD64/Intel 64, IBM System z и IBM POWER; Red Hat Enterprise Linux Workstation для архитектур x86 и AMD64/Intel 64. Ключевые новшества Red Hat Enterprise Linux 6.1: Оптимизация производительности механизмов виртуализации, главным образом направленная на приближение производительности запущенных в виртуальных окружениях систем к скорости выполнения без использования средств виртуализации. Уменьшены задержки (latency) выполнения сетевых операций за счет встраивания соответствующей функциональности в ядро; Повышение производительности дискового ввода/вывода за счет поддержки механизма virtqueue-нотификации ioeventfd, используемого в KVM и позволяющего увеличить производительность virtio-драйверов; Поддержка кэширования мета-данных для образов виртуальных машин в формате qcow2, что позволяет сократить число выполняемых операций ввода/вывода; Оптимизация использования FPU-регистров; Расширение поддержки оборудования и увеличение надёжности, поддержка новейших многоядерных CPU и периферийных устройств Планировщик задач расширен средствами для более полной поддержки возможностей архитектуры NUMA; Добавлена возможность динамического добавления памяти и процессоров для некоторых серверных систем; Обеспечена поддержка интерфейса PCI express 3.0, позволяющего добиться более высокой пропускной способности; Обновлено большое количество разнообразных драйверов для сетевых адаптеров, устройств хранения и графических плат; Повышена операционная эффективность, расширены возможности системных сервисов, добавлены новые механизмы контроля за ресурсами и производительностью Добавлена поддержка ограничения интенсивности ввода/вывода через группы контроля ресурсами Cgroup; Увеличена эффективность выполнения сетевых операций на многоядерных системах за счет оптимизации привязки к CPU потока пакетов (пакеты в рамках TCP-сессий неразрывно обрабатываются одним и тем же ядром CPU); В device-mapper-multipath увеличена производительность обработки сбоев при обращении к хранилищу; Улучшена поддержка сетевых хранилищ, включая поддержку бриджинга для FCoE и offload-акселерации для iSCSI. Улучшена поддержка IPv6 в работе виртуальных сетей и DHCP-сервисов; Добавлена поддержка создания прозрачных прокси на базе squid; Новые инструменты для разработки и мониторинга. Расширение средств для диагностики, основанных на использовании системы динамической трассировки systemtap; В отладчике Gdb улучшена поддержка C++ и Python; В системе отслеживания потребления памяти Valgrind улучшена поддержка многоядерных CPU; Обновлена среда разработки Eclipse, добавлены новые средства для работы с точками останова и генерации кода для C/C++ и Java; Обеспечение повышенной отказоустойчивости Обеспечена поддержка инфраструктуры для обеспечения автоматической миграции KVM-систем на запасной сервер в случае сбоя первичного; Добавлен агент для мониторинга NFSv4-ресурсов; Значительно расширена документация.
