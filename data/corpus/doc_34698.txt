Дистрибутив SchilliX отказался от перехода на Illumos и сформировал новый форк OpenSolaris

Спустя два года с момента прошлого выпуска увидел свет релиз консольного дистрибутива SchilliX 0.8, использующего наработки OpenSolaris. Для загрузки подготовлен Live-образ, размером 407 Мб. Разработчики SchilliX отказались от ранее озвученных планов по переходу на кодовую базу проекта Illumos, в рамках которого создано полностью свободное и развиваемое независимым сообществом разработчиков ответвление от кодовой базы OpenSolaris. Формирование дистрибутива продолжено на основе оригинального кода OpenSolaris. В качестве причины называется пригодность использования Illumos только для построения решений для файловых серверов и продолжение поддержки только пакетов IPS (поддержка пакетов SVr4 прекращена), что нарушает следование идеям Solaris. Для продолжения разработки инициирован проект SchilliX-ON, в рамках которого будет поддерживаться новое ответвление от OpenSolaris, соответствующее классическим канонам Unix-систем и продолжающее использование пакетов SVr4. Сообщается, что по сравнению с последним опубликованным открытым выпуском OpenSolaris в рамках проекта SchilliX-ON добавлено около 150 тысяч строк кода, c реализацией поддержки новых возможностей. Из новшеств можно отметить: По умолчанию команда zone манипулирует нативными изолированными контейнерами Solaris, формируемых из пакетов SVr4 и повторяющих по версиям программ базовое корневое окружение. Поддержка IPS-зон сохранена и может использоваться в качестве альтернативы; Расширены возможности системных утилит, например, в ping добавлена поддержка указания интервалов в доли секунд, в команду more добавлена возможность отображения строк жирным шрифтом, улучшена работа команды man; Добавлена поддержка автоматическоого создания пакетов Svr4; Подготовлен скрипт pkgsort для формирования отсортированных списков пакетов Svr4; Утилита "pax" заменена на открытый аналог "star"; Добавлены свободные реализации K&R "cpp", "od" и "patch"; Расширены возможности "cstyle"; Возвращена поддержка команды fmli, ранее удалённой Sun; Устранены зависимости от закрытого фреймворка "wbem".
