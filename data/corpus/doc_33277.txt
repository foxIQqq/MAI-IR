Релиз BitTorrent-клиентов KTorrent 4.2 и Deluge 1.3.4

После года разработки представлен релиз BitTorrent-клиента KTorrent 4.2.0, оптимизированного для использования в графическом окружении KDE. Одновременно доступен релиз связанной с проектом библиотеки libktorrent 4.2. Отмечается, что в новой версии KTorrent и libktorrent нет значительных новшеств, но множество мелких изменений и улучшений. Наиболее заметные изменения в KTorrent: Поддержка проверки хешей отдельных файлов из состава торрента; Улучшенная поддержка потокового видео; Вместо нескольких torrent-вкладок теперь используется простой одностраничный интерфейс; Возможность скрытия загрузок, выгрузок и не поставленных в очередь торрентов в менеджере очередей; В основной список торрентов добавлена панель поиска. Изменения в libktorrent: Поддержка расширения для частичной раздачи (partial seeding); Работы по перемещению файлов с данными теперь помещаются в очередь; Увеличена производительность работы с распределённой хэш таблицей (DHT). Одновременно увидел свет релиз многоплатформенного BitTorrent-клиента Deluge 1.3.4, в котором отмечено около 70 исправлений, в основном связанных с устранением ошибок и недоработок. Deluge написан на языке Python (используется фреймворк Twisted), базируется на библиотеке libtorrent и поддерживает несколько видов интерфейса пользователя (GTK+, web-интерфейс, консольный вариант). BitTorrent работает в клиент-серверном режиме, при котором пользовательская оболочка выполняется в виде отдельного процесса, а всеми BitTorrent-операциями управляет отдельный демон, который может быть запущен на удаленном компьютере. Код проекта распространяется в рамках лицензии GPL. Установочные пакеты с Deluge 1.3.4 подготовлены для всех популярных Linux-дистрибутивов, а также для FreeBSD и Windows. Начиная с данного выпуска в состав инсталлятора для Windows включена библиотека Gtk+, что избавляет пользователей от необходимости отдельной установки Gtk+.
