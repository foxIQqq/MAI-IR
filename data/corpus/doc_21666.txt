Проект Fedora Directory Server сменил имя на 389 Directory Server

Разработчики Red Hat объявили об изменении названия проекта Fedora Directory Server, который теперь будет развиваться под именем "389". Непосредственно LDAP сервер будет именоваться "389 Directory Server", а подпроекты - "389 Admin Server", "389 Console" и т.п. Имя "389" выбрано по номеру сетевого порта, закрепленного за службой LDAP. В качестве причины изменения названия называется желание уйти от ассоциации LDAP сервера с проектом Fedora, так как продукт прекрасно работает и в других Linux дистрибутивах. Изначально бренд Fedora не был привязан к дистрибутиву и определял серию свободных продуктов компании Red Hat, таких как ОС Fedora Core, Fedora Directory Server, Fedora Certificate System. В настоящее время имя Fedora прочно закрепилось за дистрибутивом и рассматривается пользователями в имени других продуктов как что-то неотъемлемо связанное с Fedora Linux, что мешает продвижению пакетов.
