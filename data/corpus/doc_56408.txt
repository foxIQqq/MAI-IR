Выпуск дистрибутива для резервного копирования Rescuezilla 2.3

Доступен выпуск дистрибутива Rescuezilla 2.3, предназначенного для резервного копирования, восстановления систем после сбоев и диагностики различных аппаратных проблем. Дистрибутив построен на пакетной базе Ubuntu и продолжает развитие проекта "Redo Backup & Rescue", разработка которого была прекращена в 2012 году. Для загрузки предлагаются live-сборки для 64-разрядных систем x86 (846MB) и deb-пакет для установки в Ubuntu. Rescuezilla поддерживает резервное копирование и восстановление случайно удалённых файлов в разделах Linux, macOS и Windows. Выполняется автоматический поиск и подключение сетевых разделов, которые можно использовать для размещения резервных копий. Графический интерфейс основан на оболочке LXDE. Формат создаваемых резервных копий полностью совместим с дистрибутивом Clonezilla. При восстановлении поддерживается работа с образами Clonezilla, Redo Rescue, Foxclone и FSArchiver. Основные изменения: Осуществлён переход на пакетную базу Ubuntu 21.10. Дополнительно продолжает формироваться альтернативная LTS-сборка на основе Ubuntu 20.04. Добавлена поддержка верификации целостности резервных копий. Добавлена опция "rescue", позволяющая управлять игнорированием сбойных секторов и ошибок, выдаваемых файловой системой. Реализована поддержка восстановления и изучения дисковых образов, созданных в программе "Apart" (надстройка над partclone). Улучшен процесс сканирования образов. Добавлен пакет lxappearance, упрощающий включение тёмной темы оформления. В контекстное меню добавлена опция для запуска файлового менеджера с правами root. Для запуска файлового менеджера и браузера задействована утилита "xdg-open", вместо прямого запуска pcmanfm и Firefox. Добавлен перевод на русский язык.
