GCC удалён из основного состава FreeBSD

В соответствии с ранее намеченным планом, набор компиляторов GCC удалён из дерева исходных текстов FreeBSD. Сборка GCC вместе с базовой системой для всех архитектур была отключена по умолчанию в конце декабря, а теперь код GCC удалён из SVN-репозитория. Отмечается, что на момент удаления GCC все платформы, не поддерживающие Clang, переведены на использование внешнего сборочного инструментария, устанавливаемого из портов. В базовой системе поставлялся устаревший выпуск GCC 4.2.1 (интеграция более новых версий была невозможна из-за перехода 4.2.2 на лицензию GPLv3, которая была признана неприемлемой для базовых компонентов FreeBSD). Актуальные выпуски GCC, включая GCC 9, как и раньше, можно установить из пакетов и портов. GCC из портов также предлагается использовать для сборки FreeBSD на архитектурах, которые завязаны на GCC и не могут перейти на Clang. Напомним, что начиная с FreeBSD 10 базовая система для архитектур i386, AMD64 и ARM переведена на поставку по умолчанию компилятора Clang и развиваемой проектом LLVM библиотеки libc++. GCC и libstdc++ для указанных архитектур давно перестали собираться как часть базовой системы, но продолжили поставляться по умолчанию для архитектур powerpc, mips, mips64 и sparc64.
