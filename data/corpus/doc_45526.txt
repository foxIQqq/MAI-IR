Выпуск GnuPG 2.1.16

Доступен релиз инструментария GnuPG 2.1.16 (GNU Privacy Guard), совместимого со стандартами OpenPGP (RFC-4880) и S/MIME, и предоставляющего утилиты для шифрования данных, работы с электронными подписями, управления ключами и доступа к публичным хранилищам ключей. Напомним, что ветка GnuPG 2.1 позиционируется как развивающийся выпуск, в котором продолжают добавляться новые возможности, в ветке 2.0 допускаются только корректирующие исправления, а 1.4 является старой стабильной веткой, подходящей для устаревших систем и встраиваемых решений. Изменения в GnuPG 2.1.16: Реализован новый алгоритм выбора открытого ключа для email, указанного в опциях "-r", "-R" и "--locate-key"; Добавлена опция "--with-tofu-info" для вывода содержимого записи "tfs" в виде отформатированного списка ключей; Добавлена опция "--compliance", предложенная в качестве альтернативного пути указания опций "--rfc2440", "--rfc4880" и т.п. Упрощено использование опции "--quick-gen-key"; Добавлена опция "--override-session-key-fd"; Добавлена опция "--output" для сохранения верифицированных данных; Добавлена опция "--enable-special-filenames"; В agent и dirmng добавлен режим "--supervised", востребованный с systemd; В scd добавлена поддержка кард-ридеров и возможность генерации ключа ECC; В dirmngr добавлена опция "--allow-version-check" для фоновой загрузки базы версий ПО. В gpgconf добавлена команда "--query-swdb" для проверки версий ПО по загруженной копии БД; По умолчанию включена сборка gpg-wks-client и его установка в libexec; При настройке log-file теперь можно использовать формат "socket://" для обращения к логу через сокет "S.log".
