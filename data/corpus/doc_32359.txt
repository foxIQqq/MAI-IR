Релиз минималистичного дистрибутива Tiny Core Linux 4.1

Вышел минималистичный Linux дистрибутив Tiny Core Linux 4.1, который может работать на системах с 48 Мб ОЗУ. Загрузочный iso-образ занимает всего 11.5 Мб. Графическое окружение дистрибутива построено на основе X-сервера Tiny X, тулкита FLTK и оконного менеджера FLWM. Дистрибутив загружается целиком в ОЗУ и работает из памяти. Дополнительно поставляется консольная версия дистрибутива - Micro Core, занимающая 6 Мб и построенная на основе набора утилит BusyBox. Также доступна сборка MultiCore (47 Мб), снабженная графическим инсталлятором и поставляемая с набором дополнительных расширений. Кроме стандартной начинки Tiny Core и Micro Core в состав Multi Core включено два дополнительных загрузочных пакета: install.gz - инсталлятор с поддержкой установки Tiny Core Linux в виде файла на NTFS-раздел и с возможностью установки дополнительных расширений; network.gz - готовый набор инструментов для обеспечения выхода в сеть, включая менеджер для настройки Wifi-соединений. Основные улучшения, добавленные в версии Tiny Core Linux 4.1: В tc-config добавлен новый загрузочный код pretce для обеспечения загрузки с разделов mdadm RAID и LVM2; Обновление версий программ: busybox 1.19.3, pcmciautils-018, sudo-1.7.7, freetype-2.4.7, imlib2-1.4.5, libpng-1.2.46 В tc-config улучшена поддержка определения /dev/rtc и обработки loadcpufreq, добавлена возможность запуска microcore-редакции без X-сервера; Улучшена обработка ошибок в tce-setdrive; tc-terminal-server обновлён с учетом изменений в новом ядре; В mnttool добавлена опция "-g" для определения начальной позиции при использовании из .xsession; Обновлены приложения appbrowser, appsaudit, select, exittc, tc-functions, filetool.sh, rc.shutdown, tc-restore.sh, filetool GUI, и exittc GUI; Добавлена поддержка опций "-c" (проверка), "-l" (длинный экран) и "-s" (короткий экран); Обновлены правила udev для корректной обработки прав доступа к устройствам USB; В rebuildfstab улучшен код сканирования CD; В tce-setup добавлена поддержка поиска и загрузки дополнений с CD; Добавлены оптимизации сборки разделяемых библиотек для уменьшения занимаемого ими места; В базовую систему добавлен файл /etc/protocols.
