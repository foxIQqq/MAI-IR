Открыт код серверной части системы резервного копирования SEP Sesam

Компания SEP AG, поставщик программных решений резервного копирования и восстановления данных, объявила oб открытии под лицензией GPL исходного кода продукта SEP Sesam Storage Server, являющегося центральным компонентом своей системы резервного копирования SEP Sesam. SEP Sesam Storage Server состоит из двух модулей: Sesam Multiplex Stream Server (SMS), управляющий процессом записи на резервные носители, и Sesam Transfer Protocol Server (STPD), координирующий работу процессов резервного копирования между клиентами и SMS сервером. Обратим внимание, что в 2008 году компания начала редизайн своей системы резервного копирования для open source сообщества и выпустила клиентскую часть системы для POSIX-платформ под лицензией GPL. Таким образом, открытие центрального серверного компонента позволяет сообществу сторонников открытого исходного кода получить прямой доступ к важным алгоритмам и методологии хранения данных. По данным компании, открытие исходных кодов API и интерфейса командной строки серверного компонента системы теперь позволит пользователям системы получать прямой доступ к серверу STPD и "позволит разрабатывать свои собственные программные средства управления и создавать свои собственные уникальные среды резервного копирования". Открыв исходные коды программного обеспечения, компания утверждает, что она выполняет свои обязательства по европейскому праву, которое предусматривает, что все предприятия и учреждения должны иметь возможность восстановить и прочитать свои данные без третьего участника. Georg Moosreiner, председатель SEP AG, сказал: "Вот почему мы предоставляем open source сообществу исходный код нашего основного серверного модуля. По cути, сервер управления обменом STPD - мозг программной системы резервного копирования. Нашим клиентам теперь это прибавит уверенности в безопасности эксплуатации системы резервного копирования." Подробнее об открытых компонентах системы можно прочитать на wiki-сайте проекта. Клиент для ОС Windows, GUI и модули использования баз данных и корпоративных приложений остаются под собственной лицензией SEP AG.
