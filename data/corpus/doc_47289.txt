Выпуск растрового графического редактора Krita 3.3

Доступен релиз растрового графического редактора Krita 3.3, развиваемого для художников и иллюстраторов. Редактор поддерживает многослойную обработку изображений, предоставляет средства для работы с различными цветовыми моделями и обладает большим набором средств для цифровой живописи, создания скетчей и формирования текстур. Для установки подготовлены самодостаточные образы в форматах AppImage и Snap для Linux (для Ubuntu ожидается появление пакетов в PPA), а также бинарные сборки для macOS и Windows. Ключевые улучшения: Расширена поддержка платформы Windows, в том числе реализована возможность использования Windows 8 event API для работы с перьевым вводом N-Trig, доступным в некоторых ноутбуках Dell, HP и Acer; Переработана функциональность отображения с задействованием механизмов аппаратного ускорения. Добавлена возможность использования библиотеки Angle в Windows вместо прямого обращения к OpenGL, т.е. теперь можно использовать Direct3D через прослойку; Устранены визуальные артефакты, возникающие на экранах с высокой плотностью пикселей (Hi-DPI); Обеспечен перенос заголовка при вставке изображений через буфер обмена; Во время загрузки налажен процесс загрузки режимов смешивания и преднастроек кистей; До самого свежего выпуска обновлён плагин GMIC для Linux и Windows. Настройка пути к плагину упразднена и плагин загружается из директории с исполняемым файлом krita или подкаталога gmic; Добавлено определение типа файла по содержимому для устранения проблем, когда jpeg-файл сохранён с расширением png; Цветовые каналы в формате 16- и 32-разрядных чисел с плавающей запятой при экспорте изображения в PNG теперь преобразуются в 16-разрядные целые; Добавлена возможность сохранения в PNG-файле данных о канале прозрачности, даже если в самом изображении нет прозрачных пикселей; Обеспечено корректное обновление списка файлов после изменения; Добавлены новые опции командной строки: "--nosplash" для запуска без заставки, "--canvasonly" для запуска с отображением только содержимого холста, "--fullscreen" для запуска в полноэкранном режиме и "--workspace Workspace" для запуска на заданном виртуальном рабочем столе; Опция "Выделить Всё" перед выделением всего изображения теперь вначале отменяет прошлые выделенные области; Добавлена возможность расширения области выделения за границы холста; Проведена оптимизация производительности, устранены излишние операции чтения конфигурации, что сократило время создания миниатюр для слоёв и ускорило операции рисования при отключенном аппаратном ускорении; В macOS отключено аппаратное ускорение вывода для GPU AMD, так как оно приводило к зависаниям при сохранении PNG и JPG.
