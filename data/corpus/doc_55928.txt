Релиз дистрибутива для создания межсетевых экранов IPFire 2.27

Опубликован выпуск дистрибутива для создания маршрутизаторов и межсетевых экранов IPFire 2.27 Core 160. IPFire отличается простым процессом установки и организацией настройки через интуитивно понятный web-интерфейс, изобилующий наглядными графиками. Размер установочного iso-образа составляет 406 МБ (x86_64, i586, ARM, AArch64). Система модульная, кроме базовых функций пакетной фильтрации и управления трафиком для IPFire доступны модули с реализацией системы для предотвращения атак на базе Suricata, для создания файлового сервера (Samba, FTP, NFS), почтового сервера (Cyrus-IMAPd, Postfix, Spamassassin, ClamAV и Openmailadmin) и сервера печати (CUPS), организации VoIP шлюза на базе Asterisk и Teamspeak, создания беспроводной точки доступа, организации потокового аудио и видео-сервера (MPFire, Videolan, Icecast, Gnump3d, VDR). Для установки дополнений в IPFire используется специальный пакетный менеджер Pakfire. В новом выпуске: Проведена подготовка к удалению поддержки Python 2 в следующем выпуске IPFire. В самом дистрибутиве уже нет привязки к Python 2, но данную ветку продолжают использовать некоторые пользовательские скрипты. Для снижения задержек и повышения пропускной способности при интенсивной обработке трафика в сетевой подсистеме включено прикрепление обработчиков пакетов, сетевых интерфейсов и очередей к одним ядрам CPU для снижения миграции между разными ядрами CPU и повышения эффективности использования процессорного кэша. В движок межсетевого экрана добавлена поддержка перенаправления сервисов. Графики переведены на использование формата SVG. Предоставлена возможность использования web-прокси на системах без внутренней сети. В логе обеспечен показ названий протоколов вместо номеров.
