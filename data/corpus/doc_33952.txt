Представлен PhotoBook, новый менеджер фотографий для KDE

Даниель Николетти (Daniel Nicoletti), разработчик таких KDE-приложений, как Apper, Print Manager, colord-kde и libdebconf-kde, анонсировал проект PhotoBook, в рамках которого предпринята попытка создания минималистичного менеджера фотографий для KDE, напоминающего по своей сути программу Shotwell. PhotoBook базируется на библиотеке Qt, интерфейс оформлен с использованием QML, логика обработки изображений написана на языке Си++. Главной целью проекта является создание удобного и простого средства для управления большой коллекцией фотографий рядовых пользователей. PhotoBook позволяет объединить в рамках одной коллекции все созданные пользователями фотографии, раскиданные по локальной файловой системе, перенесённые на внешние накопители и размещённые в online-сервисах. Для навигации по коллекции используется привязка фотографий к категориям, которые формируются на основании таких критериев, как место съёмки, события, люди и т.п. Дополнительно фотографиям могут быть присвоены произвольные теги, по которым может производиться дополнительная выборка. В настоящее время подготовлен только начальный прототип с реализацией минимального набора функций, определяющих базовые очертания проекта. Что касается уже существующих для KDE приложений аналогичной тематики, упоминается, что DigiKam не устроил автора новой разработки нагромождением излишней функциональности и ориентацией не на простых пользователей, а на фотографов. Gwenview не устроил тем, что изначально развивался только как просмотрщик изображений, работающий с локальной ФС (при включении nepomuk можно использовать теги) и не поддерживающий абстрактную категоризацию групп фотографий по событиям. Наиболее близким к реализации потребностей автора оказался проект Shotwell с интерфейсом на GTK+, базовые функции которого и планируется повторить в PhotoBook, перенеся также некоторые полезные для конечных пользователей возможности DigiKam.
