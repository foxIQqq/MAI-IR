Новая версия VirtualBox 4.2.14. Планы по созданию Mesa-драйверов для VirtualBox и QEMU/KVM

Компания Oracle выпустила корректирующий релиз системы виртуализации VirtualBox 4.2.14, в котором отмечено 38 исправлений, направленных на увеличение стабильности и устранение регрессивных изменений. Из наиболее заметных улучшений можно отметить: Добавление обходного пути для предотвращения крахов, проявляющихся из-за добавления в ядро Linux 3.8 кода автоматической балансировки NUMA; В дополнениях к гостевым системам на базе Linux устранена ошибка, мешавшая сборке в Ubuntu с ядром Linux 3.2; В дополнениях к X11 снижена нагрузка на CPU при запуске клиента VBoxClient в режиме drag'and'drop; В дополнениях для гостевых систем решена проблема с копированием и вставкой данных между двумя гостевыми системами, запущенными на хост-системе с X11; Устранены крахи, проявляющиеся при изменении размера окна гостевой системы и при закрытии GUI; Для хост систем на базе Windows устранено зависание при передаче данных с некоторых USB-устройств. Налажен доступ к образам VHDX, созданным в Windows 8; Дополнительно можно отметить работу инженеров Oracle в направлении создания Mesa-драйверов для VirtualBox. Вместо поставки отдельного драйвера для задействования 3D-акселерации и OpenGL в гостевых окружениях, разработчики планируют создать Gallium3D-драйвер для Mesa и добиться его штатной поставки, как это сделала ранее компания VMware. В ответ разработчики из компании Red Hat выразили интерес к созданию Mesa-драйвера для QEMU, который можно будет использовать для задействования средств 3D-акселерации в окружениях на базе KVM (в настоящее время акселерация может быть обеспечена через эксклюзивный проброс видеокарты в гостевую систему и использования обычных видеодрайверов для данной карты).
