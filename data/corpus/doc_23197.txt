Canonical раскрывает планы, касающиеся нового менеджера пакетов

Незадолго до объявления о прекращении добавления новшеств в будущий релиз Ubuntu 9.10 компания Canonical выпустила первую версию своего универсального менеджера пакетов. В процессе разработки проект именовался AppCenter и сейчас его официальное название Ubuntu Software Store. О серьезности намерений Canonical по продвижению этого проекта говорит, хотя бы, тот факт, что планы по его улучшению и интеграции с другими пакетными менеджерами расписаны вплоть до релиза Ubuntu 11.04. Целью Ubuntu Software Store является объединение различных задач, которые регулярно выполняются при работе с инсталляционными пакетами, в рамках одного приложения с универсальным интерфейсом. Оно должно заменить такие разрозненные утилиты, как update-manager, Synaptic, gdebi, что существенно облегчит новым пользователям процесс установки приложений. Canonical надеется, что уже к выходу Ubuntu 9.10, Ubuntu Software Store будет обладать приятным графическим интерфейсом, быстрым поиском, предоставлять детальную информацию о пакетах и иметь интерактивную презентацию, рассказывающую как пользоваться утилитой. Судя по информации с Software Store Wiki начиная с релиза Ubuntu 10.04 LTS управление пакетами окончательно перейдет на новую платформу. А в релизе Ubuntu 10.10 добавится интеграция Ubuntu Software Store с сервисом совместной работы над приложениями Launchpad. Первое появление Ubuntu Software Store ожидается в Ubuntu 9.10 Alpha 5.
