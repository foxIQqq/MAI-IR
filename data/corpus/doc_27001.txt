Релиз ПО для синхронизации SyncEvolution 1.0

Через четыре года после выхода первой версии 0.1, вышел стабильный релиз SyncEvolution 1.0 - программы для синхронизации данных по протоколу SyncML, в которую встроен SyncML-сервер. Протокол SyncML позволяет независимо от платформы организовать синхронизацию разноплановой персональной информации, такой как адресная книга, список задач, заметки, между мобильным устройствами, различными приложениями и сервисами. Встроенный SyncML-сервер может работать как с транспортом HTTP, так и напрямую через Bluetooth. Проект создаётся при поддержке компании Intel и распространяется в рамках лицензии GPLv2. Из настольных приложений поддерживается синхронизации данных с GNOME Evolution и KDE PIM (в разработке поддержка Akonadi), сборки SyncEvolution также доступны для систем на базе MeeGo и Maemo 5 (Nokia N900). Основные изменения, относительно предыдущей версии (0.9.2): Реализована прямая синхронизация с телефоном по протоколу Bluetooth/OBEX; Добавлена поддержка Bluetooth/OBEX соединений с использованием obexd версии 0.19 и выше; Осуществлен запуск SyncEvolution в виде примитивного сервера HTTP SyncML (в прошлых версиях необходимо было настраивать отдельный SyncML-сервер); Налажена работа со многими сервисами синхронизации, такими как ZYB.com, Horde, Mobical.net, desknow.com, Funambol, Memotoo; В модуле Evolution пересмотрен внешний вид настроек, добавлена поддержка восстановления, осуществлен переход на использование нового D-Bus API. Для выбора присоединённого телефона можно воспользоваться интерфейсом GTK sync-UI, а также создать для телефона конфигурацию на основе доступных шаблонов. Шаблоны конфигураций включены только для телефонов Nokia; для телефонов других производителей см. HOWTO и информацию в Wiki. XML-настройки движка Synthesis разбиты на несколько частей, которые можно раздобыть в /usr/share/syncevolution/xml. Настройки в ~/.config/syncevolution-xml преобладают над настройками по умолчанию, что полезно при добавлении поддержки для нового телефона.
