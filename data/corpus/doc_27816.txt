Представлен серверный дистрибутив Zentyal 2.0

Увидел свет релиз серверного Linux-дистрибутива Zentyal 2.0, развиваемого ранее под именем "eBox Platform". Дистрибутив построен на пакетной базе Ubuntu 10.04 и ориентирован на создание серверов для обслуживания локальной сети предприятий среднего и малого бизнеса. Управление всеми аспектами работы дистрибутива производится через web-интерфейс. Поддерживается быстрая организация работы шлюза, межсетевого экрана, почтового сервера, VoIP (Asterisk), VPN-сервера, прокси (squid), файлового сервера, системы для организации взаимодействия сотрудников, системы мониторинга, сервера для резервного копирования и т.п. Zentyal поставляется как в виде отдельного установочного live-дистрибутива (460 Мб), так и в виде набора пакетов для Ubuntu. Из добавленных по сравнению с веткой 1.x улучшений, можно отметить: Переход на пакетную базу Ubuntu 10.04; Полная переработка модуля для управления установкой и обновлением программ через GUI-интерфейс. Дополнительно подготовлено несколько "мастеров" для автоматического конфигурирования программ, после установки их из пакетов; Увеличение производительности: переработана система для ведения логов; осуществлена миграция на новый бэкенд для управления конфигурацией, вместо gconf для хранения параметров конфигурации теперь используется БД Redis. Интерфейс пользователя стал более дружественным: вместо оконного менеджера Blackbox отныне используется графическое окружение LXDE; В состав включен новый модуль для организации групповой работы на базе платформы Zarafa (календарь, контакты, задачи и т.п.) с интерфейсом похожим на Microsoft Exchange и поддержкой синхронизации с мобильными устройствами; Добавлен новый модуль для создания FTP-сервера, позволяющий реализовать классическую альтернативу хранилищам на основе Samba; Реализован режим работы в роли сетевого моста, дистрибутив теперь может использоваться для прозрачного связывания локальных сетей, соединенных через интернет; В модуле для развертывания web-сервера добавлена поддержка HTTPS. Доступ к web-почте, groupware и сайту компании теперь осуществляется с шифрованием данных; Расширены возможности модуля для организации межсетевого экрана, появился новый интерфейс для просмотра и быстрого удаления правил, а также для хранения лога перенаправлений; Модуль печати переписан и теперь базируется на CUPS при поиске принтеров в сети; Модуль для поднятия Jabber-сервера теперь основан на использовании ejabberd и Zentyal MVC-фреймворка; В интерфейс пользователя добавлена возможность управления PAM, что позволяет легко настроить аутентификацию пользователей в LDAP; Для организации доступа в административный интерфейс теперь может быть заведено несколько аккаунтов.
