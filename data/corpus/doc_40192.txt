Проект VideoLAN выпустил libbluray 0.6.0 с поддержкой BD-J (BluRay Java)

После шести месяцев разработки проект VideoLAN представил новые выпуски библиотек libbdplus 0.1.1, libbluray 0.6.0 и libaacs 0.7.1. Выпуски libaacs 0.7.1 и libbdplus 0.1.1 устраняют накопившиеся ошибки, в частности решена проблема сборки с новыми версиями libgcrypt и устранены проблемы с производительностью при использовании libaacs с MMC. Libbluray 0.6.0 отмечен как значительный выпуск, в котором реализована корректная и пригодная для использования поддержка формата BD-J (BluRay Java), определяющего средства для организации интерактивного взаимодействия на языке Java для дисков Blu-Ray. Поддержка BD-J проверена в Linux, Windows и OS X, c использованием J2ME и J2SE, в реализциях OpenJDK и Oracle JDK. Среди других улучшений libbluray 0.6.0 выделяется возможность сборки без libxml. Библиотека libbluray предоставляет средства для интеграции в приложения функций по проигрыванию дисков в формате Blu-Ray. Кроме базовых функций библиотека поддерживает такие возможности, как навигацию, парсинг списка воспроизведения, работу с меню и поддержку BD-J. Библиотека libaacs содержит реализацию стандарта AACS (Advanced Access Content System), используемого для ограничения копирования оптических дисков. Библиотеку можно использовать как дополнение к libbluray для обеспечения проигрывания защищённых дисков Blu-Ray (требуется наличие валидных ключей AACS). Libbdplus представляет собой экспериментальную библиотеку с реализацией спецификации BD+, определяющей компонент системы DRM (Digital Rights Management) для защиты дисков Blu-ray от незаконного копирования и воспроизведения на неавторизированных устройствах. BD+ определяет работающую на устройствах воспроизведения виртуальную машину, в которой можно запускать исполняемые файлы, подготовленные провайдерами контента и размещенные на дисках Blu-ray. Программы могут анализировать окружение плеера, проверять лицензионные ключи и преобразовывать звуковые и видео потоки (без обработки такой программой части контента не будут нести осмысленной информации). Библиотеки соблюдают требования DRM (Digital rights management), не содержат ключей и не реализуют способы для обхода коммерческих методов защиты, поэтому для проигрывания защищенных с использованием технологий AACS и BD+ дисков требуется задействование дополнительных компонентов (ключи, сертификаты, файлы конфигурации). Код библиотек распространяется под лицензией LGPLv2.1. Поддерживается работа на платформах GNU/Linux, Windows и OS X.
