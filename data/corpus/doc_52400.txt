Выпуск VirtualBox 6.1.4

Компания Oracle опубликовала корректирующий релиз системы виртуализации VirtualBox 6.1.4, в котором отмечено 17 исправлений. Основные изменения в выпуске 6.1.4: В дополнениях для гостевых систем на базе Linux обеспечена поддержка ядра Linux5.5 и решена проблема с доступом через совместные каталоги (Shared folder) к примонтированным через loopback-устройство дисковым образам; Устранено появившееся в ветке 6.1 регрессивное изменение, вызывающее проблемы с использованием инструкции ICEBP на хостах с CPU Intel; Решена проблема с загрузкой гостевых систем с macOS Catalina, после установки обновления 10.15.2; Улучшена локализация GUI; Для USB налажена изохронная передача данных в виртуальную машину при использовании USB-контроллеров xHCI; Устранены проблемы с обработкой буфера последовательного порта, приводившие к приостановке приёма данных при сбросе очереди; Улучшена поддержка проброса последовательного порта в виртуальную машину на хостах с Windows; В VBoxManage возобновлена поддержка опции "--clipboard" в команде modifyvm; На хостах с macOS задействован более защищённый runtime и обновлён osxfuse (3.10.4); На хостах с Windows улучшена совместимость общих каталогов с определённой в POSIX семантикой дополнения файла (O_APPEND). Возобновлена возможность запуска VM через Hyper-V; В реализации BIOS обеспечена пометка готовности не-ATA дисков и добавлены данные о поддержке EFI в таблицу DMI. В VGA BIOS сокращён размер стека, используемого в обработчиках INT 10h.
