Открыт код для поддержки USB мониторов DisplayLink и документация для Marvell PXA3x0 SoC

Компания DisplayLink выпустила под лицензией LGPL исходные тексты библиотеки libdlo, предоставляющей API для взаимодействия с мониторами, подключаемыми через USB интерфейс, и переходниками "USB-to-monitor", основанными на чипах DisplayLink DL-120 (1280x1024) и DL-160 (1600x1200). Совместно с компанией Novell и проектом Linux Driver Project разработчики из DisplayLink планируют в ближайшее время подготовить полноценный набор драйверов для десктоп-систем и мобильных устройств, а также добиться включения драйверов в состав основного Linux ядра и X.Org. Ранее драйверы для мониторов DisplayLink были доступны только для платформ Windows и Mac. В целом разработанная в DisplayLink технология Virtual Graphics Card (VGC) имеет более широкие возможности, например, поддерживается трансляция сигнала в сжатом виде для внешнего монитора не только через USB, но и через стандартное сетевое соединение, установленное поверх Ethernet или WiFi. В дальнейшем ожидается адаптация драйверов для подключения внешнего монитора к планшетным ПК Nokia или к мобильному телефону HTC G1, оснащенным встроенным USB-хост контроллером. В заключение, можно упомянуть об открытии компанией Marvell доступа к технической документации по одночиповым системам PXA310 и PXA320 SoC. Ранее данные документы передавались разработчикам только после подписания соглашения о неразглашении (NDA). Ожидается, что открытие документации ускорит процесс разработки Linux драйверов для этих платформ. Ранее с подобной инициативой выступила компания Texas Instruments, выпустившая без ограничений полный комплект документации по платформе OMAP3530. Единственным производителем, требующим для получения документации подписания специального соглашения, остается Samsung, что сильно мешает разработке свободных драйверов для систем S3C6400 и S3C6410, в то время как документация на ранее выпущенные SoC S3C2410/2440 была доступна без каких-либо ограничений.
