Доступны Percona Server для MySQL 5.7 и Percona XtraBackup 2.4

Компания Percona объявила о выходе Percona Server для MySQL 5.7, расширенного варианта СУБД MySQL. Одновременно представлен релиз свободного продукта Percona XtraBackup 2.4, предоставляющего средства для "горячего" резервирования MySQL и других вариантов данной СУБД, включая Percona Server и MariaDB. Percona Server и Percona XtraBackup доступны для бесплатной загрузки и распространяются под свободной лицензией GPL. Percona Server 5.7 включает в себя все улучшения, добавленные компанией Oracle в MySQL 5.7 Community Edition, и полностью совместим с данной СУБД. Помимо этого, новая версия от Percona предлагает расширенные возможности резервного копирования, мониторинга, а также оптимизированный под операции записи движок нового поколения TokuDB, обеспечивающий более высокую степень сжатия данных и эффективность работы в облаке. Percona Server для MySQL 5.7 также сохраняет курс на повышение производительности, улучшение масштабируемости и обеспечение высокой доступности. Основные возможности Percona Server для MySQL 5.7: Улучшение возможностей мониторинга Статистика по таблицам, пользователям и индексам предоставляется с минимальным воздействием на производительность и обеспечивает более полное понимание активности на серверах, а также позволяет определять источники нагрузки. Расширенное логирование медленных запросов (slow query log) делает возможным более детальный анализ производительности запросов. Распределение времени ответа на запрос дает детальную информацию по запросам на чтение и запись отдельно для каждого вида запросов и в течение заданного промежутка времени. Производительность и безопасность Пул потоков (thread pool) обеспечивает масштабирование до более чем 10 000 соединений на сервер. Плагин аутентификации PAM (Pluggable Authentication Modules) выполняет аутентификацию пользователей MySQL. Плагин аудита предоставляет возможность мониторинга на основе политик и логирования активности соединений и выполнения запросов. Блокировки для резервного копирования обеспечивают создание резервных копий без существенного снижения производительности. Оптимизированный под операции записи движок TokuDB, позволяющий создавать транзакционные хранилища для MySQL. По сравнению с InnoDB новый движок обеспечивает более высокую степень сжатия данных и существенно опережает по скорости добавления больших объёмов данных. Расширенные возможности полноценной внешней аутентификации, логирование аудита и масштабирование на уровне пула потоков. Аналогичные функциональные возможности доступны только в коммерческой версии MySQL Enterprise Edition от Oracle. Percona Server для MySQL 5.7 интегрирован с целым набором других продуктов от Percona, включая Percona XtraDB Cluster, Percona XtraBackup, Percona Toolkit и др. Основные возможности Percona XtraBackup 2.4 Более высокая скорость – Percona XtraBackup делает физические копии данных, а не логические, что позволяет добиться более высокой производительности. Непрерывная обработка транзакций во время снятия резервных копий – Percona XtraBackup позволяет пользователям направлять запросы к базе во время снятия резервных копий. Неблокирующее снятие резервных копий онлайн в реальном времени – при использовании в связке с Percona Server для MySQL, Percona XtraBackup создает неблокирующие резервные копии онлайн в реальном времени на системах обработки транзакций, что позволяет сохранить доступ к приложениям в течение запланированного времени технического обслуживания. Решение также обеспечивает высокую степень сжатия и безопасности резервных копий. Работа со всеми версиями MySQL от любого разработчика. Неинтрузивное копирование для Percona XtraDB Cluster – поддержка использования в связке с Percona XtraDB Cluster для обеспечения надёжной стратегии резервного копирования в крупномасштабных кластерных средах. Percona XtraBackup является полностью открытым и бесплатным решением. Резервные копии хранятся в зашифрованном виде с возможностью управления гранулярностью доступа к данным. Это обеспечивает дополнительный уровень защиты для публичных облачных хранилищ и тех, которые находятся в общем доступе.
