Компания IBM открыла исходные тексты операционной системы FusedOS

Представлена новая открытая операционная система FusedOS, разработанная в исследовательском центре компании IBM и развивающая идею создания комбинированных систем, занимающих промежуточную нишу между операционными системами общего назначения, позволяющими выполнять любые приложения, и специализированными ОС, которые рассчитаны на запуск только определённой категории приложений. Рабочий прототип FusedOS доступен для запуска на суперкомпьютерах на базе архитектуры IBM Blue Gene/Q. Код FusedOS общедоступен и распространяется под свободной лицензией EPL (Eclipse Public License) с использованием некоторых компонентов под лицензией GPLv2. Целью открытия кода является привлечение сторонних исследователей, заинтересованных в развитии гибридных систем. FusedOS позволяет в окружении специализированных систем, таких как IBM Blue Gene/Q, обеспечить выполнение как обычных программ для Linux, так и специализированных приложений, рассчитанных на работу с ядрами вычислительных узлов кластеров IBM (CNK, Сompute Node Kernel). Иными словами, загрузка FusedOS преобразует раздел вычислительного узла в гибридный кластер, позволяющий запускать приложения, написанные для Linux и CNK. Работа обеих типов систем осуществляется бок о бок с разделением доступных ресурсов. При этом экземпляры ОС взаимодействуют друг с другом более тесно, чем при выполнении в виртуализированном окружении, вплоть до того, что процессы могут взаимодействовать с другими процессами, выполняемыми вне текущего экземпляра ОС. Linux-окружение имеет статус базового и позволяет подсоединиться к вычислительному узлу по SSH и выполнить любую штатную программу. Для запуска CNK-программ необходимо в Linux-окружении использовать специальную утилиту "cl". Для выполнения заданий на узлах кластера поддерживается использование популярного менеджера ресурсов SLURM, который может применяться для запуска CNK-приложений с поддержкой MPI. FusedOS также снимает ограничения CNK по запуску только экземпляров одного приложения на узле, позволяя одновременно выполнять разные CNK-программы. Использование FusedOS на узлах ввода/вывода позволяет экспортировать сетевую файловую систему, обеспечивающую единое представление данных для Linux- и CNK-приложений. В случае отсутствия внешней ФС, FusedOS может выполняться с использованием локального ram-диска.
