Релиз системы виртуализации VirtualBox 6.0

После года разработки компания Oracle опубликовала релиз системы виртуализации VirtualBox 6.0. Готовые установочные пакеты доступны для Linux (Ubuntu, Fedora, openSUSE, Debian, SLES, RHEL в сборках для архитектуры AMD64), Solaris, macOS и Windows. Основные изменения: Внесены многочисленные улучшения в интерфейс пользователя: Реализован новый графический интерфейс выбора виртуальных машин; Переработан интерфейс управления виртуальными носителями (Virtual Media Manager), в котором появились средства для управления такими атрибутами, как размер, местоположение, тип и описание; Предложен новый файловый менеджер, позволяющий работать с файловой системой гостевых окружений и копировать файлы между хост-системой и гостевым окружением; Добавлен Network Manager для упрощения управления сетями и их параметрами; Переработана панель работы со снапшотами, позволяющая управлять такими атрибутами, как имя и описание снапшота. Переработан блок с информацией о снапшоте, который теперь отражает отличия от текущего состояния виртуальной машины; Обеспечено сохранение макета расположения инструментов; Существенно улучшена работа на экранах с высокой плотностью пикселей (HiDPI) и обеспечено корректное масштабирование элементов интерфейса; Добавлена возможность раздельного включения записи звука и видео; Добавлен режим автоматизированной установки гостевых систем, похожий на функцию "Easy Install" в VMware и позволяющий без лишних настроек запустить гостевую систему, лишь выбрав необходимый для запуска образ; По умолчанию задействован драйвер графических карт VMSVGA (VBoxSVGA вместо VBoxVGA). Поддержка драйвера VMSVGA добавлена в дополнения для гостевых систем на базе Linux, X11, Solaris и Windows. Значительно улучшена поддержка 3D-графики в гостевых системах Linux, Solaris и Windows; Реализована возможность прозрачного изменения размера дисковых образов во время их добавления; Улучшена интеграция с Oracle Cloud. Добавлена возможность экспорта виртуальных машин в OCI (Oracle Cloud Infrastructure) через интерфейс командной строки (VBoxManage). Добавлена поддержка всех видов виртуальных машин OCI (VM.Standard2.*); Значительно усовершенствована звуковая подсистема: Добавлена возможность включения или выключения ввода/вывода звука на лету, а также возможность горячего подключения/отключения внешних звуковых карт и звуковых драйверов VRDE без остановки работы виртуальной машины; Реализованы функции определения звуковых устройств и автоматизированного выбора драйвера для звукового бэкенда; Добавлена поддержка настройки объёмного звука (пока используется только в Windows 10 Build 1809); Эмуляция звуковых устройств HDA переведена на обработку данных в асинхронном режиме с выполнением в отдельном потоке; Устранены искажения звука при использовании бэкенда PulseAudio; Решены проблемы с записью звука при использовании бэкенда ALSA; Улучшена обработка видеорежимов при использовании EFI; В подсистеме хранения добавлена поддержка функциональности Controller Memory Buffers для устройств памяти NVMe; Добавлен новый вариант эмулируемого устройства BusLogic ISA; Добавлена возможность смены прикреплённого последовательного порта без остановки виртуальной машины; Расширен API управления гостевыми системами; Добавлен обходной путь для обеспечения сетевого взаимодействия в старых гостевых системах, в которых для virtio PCI-устройств не включается Bus mastering. Для хостов и гостевых систем с Linux обеспечена поддержка ядра Linux 4.20; В VBoxManage добавлена поддержка настройки опций DHCP; Увеличена производительность системы совместного доступа к каталогам; В дополнениях для гостевых систем улучшено автомонтирование совместно используемых каталогов. Поддержка совместно используемых каталогов добавлена в дополнения для гостевых систем OS/2; Добавлена возможность подключения образов CUE/BIN в качестве виртуальных CD/DVD-носителей, содержащих несколько треков; Для доступа на хостах с macOS к содержимому дисков гостевых систем добавлена утилита vboximg-mount; Добавлена возможность использования гипервизора Hyper-V в качестве запасного движка для выполнения виртуальных окружений на хостах с Windows; Представлены результаты первого этапа переработки стека ввода/вывода.
