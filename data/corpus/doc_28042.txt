Вышла тринадцатая альфа версия Samba 4.0.0

Вышла тринадцатая альфа-версия проекта Samba 4.0.0 (двенадцатая альфа-версия была пропущена), развиваемого параллельно с веткой 3.x и нацеленного на реализацию возможности работы в качестве Active Directory контроллера доменов. По сравнению с прошлой альфа версией, вышедшей в начале января, можно отметить следующие улучшения: Переход на новую систему сборки, основанную на использовании Python-фреймворка WAF. Большое внимание уделено ускорению процесса сборки, упрощению управления и сокращению размеров бинарных файлов; Доработка winbind с целью полноценной реализации функций, необходимых для работы модулей "libnss_winbind" и "pam_winbind", позволяющих обеспечить маппинг параметров (пользователь/группа) windows-пользователей для Unix-подобных систем. Данные аккаунты можно использовать для таких сервисов, как "ssh", "login" и "Xsession"; Поддержка управления NT ACL-ами на стороне сервера через выполнение команды "net acl"; Поддержка динамического обновления данных в DNS. Поддержка взаимодействия с Microsoft AD DNS сервером пока на стадии разработки; Переработка кода с реализацией системного реестра (registry), устранено множество ошибок и проведено более полное тестирование; Осуществлен переход на новый релиз Kerberos HEIMDAL; Проведена работа по унификации кода DCE/RPC с целью совместного использования в ветках samba 3.x и 4.x. Подробный обзор ранее внесенных в ветку Samba 4.0.0 изменений доступен на данной странице.
