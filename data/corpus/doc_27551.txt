Сравнение производительности LightSpark и Adobe Flash в Ubuntu 10.04

Опубликованы результаты сравнения производительности Adobe Flash и Lightspark, свободного Flash-плеера, основанного на технологиях LLVM и базирующегося на использовании JIT-компилятора, транслирующего ActionScript код в x86-инструкции. По заявлению разработчиков Lightspark, за счет тесного использования OpenGL и JIT-компиляции нагрузка на систему при работе данного Flash-плеера заметно меньше, чем при просмотре того же ролика в Gnash или Adobe Flash. Тестирование показало, что на самом деле на текущей стадии развития работа Lightspark приводит к значительно большей нагрузке на CPU, чем Adobe Flash. Результаты тестирования: Проигрывание одного из популярных на YouTube Flash-ролика: Firefox + Flash: CPU 44%, ОЗУ 24MiB; Google Chrome + Flash: CPU 20%, 25.4MiB; Chromium + Flash: CPU 56%, 29MiB; Firefox + Lightspark: CPU 96%, 24.8MiB. Отображение контента насыщенного Flash-ем сайта: Firefox + Flash: CPU 58%, 40MiB; Google Chrome + Flash: CPU 50%, 29MiB Chromium + Flash: CPU 52%, 23MiB Firefox + Lightspark: 100% CPU, 29.7MiB (многие функции сайта, такие как меню, оказались недоступны); Flash-игра 1 (avsg): Firefox + Flash: CPU 58%, 9.3MiB Google Chrome + Flash: CPU 52%, 20MiB Chromium + Flash: CPU 54%, 15MiB Firefox + Lightspark: игра не запустилась Flash-игра 2 (bobulous): Firefox + Flash: CPU 72%, 36MiB Google Chrome + Flash: CPU 82%, 49MiB Chromium + Flash: CPU 86%, 45.6MiB Firefox + Lightspark: игра не запустилась Выполнение тестового пакета GUIMark2: Firefox + Flash: Bitmap 12.04 fps, Vector 15.34 fps; Google Chrome + Flash: Bitmap 12.53 fps, Vector 16.65 fps; Chromium + Flash: Bitmap 9.55 fps, Vector 15.51 fps; Firefox + Lightspark: тест не запустился;
