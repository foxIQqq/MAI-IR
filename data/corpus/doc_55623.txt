Десятая платформа ALT

Объявлен выпуск Десятой платформы ALT (p10) - новой стабильной ветки репозиториев ALT, основанных на репозитории свободных программ Сизиф (Sisyphus). Платформа предназначена для разработки, тестирования, распространения, обновления и поддержки комплексных решений всех уровней - от встроенных устройств до серверов предприятий и датацентров; создана и развивается командой ALT Linux Team, поддерживается компанией Базальт СПО. ALT p10 содержит репозитории пакетов и инфраструктуру для работы с восемью архитектурами: пятью основными (синхронная сборка, открытые репозитории): 64-битные x86_64, aarch64 (ARMv8), ppc64le (Power8/9) и 32-битные i586 и armh (armv7hf); тремя закрытыми (отдельная сборка, образы и репозитории доступны владельцам оборудования по запросу): e2k (Эльбрус-4С), e2kv4 (Эльбрус-8С/1С+), e2kv5 (Эльбрус-8СВ). Для 32-битной архитектуры mipsel ветка p10 не создается, поддержка в p9 осуществляется в заявленные сроки. Для архитектур e2k ответвление варианта для p10_e2k намечено на сентябрь 2021 года. В середине 2022 года планируется отделение ветки p10 для архитектуры riscv64. Сборка для всех архитектур производится нативно, без кросс-компиляции. Десятая платформа предоставляет пользователям и разработчикам возможность использования российских систем Байкал-М, Эльбрус, Элвис и совместимых, широкого спектра оборудования мировых производителей, в том числе мощных серверов POWER8/9 производства IBM/Yadro, ARMv8 производства Huawei, а также разнообразных одноплатных систем ARMv7 и ARMv8, включая распространённые Raspberry Pi 2/3/4. Особое внимание уделено свободным решениям, позволяющим корпоративным пользователям осуществить миграцию с проприетарной инфраструктуры, обеспечить преемственность единой службы каталогов предприятий и организаций, обеспечить удалённую работу современными средствами. Что нового Ядра реального времени: для архитектуры x86_64 собраны два realtime-ядра Linux: Xenomai и Real Time Linux (PREEMPT_RT). OpenUDS VDI: многоплатформенный брокер подключений для создания и управления виртуальными рабочими местами и приложениями. Пользователь VDI через браузер выбирает шаблон и с помощью клиента (RDP, X2Go) подключается к своему рабочему столу на терминальном сервере или в виртуальной машине в облаке OpenNebula. Расширение набора групповых политик: поддерживается параметры gsettings для управления рабочими средами MATE и Xfce. Центр администрирования Active Directory: admс - графическое приложение для управления пользователями, группами и групповыми политиками AD, аналогичное RSAT под Windows. Расширение платформы deploy, предназначенной для разворачивания и настройки ролей (например, PostgreSQL или Moodle). Добавлены следующие роли: apache, mariadb, mediawiki, moodle, nextcloud; при этом для ролей mediawiki, moodle и nextcloud можно изменять пароль администратора, не заботясь о внутренней реализации в том или ином веб-приложении. Добавлен alterator-multiseat -- модуль настройки многотерминального режима. Поддержка устройств на базе процессоров Байкал-М -- платы tf307-mb на процессоре Байкал-М (BE-M1000) с ревизиями S-D и MB-A0 с SDK-M-5.2, а также платы Lagrange LGB-01B (mini-ITX). Версии ядра Linux 5.10 LTS, 5.12 и linux-rt 5.10; GCC 10.3.1, glibc 2.32, llvm 12.0, systemd 249.1, selinux 3.2; python 3.9.6, perl 5.34, php 8.0, Rust 1.53, dotnet 6.0; samba 4.14 c dc, openUDS 3.0; GNOME 40.3, KDE 5.84, Xfce 4.16, MATE 1.24; Chromium-gost 92; Firefox 90; LibreOffice 7.2. Дополнительная информация по версиям содержится на вики и на pkgs.org; в августе 2021 года также можно ориентироваться на данные Repology и DistroWatch по Sisyphus. Состав и версии других пакетов также можно посмотреть на сайте packages.altlinux.org. Для догоняющих архитектур наличие и версии пакетов могут различаться. Обновление Обновление с версий 9.x коммерческих продуктов будет возможно в рамках договора после выхода версий 10.0 соответствующих продуктов. Перед обновлением до Десятой платформы ранее установленной системы следует обязательно ознакомиться с описанием. Рекомендуем проводить массовое обновление строго после успешной пробной проверки. Доступны стартовые наборы и шаблоны для различных архитектур и систем контейнеризации/облакообразования (dockerhub, linuxcontainers); новые дистрибутивные продукты для различных категорий пользователей ожидаются осенью 2021 года.
