Разработчики LibreOffice представили новую концепцию модульного интерфейса

Организация Document Foundation, курирующая разработку LibreOffice, представила новую концепцию построения интерфейса свободного офисного пакета, развиваемую под кодовым именем MUFFIN (аббревиатура, образованная от My User Friendly & Flexible INterface). Основной идеей предложенной концепции является предоставление гибкого решения для разных предпочтений пользователей, условий работы и видов оборудования. MUFFIN предлагает комбинацию различных элементов пользовательского интерфейса, оптимальных для различных классов пользователей, и даёт возможность пользователю решать, какой предпочесть интерфейс, вместо навязывания одного универсального внешнего вида. Первые наработки MUFFIN будут включены в состав LibreOffice 5.3. Пользователям LibreOffice 5.3 будет предложено четыре вида компоновки интерфейса: стандартная с панелями инструментов (по умолчанию), минималистичная с однострочной панелью, комбинация однострочной панели с боковой панелью, панель Notebook Bar в стиле MS Office (экспериментальная, требует изменения настроек): Стандартная панель, предлагаемая по умолчанию: Однострочная панель для экономии вертикального экранного пространства; Боковая панель, для широкоформатных мониторов; Панель NotebookBar, оформленная в стиле Ribbon и привычная для пользователей Microsoft Office. Контекстные группы в панели NotebookBar, предлагающие разные инструменты в зависимости от текущего режима работы. Основные задачи, которые попытались решить разработчики MUFFIN: Учёт предпочтений пользователей, которым необходим персональный интерфейс, предлагающий набор опций для сохранения уже выработанных привычек; Предоставление возможности выбора между различными уровнями дружественности пользователю без навязывания какого-то одного подхода. Наличие вариантов как для начинающих, которым необходима простота восприятия, так и для опытных пользователей, которым важна производительность выполнения работы и наличие быстрого доступа к расширенным возможностям; Гибкость адаптации для различных аппаратных платформ: предоставление оптимального интерфейса для ноутбуков с небольшими экранами, настольных систем с большими мониторами, широкоформатных экранов. Наличие выбора режимов, оптимизированных для экранов различного размера и разрешения, вместо попытки развития одного унифицированного интерфейса.
