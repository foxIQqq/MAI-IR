Релиз системы виртуализации VirtualBox 5.2.8

Компания Oracle сформировала корректирующий релиз системы виртуализации VirtualBox 5.2.8, в котором отмечено 22 исправления. В новой версии: В гостевых системах добавлена поддержка ядра Linux 4.15; Решена проблема с отображением чёрного экрана в гостевых системах с Linux при включенном режиме ускорения 3D; Обеспечена очистка флагов suid и sgid при доступе к совместно используемым каталогам из гостевых систем с Linux; В менеджер виртуальных машин для гостевых систем добавлена поддержка использования возможностей CPU FSGSBASE, PCID и INVPCID; В GUI-интерфейсе при работе в окружении Linux устранены проблемы, приводящие к смещению и уменьшению размера окна после изменения размера на экранах HiDPI; Внешние загрузки переведены на использование HTTPS; Устранён крах, проявляющийся при запуске мастера создания новой виртуальной машины; Добавлена возможность разделения источников записи звука в микшере PulseAudio при запуске нескольких виртуальных машин; Внесена порция исправлений в бэкенд DirectSound и в код эмуляции HDA; Улучшена совместимость функции записи сеанса на видео с различными видеопроигрывателями (например, были проблемы с воспроизведением в Firefox); Решены проблемы с обработкой образов VMDK, созданных в интерфейсе экспорта виртуальных машин Amazon EC2; Налажена поддержка PXE-загрузки при использовании эмулируемого адаптера e1000; Добавлено обходное решение для старых гостевых систем, не включающих режим bus mastering для PCI-устройств virtio; Добавлена переменная окружения CR_RENDER_FORCE_PRESENT_MAIN_THREAD для управления отрисовкой 3D-контента в основном потоке.
