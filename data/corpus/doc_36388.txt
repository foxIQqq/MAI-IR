BitTorrent тестирует расширение протокола для организации потокового вещания

Компания BitTorrent, Inc, отвечающая за развитие P2P-протокола BitTorrent и клиентского ПО μTorrent, объявила о начале тестирования нового P2P-протокола BitTorrent Live, ориентированного на организацию потокового вещания в live-режиме, используя традиционные принципы работы протокола BitTorrent для децентрализованной доставки контента. Организация вещания с привлечением систем пользователей в качестве узлов ретрансляции позволяет снять нагрузку с источника потокового вещания и сделать подобное вещание доступным обычным пользователям, без дополнительных требований к пропускной способности их каналов связи. Тестовые версии BitTorrent Live доступны в том числе и для Linux (пакеты для Ubuntu). При этом Linux-версия предоставляет не только графический интерфейс, но и поддерживает работу в режиме серверного процесса при запуске с флагом '--nogui' (видео трафик следует направить на 1935 сетевой порт в форме RTMP-потока, который может быть сгенерирован, например, пакетом ffmpeg). Дополнительно поставляется код примера для организации видеовещания из web-приложения на языке JavaScript.
