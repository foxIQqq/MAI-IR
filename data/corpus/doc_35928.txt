Шуточная инициатива по созданию редакции Fedora с ядром FreeBSD

Несколько участников проекта Fedora из компании Red Hat решили пошутить на фоне недавних предложений по задействованию по умолчанию окружения Cinnamon и замены MySQL на MariaDB, выступив с инициативой обеспечения поддержки в Fedora возможности использования ядра FreeBSD в качестве опции. Инициатива во многом напоминает проект Debian GNU/kFreeBSD, в котором пользовательские компоненты GNU запускаются поверх ядра FreeBSD. В заявлении говорится, что адаптация пользовательских компонентов Fedora для работы с ядром FreeBSD позволит предоставить дополнительную свободу выбора при компоновке системы, а также даст возможность задействования таких возможностей, как файловая система ZFS, пакетные фильтры PF и IPFW2, сетевая подсистемы NetGraph. Если бы шутка была воплощена в жизнь, пользователь смог бы на своё усмотрение выбирать какое ядро использовать в системе - Linux или FreeBSD. Отмечается, что для реализации задуманного в уже существующие пакеты потребуется внести лишь незначительные изменения, так как основная работа будет связана с созданием пакета с ядром FreeBSD и специфичных надстроек для обеспечения совместимости. Из системных изменений упоминается адаптация системных компонентов GNU, портирование системного менеджера systemd, создание интерфейса для выбора ядра в инсталляторе Anaconda, разработка бэкенда nm-platform-kfreebsd в Network Manager. Дополнение: к категории шуток также можно отнести проект Starch Linux, который по мотивам ArchBSD пытается создать вариант дистрибутива Arch Linux с ядром и базовыми компонентами Linux, но набором системных утилит из проекта OpenBSD. Несмотря на заявления на сайте проекта, в предложенном для установки репозитории пока можно найти только несколько простейших типовых BSD-утилит, представленных в пакетах coreutils (sed, tr, ed) и sbase (ls,cp, mkdir, cut и т.п.). Вместо специфичных утилит OpenBSD используются базовые утилиты Linux из пакетов util-linux, iproute2, pciutils, sysfsutils, usbutils и т.п. Следует отметить, что существует проект OBase, в рамках которого ведётся работа по адаптации базового программного окружения OpenBSD для работы с использованием ядра Linux и системной библиотеки Glibc. В настоящее время портировано уже более 200 утилит.
