Релиз дистрибутива Scientific Linux 6.3

Анонсирован релиз дистрибутива Scientific Linux 6.3, построенного на пакетной базе Red Hat Enterprise Linux 6.3 и дополненного средствами, ориентированными на использование в научных учреждениях. Дистрибутив поставляется для архитектур i386 и x86_64, для загрузки доступно несколько сборок: установочный DVD (3.5 Гб), полный комплект из двух DVD (4.2 Гб + 310 Мб) и сокращенный образ для установки по сети (161 Мб). Live-версия Scientific Linux 6.3 будет подготовлена позднее. Из отличий Scientific Linux 6.3 от версии 6.2, не связанных с заимствованием из RHEL, можно отметить: OpenAFS, открытая реализация распределенной ФС Andrew File System, обновлена до версии 1.6.1-112.sl6; В состав включён пакет yum-conf-rpmfusion для активации репозитория RPM Fusion; В пакете yum-conf-sl6x обновлён список GPG-ключей, добавлены ключи CERN; Вместо пакета redhat-logos добавлен linux-logos; В пакет sl-bookmarks добавлены дополнительные ссылки на полезные ресурсы не поддерживаемые сообществом Scientific Linux; Для настройки начальной конфигурации в пакете SL_enable_serialconsole теперь используется Augeas, при удалении пакета отныне удаляется и созданная конфигурация; В инструментарий revisor перенесены изменения, добавленные разработчиками инсталлятора anaconda. С целью упрощения переименованы kickstart-файлы для автоматизации установки. Добавлен скрипт для создания образа диска, содержащего все errata-обновления; Удалены пакеты: libproxy-mozjs (требует старой версии xulrunner), libvirt-qpid (требует старой версии qpid-cpp-client), sanlock и tboot (поддерживают только архитектуру x86_64). Общие отличия Scientific Linux 6.x от RHEL 6.x: Оконный менеджер IceWM 1.2.37; OpenAFS - открытая реализация распределенной ФС Andrew File System; Набор развиваемых в рамках проекта Fedora инструментов для создания Live-дисков и производных сборок. Для создания собственного Live-диска можно использовать пакет livecd-tools, для записи образа на USB-накопитель нужно использовать liveusb-creator; Пакет yum-autoupdate для автоматизации ежедневного обновления системы; Поддержка дополнительных репозиториев: adobe, atrpms, elrepo, epel и rpmforge; Добавлено несколько скриптов для изменения принятых по умолчанию настроек: SL_desktop_tweaks (изменение оформления меню GNOME и KDE), SL_enable_serialconsole (включение входа через последовательный порт), SL_no_colorls (выключение цветного вывода для команды ls), SL_password_for_singleuser (включение необходимости ввода root-пароля для входа в однопользовательский режим (single mode)); Пакет визуализации Graphviz; Библиотека Numpy, для организации научных расчетов; Для драйвера iwlwifi включены дополнительные прошивки (firmware для беспроводных карт Intel серий 5150, 3945, 4945, 5000 и, начиная с текущей версии, 6000; Поддержка файловой системы XFS для архитектуры x86_64. В программе установки пока нельзя создать XFS раздел, но установить Scientific Linux на ранее созданный XFS раздел можно; FUSE-модули fuse-smb и fuse-sshfs, для монтирования SMB и SSH; Отдельно стоит напомнить, что большинство ранее доступных в Scientific Linux 5 научных приложений не включены в базовые репозитории Scientific Linux 6, но их можно найти в дополнительных репозиториях, таких как EPEL, elrepo.org и atrpms.net. Также прекращена поддержка репозитория contrib, в котором размещались дополнительные драйверы устройств - аналоги пакетов из contrib можно найти во внешних репозиториях elrepo и atrpms. Ранее присутствующие в SL5 пакеты, которые можно найти во внешнем репозитории EPEL: Библиотека cfitsio с реализацией поддержки формата FITS (Flexible Image Transport System), широко используемого в астрономических исследованиях, для языков Си и FORTRAN; Технология DKMS (Dynamic Kernel Module Support), позволяющая автоматически перестраивать текущие модули ядра с драйверами устройств после обновления версии ядра. Внедрение DKMS позволит автоматически устанавливать драйверы, которые необходимы для работы определенных аппаратных компонент текущей системы, но отсутствуют в стандартном Linux ядре. Для разработчиков, DKMS предоставляет средства создания драйверов без оглядки на особенности различных версий Linux ядра. Консольный почтовый клиент Alpine; Дополнительные Perl-модули: DBD-XBase, MailTools, Parse-RecDescent, SQL-Statement, TermReadKey, Text-CSV_XS, Text-Template, Tk; Ранее присутствующие в SL5 пакеты, которые можно найти во внешнем репозитории atrpms: Математическая библиотека FFTW (Fastest Fourier Transform); Утилита 915resolution для модификации видео BIOS для графических карт на базе GPU Intel 845G, 855G, 865G, 915G, 915GM и 945G, что позволяет добиться установки дополнительных разрешений экрана; GNU R (язык для статистических расчетов); Дополнительные мультимедиа кодеки (gstreamer-plugins-extras из SL4 и k3b-extras из репозитория livna); Библиотека матричных вычислений SuiteSparse; Ранее присутствующие в SL5 пакеты, которые придется загрузить с сайта разработчика: Научная библиотека SciPy, которую можно использовать для научных, инженерных и математических рассчетов, в также для построения графиков и диаграмм, на основе набора статистических данных; Sun Java/JDK - в составе теперь поставляется только OpenJDK; Набор обучающих приложений для KDE - KDEEDU.
