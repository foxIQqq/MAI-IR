Выпуск дистрибутива EndeavourOS 22.1

Опубликован выпуск проекта EndeavourOS 22.1 "Atlantis", пришедшего на смену дистрибутиву Antergos, развитие которого было прекращено в мае 2019 года из-за нехватки у оставшихся мэйнтейнеров свободного времени для поддержания проекта на должном уровне. Размер установочного образа 1.8 ГБ (x86_64, отдельно развивается сборка для ARM). Endeavour OS даёт возможность пользователю без лишних усложнений установить Arch Linux с необходимым рабочим столом в том виде, каким он задуман в своей штатной начинке, предлагаемой разработчиками выбранного рабочего стола, без дополнительных предустановленных программ. Дистрибутив предлагает простой инсталлятор для установки базового окружения Arch Linux с рабочим столом Xfce по умолчанию и возможностью установить из репозитория один из типовых рабочих столов на основе Mate, LXQt, Cinnamon, KDE Plasma, GNOME, Budgie, а также мозаичных оконных менеджеров i3, BSPWM и Sway. Ведётся работа по добавлению поддержи оконных менеджеров Qtile и Openbox, рабочих столов UKUI, LXDE и Deepin. Также одним из разработчиков проекта развивается собственный оконный менеджер Worm. В новом выпуске: Обеспечен выбор дисплейного менеджера для установки в зависимости от выбранного оконного менеджера. Помимо ранее предлагаемой по умолчанию универсальной связки LightDM + Slickgreeter теперь также выбираются Lxdm, ly и GDM. В инсталляторе Calamares интерфейс выбора окружения рабочего стола отделён от этапа выбора пакетов для установки. В Live-сборке и установках с Xfce задействован набор пиктограмм и курсоров Qogir вместо ранее предлагаемого набора Arc. Добавлена кнопка для кастомизированной установки, позволяющая вручную включить дополнительные модули инсталлятора. Переписаны развиваемые проектом модули для инсталлятора Calamares - Pacstrap и Cleaner. В инсталлятор добавлена кнопка для управления показом журнала установки, а также реализован индикатор для оценки состояния установки в online-режиме. В Live-окружении по умолчанию включён Bluetooth, но после установки Bluetooth по умолчанию остаётся не активен. При выборе при установке файловой системы Btrfs сжатие данных теперь применяется и для размещаемых в процессе установки файлов (ранее сжатие включалось после установки). Включён динамический межсетевой экран firewalld, который запускается в виде фонового процесса, позволяющего динамически изменять правила пакетного фильтра через DBus, без необходимости перезагрузки правил пакетного фильтра и без разрыва установленных соединений. Добавлено новое графическое приложение EOS-quickstart, в котором предложен интерфейс для установки наиболее популярных программ, не входящих в базовый состав. Добавлена утилита EOS-packagelist, которая заменила собой интерфейс EndeavourOS-packages-lists, применяемый для доступа к спискам пакетов, используемых в инсталляторе. Добавлена утилита Nvidia-inst для упрощения установки проприетартных драйверов NVIDIA. В утилиту EndeavourOS-mirrorlist добавлена поддержка ранжирования зеркал для выбора ближайшего зеркала. В поставку добавлен оконный менеджер Worm, развиваемый одним из участников проекта. При разработке Worm была поставлена задача создания легковесного оконного менеджера, который бы хорошо работал, как с плавающими окнами, так и с мозаичной компоновкой окон, предлагая в обоих режимах кнопки управления окнами для минимизации, раскрытия и закрытия окна. Worm поддерживает спецификации EWMH и ICCCM, написан на языке Nim и может работать только с использованием протокола X11 (поддержки Wayland нет в ближайших планах).
