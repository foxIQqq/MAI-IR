Выпуск десктоп-окружения Cinnamon 4.2

После девяти месяцев разработки сформирован релиз пользовательского окружения Cinnamon 4.2, в рамках которого сообществом разработчиков дистрибутива Linux Mint развивается форк оболочки GNOME Shell, файлового менеджера Nautilus и оконного менеджера Mutter, нацеленный на предоставление окружения в классическом стиле GNOME 2 c поддержкой удачных элементов взаимодействия из GNOME Shell. Cinnamon основывается на компонентах GNOME, но эти компоненты поставляются как периодически синхронизируемый форк, не связанный внешними зависимостями с GNOME. Новый выпуск Cinnamon будет предложен в дистрибутиве Linux Mint 19.2, который планируется выпустить в ближайшие месяцы. В ближайшее время будут подготовлены пакеты, которые можно будет установить в Linux Mint и Ubuntu из PPA-репозитория, не дожидаясь новой версии Linux Mint. Основные новшества: Добавлены новые виджеты для создания конфигураторов, упрощающие написание диалогов настройки и делающие их оформление более целостным и единым с интерфейсом Cinnamon. Переработка настроек mintMenu с использованием новых виджетов позволила в три раза сократить размер кода за счёт того, что для задания большинства опций теперь достаточно одной строки кода; В MintMenu поисковая строка перемещена в верхнюю часть. В плагине для показа недавно открытых файлов документы теперь показываются в первую очередь. Значительно увеличена производительность компонента MintMenu, который теперь запускается в два раза быстрее. Полностью переписан интерфейс настройки меню, который переведён на API python-xapp; В файловом менеджере Nemo упрощён процесс предоставления совместного доступа к каталогам при помощи Samba. Через плагин nemo-share при необходимости обеспечивается установка пакетов с samba, помещение пользователя в группу sambashare и проверка/изменение прав доступа на совместный каталог, без необходимости ручного выполнения данных операций из командной строки. В новом выпуске дополнительно добавлена настройка правил межсетевого экрана, проверка прав доступа не только на сам каталог, но и на его содержимое, и обработка ситуаций с хранением домашнего каталога на зашифрованном разделе (запрашивает добавление опции "force user"). В оконный менеджер Muffin портированы некоторые изменения из оконного менеджера Metacity, развиваемого проектом GNOME. Проведена работа по увеличению отзывчивости интерфейса и более легковесному оформлению окон. Увеличена производительность таких операций, как группировка окон, и решены проблемы с подвисаниями при вводе. Изменение режима применения VSync для борьбы с разрывами при выводе (tearing) теперь не требует перезапуска Cinnamon. В настройки добавлен блок для выбора одного из трёх методов работы VSync, предоставляющих настройки для оптимальной работы в зависимости от условий использования и оборудования. В основной состав добавлен апплет для вывода на печать, который теперь запускается по умолчанию; Проведена ревизия и упрощение некоторые внутренних компонентов, таких как DocInfo (обработка недавно открытых документов) и AppSys (разбор метаданных приложений, определение пиктограмм для приложений, определение записи для меню и т.п.). Началась, но ещё не завершена, работа по выделению обоработчиков апплетов в отдельные процессы.
