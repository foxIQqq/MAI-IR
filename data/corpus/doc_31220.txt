Canonical упрощает систему сертификации совместимости Ubuntu с оборудованием

Компания Canonical объявила об изменении в системе сертификации аппаратного обеспечения для OEM и ODM производителей. Начиная с выпуска Ubuntu 11.10 упраздняется выдача сертификатов "Ubuntu Ready", выдаваться будут только сертификаты "Ubuntu Certified". По мнению Canonical практика выдачи двух похожих меток, указывающих на полную готовность оборудования к использованию Ubuntu, сбивает с толку пользователей, поэтому следует ограничиться только одним типом сертификатов. Инструментарий, используемый Canonical для проведения сертификации, как и раньше будет доступен производителям для проведения самостоятельного тестирования. Отличие "Ubuntu Ready" от "Ubuntu Certified" лишь в том, что в первом случае проверка дистрибутива производится один раз, а во втором случае дополнительно постоянно проверяется работоспособность и совместимость всех обновлений, выпускаемых в рамках цикла поддержки дистрибутива. Для получения сертификата "Ubuntu Certified" от производителей требуется предоставление оборудования для постоянного тестирования состояния релиза с учетом выпускаемых обновлений. Повторное тестирование, с целью исключения регрессивных изменений, проводится не реже, чем раз в две недели. В случае "Ubuntu Ready" оборудование для тестирования могло быть предоставлено один раз, только на время проведения проверки совместимости.
