Началось бета-тестирование платформы Red Hat Enterprise Virtualization 3.0

Компания Red Hat объявила о доступности бета-версии промышленной платформы для организации управления виртуальной инфраструктурой - Red Hat Enterprise Virtualization 3.0. Платформа базируется на Linux-дистрибутиве Red Hat Enterprise Linux 6 и использует в работе технологию виртуализации KVM (Kernel Virtual Machine). В состав продукта также входит web-консоль для управления виртуальной инфраструктурой, позволяющая создавать и конфигурировать виртуальные машины. Для организации работы тонких клиентов используется протокол SPICE. Как и для всех остальных продуктов Red Hat, код системы является свободным и доступен на условиях лицензии GPLv2. Новая версия имеет следующие ключевые особенности: Red Hat Enterprise Virtualization Manager теперь является Java-приложением, работающим на платформе JBoss Enterprise Application Platform. Ранее приложение базировалось на платформе .NET и было написано на языке С#. Проект перешел в руки Red Hat после покупки компании Qumranet в 2008 году; Вместо MS SQL Server используется PostgreSQL; KVM гипервизор работает на RHEL6; Поддерживается до 128 логических CPU и 2TB RAM для хостов, и до 64 vCPU/2TB RAM для гостевых систем (инструментарий VMware vSphere, вышедший в июле, поддерживает вдвое меньше процессоров и памяти); Портал управления позволяет создавать шаблоны виртуальных машин, выполнять массовое развёртывание; RESTful API для программного управления RHEV-M; Новые возможности для управления, улучшенный интерфейс для пользователей и администраторов, улучшено разделение прав пользователей; Поддержка использования локальных дисков для виртуальных машин (пока без возможности онлайн миграции); Новая встроенная система отчётности по использованию ресурсов систем; Улучшения в SPICE, включая поддержку динамического сжатия, автоматической подстройки глубины цвета и эффектов рабочих столов. Улучшена поддержка рабочих столов в Linux.
