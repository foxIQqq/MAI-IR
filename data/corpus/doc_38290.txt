Выпуск Apache Flex 4.11, продолжающий развитие технологии Adobe Flex

Представлен релиз Apache Flex 4.11, кроссплатформенного открытого фреймворка для создания интерактивных (Rich Internet Applications) web-приложений, идентично выполняемых как в web-браузере, так и в виде отдельных настольных программ или мобильных приложений. Apache Flex базируется на коде Adobe Flex, который был передан в 2011 году компанией Adobe фонду Apache. Основанная на Eclipse интегрированная среда для разработки Flash Builder остаётся проектом, подконтрольным Adobe. Apache Flex 4.11 полностью совместим с продуктом Adobe Flex SDK, и содержит всю доступную в нём функциональность, в том числе средства создания мобильных и web-приложений с использованием для разработки интерфейсов разметки MXML и языка ActionScript 3. Среди сильных сторон Flex, актуальных при разработке приложений для корпоративного сектора, отмечается обеспечение целостной и полнофункциональной поддержки работы на различных платформах, эффективность компонентов Flex и модели программирования при создании сложных интерфейсов пользователя, зрелость ActionScript как языка программирования для разработки больших приложений, наличие большого числа вспомогательных инструментов для редактирования кода, отладки и профилирования. Особенности Apache Flex 4.11: Добавлена поддержка свежих версий Flash Player 11.9 и AIR 3.9. Улучшена представленная в прошлом выпуске начальная поддержка выпусков AIR и Flash Player для платформы Linux; Поддержка экранов мобильный устройств с DPI 120 и 640; Оптимизация производительности классов mx:AdvancedDataGrid и mx:DataGrid; Обеспечен доступ к информации о сортировке столбцов в mx:Datagrid, s:Datagrid и mx:AdvancedDataGrid; Возможность лёгкой смены параметров рендеринга для элементов столбцов и строк в mx:AdvancedDataGrid; Обеспечено отображение s:DataGridEditor в результатах работы утилиты ASDoc и системе инспектирования тегов; Внесены исправления для организации сборки SDK компилятором Falcon; В BitmapImage обеспечен доступ к imageDecodingPolicy; Компонент интерфейса removedFromStageHandler перенесён из области private в protected для предоставления возможности переопределения; Новый экспериментальный datagrid-шаблон spark для мобильных устройств. В spark перемещён интерфейс Mobile Callout, который теперь может быть использован в настольных и браузерных приложениях; Поддержка тега "[Experimental]" для пометки экспериментальных возможностей; OSMF swc обновлён до выпуска 2.0; Экспериментальный пакет supportClazzes переименован в supportClasses; Исправлено около 50 ошибок.
