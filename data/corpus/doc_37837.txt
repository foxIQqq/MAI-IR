Вышел mdadm 3.3, инструментарий для управления программным RAID в Linux

Представлена новая значительная версия пакета mdadm 3.3, включающего в себя набор инструментов для управления программными RAID-массивами в Linux. Среди изменений: Улучшенная поддержка формата DDF (Disk Data Format), в том числе реализация поддержки RAID10; Возможность определения кофигурации в форме директории /etc/mdadm.conf.d с файлами "*.conf", загружаемыми в алфавитном порядке; Поддержка изменения некоторых типов массивов без промежуточного создания резервных копий; Поддержка переконфигурации RAID10 для изменения числа дисков, раскладки и размера chunk-а; Возможность использования опций "--assemble --update=metadata" для преобразования формата массива 0.90 в 1.0; Поддержка ведения лога сбойных секторов; Возможность использования опций "--assemble --update=revert-reshape" для отмены только что запущенной переконфигурации; Возможность использования нецифровых идентификаторов имён устройств, например, "md_home" вместо "md0"; Для просмотра информации об устройствах в массиве можно использовать опции "--detail --export"; Для резервного копирования и восстановления метаданных массива можно использовать опции "--dump" и "--restore"; Поддержка горячей замены устройств, например "mdadm /dev/mdX --replace /dev/foo --with /dev/bar".
