Компания Sun Microsystems выпустила OpenSSO Express 9 и OpenDS 2.2

Компания Sun Microsystems объявила о выпуске новой версии открытой системы для организации работы службы каталогов OpenDS 2.2, включающей в себя LDAPv3-совместимый сервер каталогов, написанный полностью на языке Java. Версии OpenDS доступны для платформ Solaris, OpenSolaris, серверов Windows 2003/2008, Red Hat, SUSE и Ubuntu Linux. Из новшеств, добавленных в OpenDS 2.2 можно отметить: Увеличение масштабируемости систем импорта и индексирования данных, на импорт 10 млн. записей отныне тратится около 30 минут; Внешний лог изменений теперь учитывает активность в рамках репликации данных - отражает поисковые запросы и получение обновлений; Поддержка режима частичной репликаций (Fractional Replication), позволяющей определить какие из атрибутов включать или наоборот исключать из репликации; Расширяемые правила сопоставления атрибутов с датой и временем, позволяющие создавать списки доступа привязанные к определенному промежутку времени; Поддержка определения пользовательского синтаксиса, созданного на основе правил замены, регулярных выражениях и перечислениях; В панель управления добавлена возможность удаленного управления OpenDS серверами; Улучшены средства управления репликацией и группировкой, система протестирована в конфигурации с 8 управляющими (master) серверами; В панель управления интегрированы средства для автоматизации создания резервных копий, через определение периодически вызываемых заданий; Реализован дружелюбный пользователям режим работы с Dsconfig-скриптами. Одновременно выпущена открытая платформа для организации управления доступом OpenSSO Express 9, основанная на ранее развиваемом проприетарном продукте Sun Java System Access Manager. Общими требованиями для платформ является наличие установленной Java SE 5.0. Для работы DSML шлюза необходимо иметь один из следующих серверов приложений: Sun Web Applications Server, Glassfish или Apache Tomcat.
