Определено кодовое имя для релиза Fedora 13 - "Goddard". Обзор планов

Определено кодовое имя для релиза Fedora 13. Большинством голосов представители сообщества одобрили присвоение будущему выпуску имени "Goddard". Имя присвоено в честь Роберта Годдарда, американского учёного, основоположника современной ракетной техники. Напомню, что прошлому релизу, Fedora 12, было присвоено имя "Constantine" в честь одноименного города в штате Мичиган, США. Результаты голосования: Goddard 1177 Langstrom 1009 Gloriana 977 Botany 922 Loana 707 Truro 654 Manfredi 504 Из новшеств, ожидаемых в Fedora 13 можно отметить: Улучшение интеграции файловой системы Btrfs, реализация механизма удобного восстановления через снапшоты. Например, при установке набора пакетов или проведении обновления, если что-то не понравилось можно легко откатиться на прошлое состояние системы; Переход на использование NFSv4 по умолчанию (вместо NFSv3) и поддержка монтирования NFS поверх IPv6; Автоматическая система для выбора и установки драйверов для принтеров. Пакеты подобные gutenprint-cups, hpijs и pbm2l2030 будут устанавливаться при обнаружении в системе оборудования, нуждающегося в данных наборах драйверов; Добавление в репозиторий среды разработки IntelliJ IDEA Community Edition; Улучшение поддержки DisplayPort в X-сервере и драйверах, работающих на уровне ядра, для карт NVIDIA и ATI Radeon; Подготовка набора пакетов на базе Python 3, которые могут быть установлены одновременно с Python-стеком на базе Python 2; Добавление в комплект VoIP сервера sipwitch, разработанного в рамках проекта GNU Telephony. В итоге планируется подготовить для пользователей Fedora полностью свободную альтернативу системе VoIP телефонии Skype, работающую в режиме точка-точка, т.е. для прямого соединения абонентов; Продолжение интеграции системы Systemtap (аналог DTrace для Solaris), добавление поддержки статичных проверок (static probes); Добавление yum плагина, который автоматически устанавливает языковые пакеты в соответствии с выбранным пользователем родным языком.
