Компания Oracle отменила изменение политики распространения обновлений часовых поясов

После прекращения жизненного цикла Java 6 компания Oracle решила закрыть публичный доступ к сервису tzupdater, в рамках которого осуществлялось оперативное распространение обновлений базы часовых поясов для Java SE, а также для обновления часовых поясов в jvm, которая встроена в СУБД Oracle. К счастью, решение по доступности подобных обновлений только платным подписчикам теперь отменено - возможность неограниченной загрузки tzupdater восстановлена. До этого предполагалось выпускать новые данные об изменениях в базе часовых поясов только в рамках новых версий Java SE, которые выходят несколько раз в год, чего явно недостаточно для некоторых стран. Данный шаг был воспринят как угроза, прежде всего, корпоративному сектору, где клиенты не готовы ждать "очередного обновления", а также компаниям, использующим старые версии jvm или организациям, в которых перед обновлением программного обеспечения производится долгий процесс проверки качества предлагаемых обновлений. При этом выполняемая tzupdater работа по преобразованию базы часовых поясов Олсона в специфичный для Java-формат и применение изменений к установленной версии Java SE не является незаменимой, так как все функции tzupdater могут быть выполнены вручную с использованием штатных подручных инструментов.
