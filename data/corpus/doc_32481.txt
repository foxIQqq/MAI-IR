Компания Intel выпустила графический пакет Intel 2011Q4

Компания Intel выпустила графический пакет Intel 2011Q4, включающий в такие компоненты графического стека, как драйвер xf86-video-intel 2.17, Mesa 7.11.2, libdrm 2.4.27, DRM-модуль для Linux-ядра 3.1, libva 1.0.15 и Vaapi-driver-intel 1.0.15. Релиз протестирован со следующими типами видеоподсистем: Ivy Bridge, Sandy Bridge HD Graphics 3000 (Core i7 2720QM, Core i7 2630QM, Core i5 2520M, Core i5 2500K), Sandy Bridge HD Graphics 2000 (Core i7 2600), Intel HD Graphics (Core i5 670, Core i5 520M), GMA 3150 (Atom N450), GM45, G45, GM965 и G33. По сравнению с прошлым выпуском отмечено включение в DRM-модуле ядра поддержки сжатия фреймбуфера (Frame Buffer Compression), которая включена по умолчанию для карт начиная с Sandy Bridge (для других карт для активации можно использовать i915.i915_enable_fbc=1). Поддержка режима энергосбережения RC6 пока отключена по умолчанию (не решены проблемы со стабильностью для некоторых систем при использовании механизмов виртуализации intel_iommu) и может быть активирована через передачу ядру параметра i915.i915_enable_rc6=1. Для всех карт отключена поддержка семафоров, которые могут быть включены в ручном режиме через параметр i915.semaphores=1. Решены проблемы со стабильностью для карт на базе Sandy Bridge и Ivy Bridge. Оптимизация работы с LLC-кэшем позволила достигнуть существенного увеличения производительности для Sandy Bridge и более новых графических карт. Улучшена поддержка вывода через Display Port, HDMI, LVDS и SDVO. В 3D-драйвере из состава Mesa улучшена поддержка OpenGL ES (GLES) и решены многие проблемы с поддержкой архитектуры Ivy Bridge (Gen7). В 2D-драйвере внесена большая порция исправлений в бэкенд UXA 2D, связанных с поддержкой архитектур Sandy Bridge и Ivy Bridge. Внесены изменения в экспериментальную реализацию архитектуры видеоакселерации SNA (SandyBridge New Acceleration), позволяющей в разы поднять производительность выполнения операций отрисовки. Код с реализацией VAAPI вынесен в отдельный репозиторий vaapi/intel-driver. Устранены проблемы с акселерацией декодирования потоков VC-1 и MPEG2.
