21 ноября в Санкт-Петербурге состоится конференция "Linux Piter 2015"

Сформирована программа конференции "Linux Piter 2015", которая пройдёт 21 ноября в Санкт-Петербурге. Конференция посвящена свободному программному обеспечению и ориентирована на инженеров IT-компаний, использующих GNU/Linux и сопутствующие технологии при разработке собственных готовых решений и продуктов. Формат конференции – один день докладов, разделенных на три параллельных потока. Участие платное. Студентам и преподавателям ВУЗов скидка 50%. Список утвержденных докладов: Swarm: Distributed storage for Ethereum, the Turing-complete blockchain / Daniel Nagy (ePoint Systems Ltd.). DPDK – The many way to configure the kernel interfaces / John Ronciak (Intel). NTRDMA v0.1 - An Open Source RDMA Driver for PCIe NTB and DMA / Allen Hubbe (EMC). Живая миграция контейнеров: плюсы, минусы, подводные камни. / Павел Емельянов (OpenVZ). Отладка производительности MySQL / Света Смирнова (Percona). Linux tuning to improve PostgreSQL performance / Илья Космодемьянский (PostgreSQL-Consulting LLC). История, опыт, ошибки и успехи в процессе создания по-настоящему масштабируемых систем хранения данных / Евгений Поляков. Усечение небезопасных графических операционных систем до стадии виртуализованных окружений рабочего стола / Дмитрий Костюк (Брестский Государственный Технический Университет). Оптимизация производительности в Linux: время удивительных историй / Александр Чистяков (Git in Sky). Приложение настраивает сеть: специфика, проблемы, решенияи / Василий Толстой (EMC). OpenStack CI: flows, tooling, and more / Aleksandra Fedorova (Mirantis). Container security / Алексей Залесов (Altoros). Персистентная память в Linux / Дмитрий Кривенок (EMC). Сетевой стек OpenOnload. В чем и почему он обыгрывает ядро Linux / ** Константин Ушаков (OKTET Labs). Макраме из контейнеров: mkimage-profiles / Михаил Шигорин (Базальт СПО). Самый SoC, linux, u-boot, грабли / Павел Курочкин (CTO). MariaDB MaxScale: how to improve database scalability by an intelligent database proxy / Timofey Turenko (MariaDB Corporation AB). Вертикальное масштабирование PostgreSQL / Дмитрий Васильев (Postgres Professional).
