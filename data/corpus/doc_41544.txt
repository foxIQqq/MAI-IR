В ночные сборки Firefox добавлен API для 3D-шлемов и изоляция NPAPI-плагинов

В ночные сборки Firefox приняты наработки проекта MozVR, в рамках которого развивается Web VR API, предназначенный для использования шлемов виртуальной реальности в web-приложениях и на сайтах. При помощи Web VR API разработчики могут воплотить в жизнь новые методы взаимодействия с пользователем и развивать новые техники 3D-навигации по сайтам. Интеграция с оборудованием выполнена через специальное дополнение, которое пока доступно только для устройств Oculus Rift. Работа над интеграцией поддержки Web VR ведётся не только для настольной версии Firefox, но и для Firefox for Android и Chromium. Предложенный Web VR API позволяет организовать отображение в 3D-шлеме контента, сформированного с использованием специальных VR-расширений WebGL. Дополнительно ведётся разработка VR-расширений CSS, которые позволят организовать размещение DOM-элементов внутри сцены виртуальной реальности. В качестве простейшего рабочего примера представлена система позиционирования плоского 2D-контента в форме цилиндра, окружающего пользователя. Другим появившимся в ночных сборках новшеством стал режим sandbox-изоляции для NPAPI-плагинов, который дополнил ранее добавленный экспериментальный режим изоляции отдельных вкладок. По умолчанию изоляция плагинов отключена и для применения требует изменения в about:config значения переменной dom.ipc.plugins.sandbox.default для включения изоляции всех плагинов. Для выборочного управления изоляцией отдельных плагинов можно использовать переменные dom.ipc.plugins.sandbox.имя_плагина (например, dom.ipc.plugins.sandbox.flash). Как и в случае изоляции вкладок, для ограничения плагинов задействованы наработки, используемые в Chromium и Chrome. В настоящий момент sandbox-изоляция NPAPI-плагинов пока доступна только для платформы Windows.
