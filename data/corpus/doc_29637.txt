Релиз свободной системы отслеживания ошибок Bugzilla 4.0

Выпущен релиз Bugzilla 4.0, написанной на языке Perl системы для ведения базы данных ошибок, контроля за их исправлением и общего координирования процесса разработки. Из новых возможностей можно отметить: Полностью переработанный интерфейс расширенного поиска, который теперь с одной стороны значительно упрощен и адаптирован для новичков, но в с другой стороны сохранил все дополнительные возможности для экспертов; Система автоматического выявления дубликатов, позволяющая выявлять близкие по значению записи на этапе добавления новых сообщений об ошибках; Поддержка выпадающего диалога автодополнения для всех полей с именем пользователя, email или ключевыми словами; Новый пользовательский интерфейс для просмотра, комментирования и редактирования информации о приложениях к записи; Проведение большой работы по улучшению юзабилити интерфейса, редизайн домашней страницы и новый набор графических пиктограмм; Возможность полного управления и редактирования ошибок через Web-сервис из других установок Bugzilla, включая функцию обновления уже ранее созданных заявок; При установке теперь используется новый набор статусов: не подтвержден (по умолчанию), подтвержден, в процессе, решен и проверен. Одновременно с выходом Bugzilla 4.0 объявлено о прекращении поддержки ветки Bugzilla 3.2.x, последним релизом которой останется 3.2.10.
