Инициатива по очищению кода Linux от запатентованных технологий Microsoft

Энтузиасты из проекта "Патент в день" задались целью создать патент-портфолио, в котором будут указаны те патенты Microsoft, которые гипотетически нарушает Linux. После этого разработчики надеются определить спорный код, заменить его и заняться пропагандой актуальности проблемы возможного нарушения патентов среди пользователей. При благоприятных условиях примерно через год из Linux может быть удален весь спорный код. За первые несколько дней существования проекта уже удалось получить данные по 40 патентам, которые могут быть использованы в открытых приложениях, поставляемых в стандартных Linux дистрибутивах. Два года назад компания Microsoft заявила о нарушении в Linux 235 патентов. Список данных патентов не был публично обнародован, а Microsoft пообещала не выдвигать претензий против конечных пользователей Linux и использовала данный набор патентных нарушений для оказания давления на коммерческие структуры. В частности, были заключены патентные соглашения с такими компаниями, как Novell, Fuji-Xerox, Samsung и Xandros. С большинством из производителей оборудования подобные сделки совершались с обязательным подписанием договора о неразглашении, но огласку подобная практика получила после конфликта к компанией TomTom, отказавшейся заключить договор на кросс-лицензирование технологий Microsoft.
